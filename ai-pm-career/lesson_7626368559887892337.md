# 教案：Agent Skill 与 MCP 的本质差异

> 来源视频：[抖音博主 · 抖音](https://www.douyin.com/video/7626368559887892337)
> 适合人群：准备 AI 产品经理面试、希望深入理解 Agent 架构的产品人和开发者

## 一、视频原文

请你说一下 Agent Skill 和 MCP 根本的差异是什么。

好的，面试官。我觉得这两个东西表面上看都像是在增强 Agent 能力，但是如果我们深层看的话，它解决的其实不是同一个问题。Agent Skill 更像是在教 Agent 怎么去做事，而 MCP 更像是给 Agent 接触外部世界的接口。

也就是说，Skill 解决的是能力、策略和行为范式的问题，MCP 解决的是标准化连接工具、数据和系统的问题。

但如果说得更直白一些，Skill 更偏方法论，MCP 更偏基础设施协议。

比如 Agent Skill，它的作用就是让模型在某类任务上表现得更稳定、更专业、更可复用。比如你可以给它一个协作写 PRD 的 Skill，那以后遇到这些问题，它就会按照这套步骤、结构和标准来做。也就是说，Skill 把更优秀的经验、工作流、行业 know-how 注入给模型，让模型知道这件事情应该按照什么套路来做。

但 MCP 不一样，MCP 本质上是一套开放的连接协议，通过 MCP，模型可以统一接入外部的工具，比如文档系统、数据库、日历、钉钉或者前端部署的系统。也就是说，MCP 不是在告诉模型如何思考，而是在解决模型怎么安全、标准化地调用外部能力这个问题。

所以从产品经理的视角来讲，我会把它理解为两个层次：Skill 是认知层和决策策略层，MCP 是连接层和能力共击层。

Skill 决定的是这个 Agent 在做某类任务时它应该遵循什么样的流程、输出什么样的结构、注意哪些规则；MCP 决定的是这个 Agent 它能够连接到哪些系统、读取什么样的数据、调用哪些工具。

Skill 即使没有外部系统也能发挥作用，因为它本质上是对模型行为的约束和增强。但 MCP 如果没有对应的服务器，这套连接架构本身是跑不起来的，因为它本质是一个协议机制，强调安全边界、权限控制和能力协调。

所以如果用一句话总结，我觉得：Skill 是在回答"这个 Agent 应该怎么做"，MCP 是在回答"这个 Agent 可以连到什么、调用什么"。

在真正做 AI 产品的时候，通常会把两者配合起来看：先用 MCP 把外部能力接进来，再用 Skill 把这些能力组织成一个稳定、可复用、可交付的工作流。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**Agent Skill 是教 Agent 怎么做事的方法论层，MCP 是让 Agent 连接外部世界的基础设施协议层，两者分工不同，配合使用才能构建完整的 AI 产品能力。**

---

### 2.2 知识地图

```
Agent Skill vs MCP
├── 表面相似
│   └── 都在增强 Agent 能力
├── 本质差异
│   ├── Agent Skill
│   │   ├── 定位：方法论 / 认知层 / 决策策略层
│   │   ├── 解决：能力、策略、行为范式
│   │   ├── 作用：注入经验/工作流/行业 know-how
│   │   └── 特点：无需外部系统也能发挥作用
│   └── MCP
│       ├── 定位：基础设施协议 / 连接层 / 能力共击层
│       ├── 解决：标准化连接工具、数据、系统
│       ├── 作用：统一接入文档/数据库/日历/钉钉等外部能力
│       └── 特点：需要对应服务器，强调安全边界和权限控制
└── 配合使用
    └── MCP 接入外部能力 → Skill 组织成稳定可复用工作流
```

---

### 2.3 核心概念精讲

#### Agent Skill（技能层）

**是什么**：注入给模型的经验、工作流和行业 know-how，让模型在某类任务上表现更稳定、更专业、更可复用。

**为什么重要**：模型本身是通用的，Skill 让它在特定领域按照最优路径执行，避免每次都从零摸索。

**详细解释**：

| 维度 | 说明 |
|---|---|
| 本质 | 对模型行为的约束和增强 |
| 解决的问题 | 能力、策略、行为范式 |
| 层次定位 | 认知层 / 决策策略层 |
| 依赖条件 | 无需外部系统也能工作 |
| 典型例子 | 协作写 PRD 的 Skill：按固定步骤/结构/标准执行 |

**实际应用场景**：给客服 Agent 注入一个"投诉处理 Skill"，它就会按照"先安抚→核实信息→给出方案→跟进确认"的固定流程处理，而不是每次随机发挥。

---

#### MCP（Model Context Protocol，模型上下文协议）

**是什么**：一套开放的连接协议，让模型能够统一、安全、标准化地接入外部工具和系统（文档、数据库、日历、钉钉等）。

**为什么重要**：模型本身无法直接访问外部世界，MCP 解决的是"连接"问题，而不是"思考"问题。

**详细解释**：

| 维度 | 说明 |
|---|---|
| 本质 | 开放的连接协议 / 基础设施 |
| 解决的问题 | 标准化连接工具、数据和系统 |
| 层次定位 | 连接层 / 能力共击层 |
| 依赖条件 | 需要对应的 MCP 服务器运行 |
| 核心强调 | 安全边界、权限控制、能力协调 |
| 典型例子 | 通过 MCP 让 Agent 读取数据库、操作钉钉、访问文档系统 |

**实际应用场景**：企业内部 Agent 通过 MCP 接入 OA 系统、CRM、知识库，实现跨系统的数据读取和操作，而不需要为每个系统单独开发接口。

---

#### Skill + MCP 的配合模式

**是什么**：先用 MCP 把外部能力接进来，再用 Skill 把这些能力组织成稳定、可复用、可交付的工作流。

**为什么重要**：单独使用任何一个都不完整——MCP 给了能力但没有策略，Skill 有策略但没有外部数据源。

**实际应用场景**：
- MCP：接入公司数据库、日历系统、钉钉
- Skill：定义"每日工作汇报"的生成流程（查日历→查任务→整合数据→按模板输出）
- 结果：Agent 每天自动生成符合规范的工作汇报，稳定可复用

---

### 2.4 速查表

| 对比维度 | Agent Skill | MCP |
|---|---|---|
| 本质 | 方法论 / 行为约束 | 连接协议 / 基础设施 |
| 层次 | 认知层 / 决策策略层 | 连接层 / 能力共击层 |
| 解决问题 | 怎么做（能力/策略/行为范式） | 连什么/调什么（工具/数据/系统） |
| 依赖 | 无需外部系统 | 需要 MCP 服务器 |
| 类比 | 给员工培训 SOP | 给员工开通系统访问权限 |
| 配合关系 | 组织能力成工作流 | 接入外部能力 |

---

### 2.5 延伸思考

1. 视频中说 Skill 是"对模型行为的约束和增强"，那么当 Skill 中定义的流程与实际业务场景不匹配时，Agent 应该如何处理？Skill 的灵活性边界在哪里？
2. MCP 强调"安全边界和权限控制"，在企业级部署中，如何设计 MCP 的权限体系，既能让 Agent 高效访问数据，又不引入安全风险？
3. 随着 MCP 协议逐渐标准化，Skill 的设计是否也会走向标准化？行业通用 Skill 库和企业私有 Skill 库应该如何分工？

---

### 2.6 一句话记住它

> Skill 教 Agent 怎么做事，MCP 给 Agent 通外部世界——先接能力，再组流程，两者缺一不可。
