## 一、视频原文

纯靠兴趣自学，直接拿下36K AI应用开发的offer，甚至只学了两个月。别不信，90%的人学AI不是学不会，是瞎学、乱学，把时间都浪费在了没用的知识点上。那今天我直接给你一套两个月转行AI应用开发的极简路线，纯干货，不看就是你的损失。

先问你一句话，25-26年最爆的赛道是什么？绝对是AI大模型。从最近的招聘市场来看，一个最直观的感受是：大厂在不断地推出AI应用开发的岗位，AI业务相关的新司远远高于传统业务。封口的出现往往意味着机会的来临，且不管未来AI能发展成什么样子，对于我们来说，我们没有知道过一次红利期，独一把又能有什么损失呢？

那第一个阶段，主要是打技术基础，为后面的项目做准备。比如首先选Python，你不要听别人说什么"加娃也能干"，主流是Python，我们就干主流。那你不要听人家说什么"Node.js不就是路到血的吗"，但它不是主流，主流还是Python。然后RAG检索增强、向量数据库、LangChain、Agent流程、提示词工程、大模型API，包括模型微调训练，这些都是前置的知识。当你掌握了这些，在初级市场找工作时，OK，已经够用了，但在公司就是重担一个扎扎实实的角色。

第二个是项目上手。很多人一上来把LangChain项目做十个，有什么用？万一你去面的公司人家不用LangChain，人家用Python原生的写法怎么办？所以项目结论，我们要考虑业务匹配度跟技术匹配度。那你在做的过程中就要考虑，我的项目整个流程怎么从零到一，AI的服务是怎么编排的，包括LangGraph、Skills、MCP的服务是怎么部署怎么设计的，还有项目的健全，包括整个项目的全流程。到这里你已经是项目骨干了，说你拿不到20K说出去信吗？

那我也把AI应用开发要学的那些整理成了一份学习路线，先学什么、后学什么、看什么教程、做什么项目，还有对应的学习教程，需要的私信安排。

那第三个阶段，我们要对整个项目去做持续的优化。首先你得去学习大模型的原理，比如KV缓存、前缀缓存、不合检索，那怎么提升召回率？以及我们项目做到一定阶段，可能还得要模型微调，将会学习SFT、DPO，以及我们为模型的数据，你要不要做数据清洗，包括评测集，那你怎么保障你的大模型输出的结果符合用户需求，这个就要评测集去解决。到这里你已经是项目负责人了，那第三个阶段对应市场上的30K，没什么问题。

第四个是Agent的集群，比如Agent的路由、跨Agent的记忆、可观测性、测试、怎么去鉴权，这些都是高级别、现在最需要的管理者。大家一定要对照这个路线去学，很多人学了很多知识点，到最后还是什么都不会，因为你的知识点是乱的，没有一个体系。你按顺序学两个月，不一定能让你一步登天，但足够让你从完全看不懂变成能做项目、能落地级、能面试。

### 2.3 核心概念精讲

#### RAG（检索增强生成）

**是什么**：让大模型在回答时，先从外部知识库检索相关内容，再生成答案的技术。

**为什么重要**：解决大模型知识截止和幻觉问题，是AI应用开发最核心的技术之一。

**详细解释**：
- 需要掌握：向量数据库（存储检索内容）+ 召回率优化（找到最相关内容）
- 第一阶段打基础，第三阶段深入优化召回率

**实际应用场景**：企业知识库问答、客服机器人、文档检索系统。

---

#### Agent流程与MCP

**是什么**：Agent是能自主规划并调用工具完成任务的AI系统；MCP是Agent连接外部工具的标准协议。

**为什么重要**：Agent是当前AI应用开发最热门的方向，招聘市场需求量大。

**详细解释**：
- 第二阶段：学会AI服务编排（LangGraph / Skills / MCP部署与设计）
- 第四阶段：进阶到Agent集群（路由、跨Agent记忆、可观测性）

**实际应用场景**：自动化工作流、多步骤任务执行、企业AI助手。

---

#### 模型微调（SFT / DPO）

**是什么**：在预训练大模型基础上，用特定数据进一步训练，使其更符合业务需求的技术。

**为什么重要**：是从"能用"到"好用"的关键跨越，对应30K薪资水平。

**详细解释**：
| 方法 | 说明 |
|------|------|
| SFT（监督微调）| 用标注数据直接训练模型 |
| DPO（直接偏好优化）| 通过人类偏好对比数据训练 |

**实际应用场景**：让模型输出符合公司业务规范、特定风格或领域知识。

---

#### 分阶段学习路线

**是什么**：将AI应用开发技能按难度和市场价值分为四个阶段，按顺序学习。

**为什么重要**：大多数人"瞎学、乱学"导致学了很多但做不了项目，系统路线解决这个问题。

**详细解释**：
| 阶段 | 内容 | 对应薪资 |
|------|------|---------|
| 第一阶段 | Python + RAG + LangChain + API | 初级岗位 |
| 第二阶段 | 项目实战 + MCP服务部署 | ~20K |
| 第三阶段 | 大模型原理 + 微调 + 评测 | ~30K |
| 第四阶段 | Agent集群 + 路由 + 可观测性 | 高级管理者 |

**实际应用场景**：适合零基础或有编程基础的人系统规划转行路径。

### 2.5 延伸思考

1. 视频强调"业务匹配度+技术匹配度"来选择项目，那面试前如何调研目标公司的技术栈，让自己的项目经验更有针对性？

2. 第四阶段的"Agent集群"（路由、跨Agent记忆、可观测性）在实际生产环境中面临哪些工程挑战，和单Agent相比复杂度提升在哪里？

3. 视频提到评测集是保障大模型输出符合用户需求的关键，但如何设计一套有效的评测集？评测指标应该怎么选？
