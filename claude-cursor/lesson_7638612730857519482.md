## 一、视频原文

Skill 装多了，真的非常容易乱。我自己呢，现在常用的 Skill 是 11 个，按场景分成了三类。

我先说第一类，它是属于效率工具。

首先是 Agent 的位置。以前我们搜资料需要一个平台一个平台去切换，现在你只需要一句话说清楚你要什么，这个 Skill 它能帮你跨平台去找——YouTube、小红书、推特、公众号都能支持，还能直接提取字幕，不花 API 的费用。

第二个呢，是 NotebookLM Skill。这个不是 NotebookLM 本身，而是在终端里面操控它的 Skill，把采集进来的资料批量导入，让它去做归纳分析，你不用打开软件，在另一个窗口里面直接搞定。

第三个是 Browser Skill，这个是让 AI 直接接管你的 Claude，打开网页、点按钮、填表单、截图、爬数据，跟你自己坐在电脑前面操作没有任何区别。每天你要重复的流程，你丢给它就可以。

接下来呢，是 Skill Play Store——Claude 的官方出品，写 Skill 的 Skill。把你每天重复的工作流程用一句话描述出来，它能帮你封装成可以反复调用的 Skill，让 Claude 按照你的方式工作。我觉得这个 Skill 长期的价值会非常非常高。

接下来呢，是 Find Skill，相当于 Skill 的一个搜索引擎。你不知道该用哪个 Skill 的时候，把你的需求告诉它，它根据上下文帮你找到最合适的，不用自己一个一个去翻。

好，接下来讲第二类，是产品开发。

JustStack，这个是硅谷创业圈出来的开源技能库，里面的 Offers 是我用得最多的。说任何事之前，先被 AI 追问 6 个问题，把目标和边界条件逼出来。跳过这一步，做出来的东西返工率高很多。

Discipline Powers，这个是给 Claude Code 装的一套工程师的工作习惯——拆任务、写测试、代码审查这些流程，它默认就会走，不用你每次都提醒。本质上就是把 AI 编程从"能跑就行"推到"可以交付"。

然后 Frontend Zen 和 EYE-EYE Pro Max，这两个是做前端设计的 Skill。你不懂设计没关系，排版配色、江湖细节它自己去判断。一个负责从零去生成页面，一个负责在现有界面上做优化。

然后是 Playwright MCP，自动化测试产品。做完以后，让它模拟用户的操作跑一遍，哪里点不动、哪里报错了，全部报出来，你不用自己手动一个一个点。

OK，那再讲最后一类，就是自媒体，只有一个，但我几乎每天都在用——Humanizer。这个是 Guy 出品的，AI 写的东西通顺归通顺，就是读起来不像人话，这个 Skill 专门去处理这个问题，过一遍之后自然度明显不一样，做内容的基本上都用得上。

所以你看，三条线，11 个 Skill，基本上覆盖了我 90% 的日常场景——提效、做产品、做内容。

### 2.3 核心概念精讲

#### 概念一：Skill（Claude 技能插件）

**是什么**：安装在 Claude 上的功能扩展模块，类似手机 App，每个 Skill 赋予 Claude 某种特定能力或工作习惯。

**为什么重要**：Claude 本身是通用模型，Skill 让它在特定场景下变得更专业、更自动化，减少每次手动提示的重复成本。

**详细解释**：
- Skill 可以是官方出品（如 Skill Play Store），也可以是社区开源（如 JustStack）
- Skill 装多了容易混乱，博主建议聚焦在真正高频的场景，按需精选
- 有专门的 Find Skill 帮你在大量 Skill 中做匹配，解决"选择困难"问题

**实际应用场景**：日常工作中需要重复执行的任务（搜资料、写代码、做测试、发内容），都可以找对应 Skill 来接管。

---

#### 概念二：工作流自动化（Browser Skill + Skill Play Store）

**是什么**：让 AI 代替人完成浏览器操作或把重复流程封装成可复用模块的两类 Skill。

**为什么重要**：重复性操作是效率最大的杀手，这两个 Skill 的组合可以让 Claude 真正"干活"而不只是"回答问题"。

**详细解释**：
- **Browser Skill**：AI 直接控制浏览器——打开网页、点按钮、填表单、截图、爬数据，和人工操作无差异
- **Skill Play Store**：用一句话描述你的工作流程，它帮你封装成可反复调用的 Skill，让 Claude 按你的习惯工作

**实际应用场景**：每天重复的数据采集、表单填写、内容发布流程，描述一次，之后全部交给 AI 执行。

---

#### 概念三：AI 编程质量提升（JustStack + Discipline Powers）

**是什么**：两个专门针对 Claude Code 使用场景的 Skill，分别解决"需求不清"和"代码不规范"两个核心问题。

**为什么重要**：AI 写代码容易"能跑但不能交付"，这两个 Skill 把工程师的工作方法论植入进来，让输出质量上一个台阶。

**详细解释**：

| Skill | 解决的问题 | 具体做法 |
|---|---|---|
| JustStack | 需求模糊导致返工率高 | 开始前强制追问 6 个问题，逼清目标和边界条件 |
| Discipline Powers | 代码只能跑不能交付 | 默认走拆任务→写测试→代码审查的工程师流程 |

**实际应用场景**：用 Claude Code 开发产品功能时，先用 JustStack 对齐需求，再用 Discipline Powers 保证代码质量。

---

#### 概念四：Humanizer（AI 内容去机器味）

**是什么**：专门处理 AI 生成文字"通顺但不像人话"问题的 Skill，让内容自然度明显提升。

**为什么重要**：AI 写作的最大痛点不是语法错误，而是读起来有明显的"AI 腔"，Humanizer 针对性解决这个问题。

**详细解释**：
- AI 生成的内容通顺归通顺，但缺乏人的语感和表达习惯
- 过一遍 Humanizer 之后，自然度明显不同
- 博主几乎每天都在用，是自媒体场景下唯一入选的 Skill

**实际应用场景**：所有用 AI 辅助生成的文章、脚本、社媒文案，发布前过一遍 Humanizer。

### 2.5 延伸思考

1. **Skill Play Store 的长期价值在哪里？** 博主特别强调这个 Skill"长期价值非常非常高"——如果每个人都能把自己的工作流封装成 Skill，是否意味着"个人工作方法论"本身会成为一种可复用、可分享甚至可交易的资产？

2. **"先被追问 6 个问题"这个设计说明了什么？** JustStack 在开发开始前强制逼清需求，本质上是在纠正一个习惯——人们倾向于把模糊的想法直接扔给 AI。这个设计对你自己平时使用 AI 的方式有什么启发？

3. **11 个 Skill 覆盖了 90% 的场景，那剩下 10% 是什么？** 博主装过上百个 Skill，最终只留下 11 个高频使用——那些被淘汰的 Skill 失败在哪里？是功能重叠、使用门槛高，还是场景太窄？这对你选择和管理自己的 AI 工具箱有什么参考意义？
