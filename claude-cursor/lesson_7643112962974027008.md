## 一、视频原文

这段时间一直在用AI做工具，今天就分享几个比较逆天的Skill。如果你有想用AI来帮你做视频，包括找热点、找素材、然后优化剪辑，那么这七个Skill一定要去用起来，效果直接拉满。

第一个：Agent率去，主要用来找对标素材。覆盖小红书、推特、公众号这些平台，直接口头跟他说需求就行了，比如说收YouTube上关于口播技巧的热门视频，或者是让他帮你提取某个视频的字幕，一句话就能搞定，而且所有平台都支持，用的是API，免费，你完全不用担心算力费用。

第二个：OpenCLI。怎么说呢，刚才第一个技能是帮你找内容，而这个是帮你自动操作浏览器的手脚。比如登录某某平台、截图、填表、爬数据，这些都是自动化完成的，不用你手动折腾，能省很多时间。

第三个：Notebook LM Skill。注意不是Notebook LM本体，他能把Agent率去收集的文章、PDF、YouTube视频的素材，一键导入Notebook LM，不用打开软件就能操作，直接帮你分析素材，产出可用的内容。

第四个：Lisson号。这个简直就是视频创作的神器，它是一个由7个Skill打包的创作者工作流。你只要给它一个主题，就能给你一条链接，一次性生成公众号长文、重点摘要，而且在图文方面还有段落插图、想法图文卡片、口播稿，而且口播稿还能一键转播客。联动起来更绝：给一个视频链接，从提取内容到做播客、剪辑视频，一条龙搞定。

第五个：Lisson（视频生成）。这是一个用代码做视频的神器，那些炫酷动效的视频很多都是它做的，主要是入门超级简单，直接说需求，Claude会自动生成代码并渲染视频。不过如果你想产出高质量的成片，那还是建议你去补充点视频创作的基础知识。

第六个：Humanizer。一个去AI味最强的技能。相信你平时用豆包生成的那些文章，那股浓浓的豆包味，别说有多难受，用它一下就能搞定了，谁用谁知道。

第七个：宝玉推特。宝玉老师自己在用的Skill，大概20多个技能，深度文章自动配图，主要的特点就是一键能把内容推到公众号、微博或者是微博，关键走真实浏览器路线，不是调用API，安全性十足。

今天就分享这些吧，评论区里面如果你觉得哪个使用很短，聊聊。

### 2.3 核心概念精讲

#### 概念一：Agent率去（多平台内容采集代理）

**是什么**：一个覆盖小红书、推特、公众号、YouTube等平台的AI内容采集Skill，支持口语化指令直接操作。

**为什么重要**：自媒体创作最耗时的环节之一是找对标内容和提取素材，Agent率去将这个过程从手动浏览变成一句话指令。

**详细解释**：
- 支持平台：小红书、推特、公众号、YouTube等
- 典型用法：搜索热门视频、提取视频字幕
- 费用：基于API，免费使用，无算力费用压力
- 操作方式：直接口头描述需求，无需复杂配置

**实际应用场景**：创作者想研究某类内容的爆款规律，直接让Agent率去抓取该话题下热门视频的字幕，快速分析选题结构。

---

#### 概念二：Lisson号创作者工作流

**是什么**：由7个Skill打包组合的内容创作工作流，输入一个主题或视频链接，自动输出多种内容形态。

**为什么重要**：内容创作者需要将同一个核心内容适配多个平台和格式，Lisson号将这个"一稿多发"的过程自动化。

**详细解释**：
| 输入 | 自动输出 |
|------|----------|
| 一个主题 | 公众号长文 + 重点摘要 + 段落插图 + 图文卡片 + 口播稿 |
| 口播稿 | 一键转播客 |
| 一个视频链接 | 内容提取 + 播客制作 + 剪辑视频（一条龙） |

**实际应用场景**：博主录制完一期视频后，把视频链接丢给Lisson号，自动生成公众号推文、播客音频和剪辑版视频，一套内容多平台分发。

---

#### 概念三：Humanizer（去AI味技能）

**是什么**：专门用于消除AI生成文本中"AI腔"的Skill，让输出内容更接近人类写作风格。

**为什么重要**：AI生成的内容往往有明显的模式化表达（如"豆包味"），读者感知明显，影响内容质量和信任度；Humanizer解决这个痛点。

**详细解释**：
- 针对问题：AI生成文章语气千篇一律，缺乏个人风格
- 典型症状：过于正式、结构过于整齐、用词重复、缺乏情感波动
- 解决方式：通过Humanizer技能对AI输出进行风格化改写

**实际应用场景**：用豆包或其他AI生成初稿后，经Humanizer处理，使文章语气更自然，适合直接发布到公众号或社交媒体。

---

#### 概念四：宝玉推特（真实浏览器路线发布）

**是什么**：宝玉老师开发的内容发布Skill，约20个子技能，支持深度文章自动配图，通过真实浏览器操作（非API调用）完成多平台发布。

**为什么重要**：API调用发布内容存在账号安全风险，真实浏览器路线模拟人工操作，安全性更高，适合长期稳定使用。

**详细解释**：
| 特性 | 说明 |
|------|------|
| 技能数量 | 约20个子技能 |
| 核心功能 | 深度文章自动配图、一键多平台发布 |
| 支持平台 | 公众号、微博等 |
| 技术路线 | 真实浏览器操作（非API），安全性高 |

**实际应用场景**：创作者写完深度文章后，让宝玉推特自动配图并一键发布到公众号和微博，节省排版和手动发布时间。

### 2.5 延伸思考

1. 视频中介绍的7个Skill覆盖了内容创作的不同环节，但它们之间如何串联成一个完整的工作流？一个创作者应该按什么顺序组合使用这些工具？

2. Humanizer的"去AI味"本质上是在修改AI的输出风格，这是否意味着AI生成内容的"标准化问题"是当前工具链的一个系统性缺陷？未来的AI写作工具应该如何在生成阶段就解决这个问题？

3. 宝玉推特选择"真实浏览器路线"而非API调用来保证安全性，这种设计取舍背后反映了哪些平台的反自动化策略？这对AI工具的长期可用性有什么影响？
