### 2.3 核心概念精讲

#### 概念一：Claude Skill

**是什么**：一种可以封装特定工作流程的 AI 能力模块，在 Claude Code 中按需加载和触发，相当于给 AI 装上一个"专属工种"。

**为什么重要**：
普通的对话式 AI 每次都要重新交代背景和要求，而 Skill 把 SOP、风格偏好、文件管理规则一次性写入，之后只需自然语言触发，AI 就能自动执行完整流程，且按需加载节省 token。

**详细解释**：

| 组成文件 | 作用 |
|---|---|
| `skill.md`（必须） | 定义 Skill 的 Name 和 Description，AI 依此判断何时自动触发 |
| Reference | 参考文档，如写作规范、风格指南 |
| Script | 运行脚本，定义执行逻辑 |
| Template | 输出模板，规范成稿格式 |

**实际应用场景**：
输入"帮我写一篇关于 XX 产品的测评"，Skill 自动识别意图，依次完成信息搜集、选题讨论、风格匹配、出稿、审校，全程无需人工干预每个步骤。

---

#### 概念二：写作 SOP 的 7 步流程

**是什么**：将专业创作者的完整写作流程标准化，分解为 7 个可被 AI 执行的明确步骤。

**为什么重要**：
AI 写作质量差的根本原因是"流程缺失"——直接让 AI 写文章，它没有上下文、没有风格参照、没有校对机制。7 步 SOP 把人类创作者的隐性经验显性化，让 AI 有章可循。

**详细解释**：

| 步骤 | 动作 | 关键输出 |
|---|---|---|
| Step 1 理解需求 | 解析用户输入 | 自动保存 Brief 到对应文件夹 |
| Step 2 信息搜集 | 多渠道检索 | 含时间戳和来源的资料文档 |
| Step 3 选题讨论 | 与用户协商，不直接动笔 | 3-4 个选题方向 + 大纲 + 工作量评估 |
| Step 4 任务清单 | 识别测试/配图需求 | 协作文件夹中的任务清单文档 |
| Step 5 学习风格 | 读取历史爆文 | 提取开头方式、结构、语言、行文特征 |
| Step 6 出稿创作 | 结合风格+时效案例 | 框架完整的初稿（不求完美） |
| Step 7 三重审校 | 模拟主编审稿机制 | 数据校对 → 去 AI 味 → 细节打磨 |

**实际应用场景**：
Step 3"先讨论选题，不直接写"是关键设计——避免 AI 自作主张跑偏方向，把人的判断留在最关键的决策节点上。

---

#### 概念三：知识库与风格记忆

**是什么**：在 Skill 的目录结构中，持续沉淀个人创作资产（爆文、技巧、规范、人设），让 Agent 随时间越用越懂你。

**为什么重要**：
这是让 AI 写出"你的味道"而非"通用 AI 腔调"的核心机制。风格参考、用户人设等文件夹让模型在每次运行时都能调取创作者的个人特征。

**详细解释**：

| 文件夹 | 存放内容 | 作用 |
|---|---|---|
| 风格参考 | 历史数据好的爆文 | 让 AI 学习你的写作风格 |
| 写作技巧 | 个人整理的 Markdown 笔记 | 提供写作方法论参考 |
| 规范指南 | 防流水账指南、排版规则等 | 约束输出质量底线 |
| 用户人设 | 个人信息、写作需求与偏好 | 让 AI 理解"我是谁，我想写给谁看" |

**实际应用场景**：
资料越积越多，Agent 越来越专业——这是一个正向飞轮：每篇好文章沉淀进知识库，下一篇的起点就更高。

### 2.4 速查表

| 对比维度 | 传统写作流程 | Claude Skill 自动化写作 |
|---|---|---|
| 耗时 | 2 小时起步 | 约 20 分钟 |
| 选题方式 | 人工头脑风暴 | AI 提供 3-4 个方向供选择 |
| 风格一致性 | 依赖个人状态 | 读取历史爆文，稳定输出 |
| 审校机制 | 人工逐一检查 | 三重自动审校（数据/去AI味/细节） |
| 文件管理 | 手动整理 | 按用途自动分类写入对应文件夹 |
| token 消耗 | 不适用 | 按需加载，节省 token |
| 可复用性 | 低（每次重新配置） | 高（Skill 一次配置，持续调用） |
| 知识积累 | 存在个人脑中 | 沉淀进知识库，Agent 越用越强 |

### 2.5 延伸思考

1. **SOP 的质量决定 Skill 的上限**：视频中强调要先把工作流程"拆解清楚"再让 Claude 生成 Skill。那么，如果你自己的写作流程本身就不清晰，AI 能帮你梳理出一套好的 SOP 吗？还是说创作者需要先有足够的自我认知？

2. **"先讨论选题，不直接写"这个设计背后的逻辑**：视频特别强调 Step 3 不能让 AI 直接动笔，而是先和人讨论。这个"人在回路"的节点设计，哪些环节适合保留人的判断，哪些环节可以完全交给 AI？

3. **知识库的边界在哪里**：风格参考放的是"数据好的爆文"，但爆文的风格不一定适合所有选题。当知识库积累的历史风格与新选题的调性冲突时，Agent 应该如何取舍？你会怎么在 Skill 中设计这个规则？
