### 2.3 核心概念精讲

#### 概念一：Anthr Gravity —— 免费接入 Claude 和 Gemini 的桥梁

**是什么**：Google 推出的一款 AI 编程工具，内置了 Gemini 和 Claude 两个顶级模型，可作为 OpenCode 的模型提供方插件使用。

**为什么重要**：Claude 和 Gemini 的官方 API 均需付费，通过 Anthr Gravity 可以用 Google 账号免费登录，实现零成本接入两大顶级编程模型。

**详细解释**：

| 步骤 | 操作 |
|------|------|
| 1 | 在 GitHub 找到 Anthr Gravity 插件，复制安装指令 |
| 2 | 粘贴到 OpenCode，自动下载安装 |
| 3 | 重开终端，运行配置指令，选择 Google 登录方式 |
| 4 | 浏览器弹出 Google 登录页，登录后复制授权码粘贴回终端 |
| 5 | 重启 OpenCode，输入 `/model` 验证是否出现 Gemini 2.5 Pro 和 Claude 4.5 Opus |

**实际应用场景**：不想花钱买 API Key，但希望在 OpenCode 中使用 Claude 4.5 Opus 或 Gemini 2.5 Pro 进行日常编程辅助。

---

#### 概念二：Omai OpenCode —— 多智能体编程插件包

**是什么**：OpenCode 最热门的第三方插件包，内置7个专职编程 Agent，通过多智能体分工协作自动完成复杂开发任务。

**为什么重要**：单个 AI 模型处理复杂项目时容易遗漏细节，多 Agent 协作可以把写代码、查 Bug、梳理结构等任务分配给最合适的智能体，据称作者投入了2.4万美金开发。

**详细解释**：

| 使用方式 | 说明 |
|----------|------|
| `/item` 命令 | 手动从7个 Agent 中选一个执行特定任务 |
| 提示词输入 `ultrawork` | 自动调用 Omai OpenCode 的完整多 Agent 工作流 |

**实际应用场景**：面对一个需要同时写代码、调试、整理目录结构的中大型项目时，用 ultrawork 让多个 Agent 并行处理，节省大量手动操作时间。

---

#### 概念三：Skills —— 可复用的自定义指令模块

**是什么**：预先定义好的功能模块（类似宏或模板），可以让 OpenCode 一键执行特定任务，比如"翻译"、"代码审查"等。

**为什么重要**：把常用操作固化成 Skills，避免每次重复输入长提示词，同时 OpenCode 完全兼容 Claude Code 已有的 Skills，迁移成本为零。

**详细解释**：

| 创建方式 | 操作 |
|----------|------|
| 迁移已有 Skills | 在项目根目录新建 `.claude/skills/` 文件夹，把 Claude Code 的 Skills 文件复制进去 |
| 自然语言创建新 Skills | 安装 Skills Creator 插件后，直接告诉 OpenCode 你要什么功能（如"创建一个翻译 Skills"），它自动生成 |

**实际应用场景**：团队有一套已在 Claude Code 中调试好的代码审查 Skills，可以直接复制到 OpenCode 继续使用，无需重新编写。

---

#### 概念四：MCP —— 让 OpenCode 连接外部工具的协议

**是什么**：Model Context Protocol，一种让 AI 工具连接外部服务（如浏览器、数据库、API）的标准协议，安装后 OpenCode 可以直接操控这些外部工具。

**为什么重要**：有了 MCP，OpenCode 不再局限于纯文本对话，可以真正操作浏览器、读取文件、调用外部 API，变成能干实事的自动化助手。

**详细解释**：

| 步骤 | 操作 |
|------|------|
| 安装 MCP | 直接在 OpenCode 输入自然语言，如"安装 Chrome MCP Server" |
| 配置 Key | 把 API Key 告诉 OpenCode，让它自动写入配置文件 |
| 查看已装 MCP | 输入 `/mcps` 查看所有已配置的 MCP 列表 |

**实际应用场景**：安装浏览器 MCP 后，可以让 OpenCode 自动打开网页、抓取内容、填写表单，实现真正的浏览器自动化。

### 2.5 延伸思考

1. **Anthr Gravity 的免费额度是否有上限？** Google 提供的免费接入通常有速率限制或每日配额，在实际高强度编程场景中，免费额度是否够用？超出后有哪些替代方案？

2. **Omai OpenCode 的7个 Agent 是如何分工的？** 视频只提到了多智能体协作的结果，但没有展示各 Agent 的具体职责划分。不同类型的任务（前端/后端/数据处理）应该如何选择对应的 Agent 才能发挥最大效果？

3. **Skills 和 MCP 的边界在哪里？** Skills 是固化提示词逻辑，MCP 是连接外部工具——当一个任务既需要自定义提示逻辑又需要调用外部服务时，两者应该如何配合设计，才能避免重复或冲突？
