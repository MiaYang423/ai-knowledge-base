## 一、视频原文

我最近高强度用了两周 Claude Code，两个 Max 账号都不够用的那种。我现在最大的感受就是它真的特别强，但你没用好就随时会功亏一篑。

我今天不会去讲教程，我想讲三个我现在最真实的感受吧。

**第一个感受就是一定要保存上下文。** 我最崩溃的一次是跟 Claude Code 聊了差不多一两个小时，我跟它聊清楚了任何我关于这个项目的一些想法，我把项目目标、背景资料、好的标准、想要做成什么样全跟它聊清楚了。而且我当时是用语音调的，越调越进入状态，最后它真的帮我做出来了一个还不错的一个原版 scale。结果第二天我准备继续调的时候，你猜怎么着，账号被封了。那一刻我真的是太崩溃了，我是又崩溃又无力又无语。我崩溃不是因为没了一个账号，而是前面所有的上下文、所有的共识、所有已经铺垫好的状态都没了。

所以如果你现在正在用 Claude Code，像我这样今天没有把东西做出来，一定要去不断地保存你们聊到哪了、当前的方案是什么、下一步要怎么做、判断的标准是什么。我希望大家都可以保存下来。AI 真的很强，但是它一旦失去上下文，我觉得就像一个突然失忆的队友。

**第二个感受就是不要什么都自己想，一定要让 AI 来主导。** 我以前用 AI 也会把步骤拆好，第一步该怎么着，第二步该怎么着，第三步怎么着。后来我发现，当 Claude Code、当 AI 足够聪明的时候，你其实不需要把它怎么做都想好。人更重要的是想清楚：我要做什么、为什么做、做到什么程度算好。

你就可以跟它说这句话，这句话太重要了，我也经常跟我朋友分享：**"你可以问我任何问题，直到你觉得你可以把这件事情做好为止。"** 这句话真的特别好用，所有用过的朋友都给我非常不错的反馈。你让它反过来问你各种问题，它做好这件事需要哪些信息，直到它能做好这件事。这时候你跟 AI 的关系就变了，不是你一句一句地指挥，而是你负责方向，它负责推进。想清楚做什么是你的事，怎么把这个事做出来是 Claude 的事。

**所以第三个感受就来了，Claude Code 真正强的地方不是单点能力，而是组合能力。** 当它只是一个聊天工具的时候，它确实已经很聪明了。但当你把各种 skill、MCP 工具链串起来之后，你的感觉绝对会完全不一样。它可以帮你写文案，可以浏览网页帮你查资料，可以读公众号抓信息然后整理很多信息结构，它可以围绕一个目标持续地帮你推进。这个感觉你真的会超级爽。

这个感觉就有点像英雄联盟，我们都会给英雄去出装备。这个装备不一定有就好，或者越多越好，而是怎么搭配这个事更重要，怎么搭配让它进入到更强的一个状态。我觉得工具也是一样，不是装一堆 skill 就厉害，真正厉害的是你怎么组合它们，让它们进入到最适合当前任务的状态。当这套装备跑通的时候，你会明显地感觉到，它不再只是一个工具，更像是一个能跟你一起干活的队友。

所以这两周我用下来最大的一个判断就是：**Claude Code 不单是一个代码工具，它更像是一个依赖上下文、任务定义和工具组合的执行系统。** 现在来说，其实你能想到的事它应该都可以尝试去做。如果你还只是问一句答一句，那你可能真的只用了它很小的部分能力。但如果你能把目标讲清楚，把上下文保存好，再把工具链搭起来，它真的能帮你把很多事情往前推。

这就是我这两周最强烈的一个感受。后面我也会继续分享，关于 skill 和 MCP 这套装备该怎么组合才能发挥到最大的效果。OK，先这样。

### 2.3 核心概念精讲

#### 概念一：上下文即资产

**是什么**：在与 Claude Code 协作过程中，双方共同建立的项目背景、目标共识、判断标准和推进状态，统称为"上下文"。

**为什么重要**：
作者亲历账号被封，一两小时积累的所有共识瞬间清零。他的崩溃不是因为丢了账号，而是因为"所有铺垫好的状态都没了"——这说明上下文本身就是协作的核心价值所在。

**详细解释**：

| 上下文包含的内容 | 具体示例 |
|---|---|
| 项目目标 | 这个功能要解决什么问题 |
| 背景资料 | 已有代码结构、产品逻辑 |
| 判断标准 | 什么叫"做好了" |
| 当前方案 | 现在走的是哪条路 |
| 下一步计划 | 接下来要做什么 |

**实际应用场景**：
每次会话结束前，让 Claude 输出一份"当前状态摘要"，包含以上五项内容，保存到本地文档，下次开新会话时直接粘贴导入。

---

#### 概念二：反向提问法

**是什么**：不是由人来拆解步骤指挥 AI，而是把目标交给 AI，让 AI 主动向人提问，直到它掌握足够信息能独立完成任务。

**为什么重要**：
当 AI 足够聪明时，人逐条拆步骤反而是在用人脑做 AI 擅长的事。真正稀缺的是人对"做什么、为什么做、做到什么程度算好"的判断，而不是执行路径的设计。

**详细解释**：

核心咒语（可直接复制使用）：
> **"你可以问我任何问题，直到你觉得你可以把这件事情做好为止。"**

| 旧模式 | 新模式 |
|---|---|
| 人拆步骤 → 逐条给 AI | 人说目标 → AI 反向提问 |
| AI 是执行工具 | AI 是协作推进者 |
| 人需要懂"怎么做" | 人只需要想清楚"做什么" |
| 容易遗漏关键信息 | AI 主动补全所需信息 |

**实际应用场景**：
启动一个新任务时，先用一段话描述目标和背景，然后说出这句咒语，让 Claude 开始提问，你只需要回答它的问题，直到它说"我已经可以开始了"。

---

#### 概念三：工具组合 = 装备搭配

**是什么**：将 Claude Code 与外部工具（skill、MCP 等）串联，形成能围绕特定目标持续执行的工具链，而不是把 Claude 当作单一聊天窗口使用。

**为什么重要**：
单独使用时，Claude 是"很聪明的聊天工具"；串联工具链之后，它变成"依赖上下文、任务定义和工具组合的执行系统"——两者的能力边界完全不同。

**详细解释**：

作者用英雄联盟出装来类比：

| 游戏类比 | 工具使用类比 |
|---|---|
| 英雄本体 | Claude Code 基础能力 |
| 装备 | skill / MCP 工具 |
| 装备搭配 | 工具链的组合方式 |
| 不是越多越好 | 不是装越多 skill 越厉害 |
| 搭配适合当前英雄 | 组合适合当前任务 |

视频提到的工具能力举例：
- 浏览网页 → 查资料
- 读公众号 → 抓取并整理信息结构
- 写文案
- 围绕目标持续推进多步骤任务

**实际应用场景**：
针对一个具体任务（如"整理竞品信息报告"），思考它需要哪些子能力，再选择对应工具串联，而不是打开 Claude 直接开始问。

### 2.4 速查表

| 维度 | 错误用法 | 正确用法 |
|---|---|---|
| **上下文管理** | 会话结束就关掉，下次重新开始 | 每次保存进展、方案、标准、下一步 |
| **任务分工** | 人拆步骤，逐条指挥 AI | 人定方向，AI 反向提问后自主推进 |
| **工具使用** | 只用聊天窗口问答 | 串联 skill / MCP 构建工具链 |
| **能力认知** | Claude = 聪明的问答工具 | Claude = 上下文 + 任务定义 + 工具组合的执行系统 |
| **核心咒语** | "第一步做 X，第二步做 Y……" | "你可以问我任何问题，直到你能把这件事做好为止" |

### 2.5 延伸思考

1. **上下文的边界在哪里？** 保存上下文固然重要，但什么信息值得保存、什么信息是噪音？如何设计一套轻量的上下文管理模板，让每次会话的"状态交接"成本最低？

2. **反向提问法的适用边界是什么？** 这个方法在目标模糊时非常有效，但如果连"做什么"本身都不清晰，让 AI 提问是否会陷入"盲人摸象"？人在什么情况下还是需要先自己想清楚再交给 AI？

3. **工具组合的"最优解"是任务相关的还是通用的？** 作者说搭配比数量更重要，那么对于不同类型的任务（写作、调研、编程、产品设计），各自的最优工具组合是否存在规律？还是每次都需要重新设计？

---

### 2.6 一句话记住它

> 用 Claude Code 的上限，不取决于它有多聪明，而取决于你的上下文保存得多完整、目标定义得多清晰、工具链搭配得多合理。
