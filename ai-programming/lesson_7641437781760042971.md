那么过往在我们讲 Context Window 的时候，我提到过一个人的 Claude Code 里面可能有很多 Skill，但是它不会在每次打开新的 Session 的时候全部被 Load 进来，因为这样的话 Context Window 马上就会满掉。所以 Claude Code 基本上只是会先去记一下每个 Skill 的名称和它的简介，然后需要的时候才会去读它的具体内容和调用它的具体操作。

那这种情况下，我会花个两三分钟来了解一下它是什么东西、能做什么，就好像 Claude Code 去读它的标题和简介一样，然后我会把它存在一个地方。这样哪一天我如果突然需要一个类似这样的工具或者产品的时候，我才会想起来我上周或者上个月读到了一个很适合的东西，我现在去我的文件夹里打开它，然后我才去学习它的具体内容。

那么对于 Claude Code 来说，这样做的好处就是让这个 Context Window 不会一下子被占得特别满。那么对于我来说，它的好处就是让我的时间不会被一些我大概率用不到的东西马上占满，同时可以节省一些我的脑容量，最后也是让我有更少的 AI 焦虑。

有很多 AI 工具虽然风很大、很红、很多人都在讨论，但是如果我真的觉得我现在不需要或者用不到的话，也不会立刻花这个时间去研究它。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**借鉴 Claude Code 的"渐进式披露"机制，用"按需学习"代替"全量焦虑式跟进"，让有限的时间和脑容量只花在真正用得到的 AI 工具上。**

### 2.3 核心概念精讲

#### 概念一：Progressive Disclosure（渐进式披露）

**是什么**：Claude Code 的一种上下文管理机制——不在 Session 开始时加载所有 Skill 的完整内容，而是先只加载名称和简介，需要时再调取详情。

**为什么重要**：Context Window 的空间是有限的，如果一次性塞满，就没有空间处理真正当下需要的任务。

**详细解释**：

| 阶段 | Claude Code 的做法 | 对应的人类学习行为 |
|------|-------------------|-------------------|
| 初次接触 | 读取 Skill 名称 + 简介 | 花 2~3 分钟了解工具是什么、能做什么 |
| 存储方式 | 保留索引，不展开全文 | 存进文件夹/笔记，不深入研究 |
| 触发深入 | 任务实际需要该 Skill 时 | 自己的项目真正用到时再去学 |

**实际应用场景**：在 Claude Code 中管理大量自定义 Skill 时，避免 Context Window 被无关内容撑满，保持对话效率。

---

#### 概念二：按需学习（Need-based Learning）

**是什么**：不追求对所有 AI 新工具的即时全面了解，而是根据"当下是否用得到"来决定学习深度的个人学习策略。

**为什么重要**：AI 工具的迭代速度远超个人学习速度，全量跟进既不可能实现，也会持续消耗时间和注意力资源。

**详细解释**：

遇到新工具/产品时，做两个判断：

- **判断 1**：我现在手上的项目能用到它吗？
- **判断 2**：我现在就迫切想尝试它吗？

| 判断结果 | 对应行动 |
|---------|---------|
| 两个都是"否"（占 80~90%） | 花 2~3 分钟了解概要，存档，不深入 |
| 任意一个是"是" | 立即深入学习具体内容和操作 |

**实际应用场景**：刷 X、小红书、科技新闻时，看到新 AI 工具的推送，用这套判断框架快速决策，而不是每次都陷入"要不要现在学"的纠结。

---

#### 概念三：信息存档系统（Personal Knowledge Index）

**是什么**：将"了解过但暂时不深入学习"的工具或内容，以简要形式存入一个固定的地方，方便日后按需检索。

**为什么重要**：渐进式披露策略能生效的前提，是"索引"真实存在且可被找回——否则当初的浏览就彻底浪费了。

**详细解释**：类比 Claude Code 保留 Skill 的名称和简介作为索引，个人也需要一个实际的存储位置（文件夹、笔记应用等），记录：
- 工具/产品名称
- 它能做什么（一两句话）
- 来源链接

当未来某个项目产生真实需求时，才回到这个索引里找，再展开深度学习。

**实际应用场景**：建立一个专门的"AI 工具待学清单"文件夹或笔记页，定期在有真实需求时回顾，而不是让它变成一个永远不看的收藏夹。

### 2.4 速查表

| 对比维度 | 焦虑式全量跟进 | 渐进式按需学习 |
|---------|-------------|-------------|
| 触发条件 | 看到新工具就深入研究 | 判断当下是否真的用得到 |
| 时间投入 | 每次都花大量时间 | 暂时不需要时只花 2~3 分钟 |
| 脑容量消耗 | 高，记忆大量暂时用不到的内容 | 低，只存索引 |
| AI 焦虑程度 | 高，永远追不上 | 低，主动选择学什么 |
| 类比来源 | — | Claude Code 的 Progressive Disclosure |
| 核心工具 | — | 两个判断问题 + 个人存档系统 |

### 2.5 延伸思考

1. **Claude Code 的渐进式披露是系统自动决策的，而人的"按需学习"需要主动判断**——你有没有可能在实际操作中，因为好奇心或 FOMO（错失恐惧）而绕过这两个判断，还是回到全量跟进？如何给自己设计一个更强的执行机制？

2. **存档系统本身也有"容量"问题**——如果待学清单越积越长，永远不被打开，它和"没存"有什么本质区别？怎样设计一个真正会被用到的索引，而不是另一个让人焦虑的待办清单？

3. **视频提到 80~90% 的新工具"当下用不到"**——这个比例是否因人而异？对于一个正在密集做 AI 项目的开发者，和一个普通用户，这个阈值会不会很不一样？"按需"的"需"应该怎么更精确地定义？

---

### 2.6 一句话记住它

> 你的大脑和 Claude Code 的 Context Window 一样有限——学 Claude Code 的做法，先存索引，真正用到时再展开，焦虑自然就少了。
