# 教案：Claude 从入门到自动化的正确路径

> 适合人群：想系统学习 Claude 但不知道从哪里开始、或者用了感觉"没什么用"的 AI 初学者

---

## 一、视频原文

我觉得很多人学 Claude 走了弯路，不是因为贪难，而是顺序搞反了。正确的顺序是：先把它用起来，再把它配好，最后让它自动跑。

所以第一阶段是去认识它。你不要一上来就去研究什么高阶玩法，你先去完成最简单的三件事情：账号要注册好，丢几个真实任务体验一下，它和搜索引擎到底有什么本质的区别。Claude 不是去回答问题的工具，它是在交付结果。这个认知转变越早建立越好。模型选择呢也可以在这个阶段去确立，你看，小任务可以用 Haiku 这个模型，日常用 Sonnet，难题用 Opus，别一把去说哈用最贵的。

第二阶段就是你要去学会提需求了。这个也是最容易被跳过的，也最影响体验的一步。Claude 感觉没用，90% 的原因还是在你的提示词，不是在模型本身。那核心方法是三段式提示词：背景，就是你是谁、你在做什么、目标是什么；任务，就是你要它做什么，越具体越好；规则，就是格式、语气、字数、结构，告诉它怎么去交付。掌握这个以后，输出的质量会立刻从"能看"变成"能用"。你这一步做扎实，后面所有的阶段都会更顺利。

第三阶段就是配置你的工作环境。这阶段的目标就是把 Claude 从孤立的对话框变成一个生产力系统的一部分。核心有三个功能：第一个是 Projects，这是体验质变的一个关键，把背景资料和规则一次性放进去，后续在同一个项目里面去开对话，它就自动读取你的上下文，不需要每次重新解释，也要让输出风格统一，不用每次在提示词里去重复格式要求。Connected，Claude 接入了 Google Drive、日历等工具，它才能真正参与到你的工作流，不是一个独立存在。另外要养成一个习惯，对话变长就压缩上下文，并且开新的对话去继续。越聊越乱不是 Claude 的问题，是信息噪音把它给淹没了。

第四阶段就是进阶自动化。当你发现自己在重复地一直在做同一样事情的时候，就是进入这一层的时机。Skills，把高频的提示词做成一键触发的工作流；Claude 把整个重复的流程交给它在后台自动执行；Claude Code 是写代码的人用了就回不去的体验；Research Mode 是需要做深度研究报告时用的。这一层不用急着全部上手，按需以后就可以。

所以 Claude 的正确用法不是去问问题，而是交任务。学这个东西也是一样，先跑起来，然后学会下任务，然后配好环境，然后自动化去重复、去替代你现在的工作。

### 2.3 核心概念精讲

#### 概念一：Claude 的本质定位

**是什么**：Claude 不是搜索引擎式的"问答工具"，而是一个"交付结果"的执行者。

**为什么重要**：
这个认知决定你怎么跟它交互。用"问问题"的心态，你得到的是信息；用"交任务"的心态，你得到的是可以直接用的输出。

**详细解释**：
| 旧认知（搜索引擎思维） | 新认知（任务交付思维） |
|---|---|
| "Claude，什么是季度报告？" | "帮我写一份 Q3 销售季度报告，数据如下……" |
| 期待得到解释和信息 | 期待得到可直接使用的成品 |
| 你来整合信息 | Claude 替你完成整合和输出 |

**实际应用场景**：
把你平时自己动手做的事情（写报告、整理会议纪要、分析数据）直接作为任务丢给它，而不是先问它"怎么做"。

---

#### 概念二：三段式提示词

**是什么**：一种结构化的提示词写法，包含"背景 + 任务 + 规则"三个部分。

**为什么重要**：
视频明确指出，Claude 感觉没用，90% 的原因在提示词。这一步做扎实，后面所有阶段都会更顺。输出质量能从"能看"直接跳到"能用"。

**详细解释**：

| 段落 | 作用 | 示例 |
|---|---|---|
| **背景** | 告诉 Claude 你是谁、在做什么、目标是什么 | "我是一名市场专员，正在准备一场面向 B 端客户的产品发布会" |
| **任务** | 告诉它具体要做什么，越具体越好 | "帮我写一封邀请函，邀请 20 家合作商参加发布会" |
| **规则** | 规定格式、语气、字数、结构等交付要求 | "语气正式但不生硬，300 字以内，包含时间/地点/回复方式" |

**实际应用场景**：
任何你希望 Claude 输出"能直接用"内容的场景，都应该套用这个结构，而不是随手打一句话发过去。

---

#### 概念三：Projects（项目功能）

**是什么**：Claude 的一个功能模块，允许你把背景资料和规则一次性存入，后续在同一项目内开对话时自动读取。

**为什么重要**：
这是从"偶尔用用"到"生产力系统"的关键跨越。不用每次开对话都重新解释背景，也不用每次在提示词里重复格式要求，风格和规则自动统一。

**详细解释**：
- 把你的角色设定、常用规则、项目背景资料放进 Projects
- 之后每次在该项目下开新对话，Claude 自动带入这些上下文
- 配合"对话变长就压缩上下文、开新对话继续"的习惯，避免信息噪音把 Claude 淹没

**实际应用场景**：
长期进行的工作项目（如持续运营的公众号、固定格式的周报、某个产品的文案体系），建一个 Project 后一劳永逸。

---

#### 概念四：自动化阶段的进入时机

**是什么**：当你发现自己在重复做同一件事时，就是进入自动化阶段的信号。

**为什么重要**：
很多人急着一开始就学自动化，反而没用好基础功能。视频给出了清晰的进入条件：先有重复，再谈自动化。

**详细解释**：

| 工具 | 适用场景 |
|---|---|
| **Skills** | 把高频提示词做成一键触发的工作流 |
| **自动化流程** | 把整段重复流程交给 Claude 后台自动执行 |
| **Claude Code** | 写代码的人专用，上手后很难回到手写 |
| **Research Mode** | 需要输出深度研究报告时使用 |

**实际应用场景**：
每周都要做竞品分析？每天都要整理客户反馈？这类固定重复的任务，就是自动化的切入点。

| 阶段 | 目标 | 核心动作 | 关键工具/方法 |
|---|---|---|---|

1. **提示词的"三段式"结构，和你平时给人类同事布置任务的方式有什么相似之处？** 如果把 Claude 当成一个新入职的实习生，你会怎么给它交代工作？
