## 一、视频原文

最近看到一个设计师 Skill 给它，有一种颠覆性，1.7万星。它补的刚好就是很多人做 Vibe Coding 最头疼的一步——页面能做出来，但就是容易丑，看起来很散，而且后面花很多时间反复调。

Google 最近推了一个 Design.md 的思路，本质上就是先写一份设计规则，让 AI 先读，再开始做页面。这个仓库做的事情更直接，它把很多现成的设计风格都整理好了，像 Apple、Notion、Linear、Cloudflare 这些网站的感觉都已经有人拆成了可以直接用的 Design.md。里面写得是比较具体的设计约束：页面整体气质、颜色和字体怎么用、组件偏什么感觉、间距布局层级怎么统一。你直接挑一个接近的风格，把这份 Design.md 放进去，AI 做出来的第一版就会好很多。

我自己实测了一下，最明显的变化就是做出来的页面更有高级感、更有 Taste 了。不过这里也有个小坑：这些仓库网站大多是英文语境做出来的，如果你要做中文页面，字体和留白还是得自己再调一轮。

所以如果你现在也在 Vibe Coding，这个仓库可以去看看。我是 AI 支持派，关注我带你了解更多好用的 AI 工具，我们下期见。

### 2.3 核心概念精讲

#### Design.md 方法论

**是什么**：在让 AI 生成页面之前，先提供一份结构化的设计规则文档（Design.md），约束 AI 的设计决策，从而保证输出质量。

**为什么重要**：AI 在没有设计约束的情况下会做出功能正确但视觉散乱的页面；Design.md 相当于给 AI 一个"设计师的眼光"，从源头提升颜值。

**详细解释**：
| 对比维度 | 没有 Design.md | 有 Design.md |
|---|---|---|
| 第一版页面质量 | 功能可用，视觉散乱 | 功能可用，视觉有高级感 |
| 后续调整成本 | 高（反复调颜色/布局） | 低（设计一致性有保障） |
| AI 设计决策 | 随机，无约束 | 受规则约束，有 Taste |

**实际应用场景**：从 GitHub 仓库选一个接近目标风格的 Design.md，放入 AI 工具的上下文，直接生成有设计感的页面。

---

#### GitHub 现成设计风格库

**是什么**：一个 GitHub 开源仓库（一周获得1.7万星），收录了多种知名网站（Apple、Notion、Linear、Cloudflare 等）的设计风格，并将其拆解为可直接使用的 Design.md 文件。

**为什么重要**：省去了自己总结设计规则的时间，直接复用已被验证的高质量设计语言，降低了 Vibe Coding 的设计门槛。

**详细解释**：每份 Design.md 包含的具体约束：
- **页面整体气质**：定义整体风格感受（简洁/专业/活泼等）
- **颜色和字体**：主色调、辅助色、字体家族和大小规范
- **组件感觉**：按钮、卡片、表单等组件的视觉风格
- **间距布局层级**：页面元素的空间关系和视觉权重

**实际应用场景**：做一个 SaaS 产品页面，选 Linear 风格的 Design.md，AI 生成的页面会自带 Linear 那种简洁专业的感觉。

---

#### 中文页面适配注意事项

**是什么**：由于现有 Design.md 仓库主要基于英文语境设计，中文页面在字体和留白方面需要额外调整。

**为什么重要**：中英文字体的视觉特性差异较大（中文字符更密集），直接套用英文设计规则会导致中文页面字体拥挤、留白不足。

**实际应用场景**：使用 Design.md 生成中文页面后，需要手动调整中文字体选择和行间距/字间距设置。

### 2.5 延伸思考

1. **Design.md 解决了"第一版丑"的问题，但后续迭代时如何保持设计一致性？** 当页面功能不断扩展时，Design.md 需要同步更新吗？AI 能否在多次对话后仍然遵守最初的设计规则？

2. **现有的 Design.md 风格库都是模仿知名产品（Apple、Notion 等），这会带来版权或同质化问题吗？** 如何在借鉴设计语言的同时，保持自己产品的独特性？

3. **视频提到中文页面需要额外调整字体和留白，有没有专门针对中文语境的 Design.md 模板？** 如果没有，如何自己编写一份适合中文产品的 Design.md？
