## 一、视频原文

当你用AI生成男生在拿手机打游戏的时候，偶尔会得到这样的结果——为什么连科幻大片都能生成，AI画好一个打游戏的少年就这么难？如果你也被这个问题困惑了好久，那么恭喜你，这条视频将会彻底解开这一切。

真正的原因其实就藏在提示词里。先看一个简单的例子：如果你对AI说"生成一个人戴着护士帽"，不用想，这个人不仅会戴着护士帽，还一定会穿着护士服，还一定会出现在医院里。如果你说"生成一个人不戴着护士帽"呢？结果一样，AI还是会把他丢进医院。哎，意思来了——"不戴安全帽"，这个人就会出现在工地；"不戴圣诞帽"，这个人就会出现在圣诞节；"不戴厨师帽"，这个人就一定在厨房里做饭。

这种现象就是提示词污染，我更喜欢称它为"关键词的联想陷阱"。AI会因为你提到了某个词，而自动脑补出一整套刻板印象，这些词哪怕被否定掉了，也会强行拉回AI的注意力。

回到男生打游戏的例子——"游戏"这个词就是污染源。AI为了能让你看见"打游戏"这个场景，甚至可以违反客观规律，让男生反向拿着手机，只为了让你能看到可以显示出游戏画面的屏幕。

讲到这里可能有人要问了，那怎么才能生成一个正常打游戏的男孩啊？其实很简单，我们只需要玩一个文字游戏——"一个男生在卧室里打游戏"，将"打游戏"改为"双手横向拿着手机"，驱除掉"游戏"这个污染源，问题顺便就被解决了。

好了，你可以划走了。你知道的，大师的视频永远是后半段更精彩。

知道了这个坑，我们甚至可以反向利用它。比如说生成"一个男生在电脑上修图"——"修图"就是会抛弃你和客观规律的污染源。即使你修改成"男生看向显示器正面"，AI也依旧会翻转屏幕，让你而不是图片中的那个男生看到这个屏幕。那么你可以试着改成"一个男生看向显示器背面"——神奇的事情发生了，人理解起来不同寻常的语句，竟然可以让AI正确生成效果。是不是瞬间有一种拿捏住的感觉？

搞定了这个道理，想画一个身材好的男生，不要用"八块腹肌"这种"妈妈你看我有腹肌了"的表达，可以换成"尖锐腰窄"，这是一种透过衣服也能看到非常好效果的画面。熟练地找到这个技巧，将会大大降低你的充值概率。

如果说关键词联想陷阱是一种隐性的坑，那么接下来我们来聊一聊AI的一些显性的坑。

生成"宇航员背对着镜头、望向地球"的画面——为什么你明明写了"背对镜头"，AI却总是给你正脸？因为现代的AI还没有完善的空间模型，它在处理抽象动作的时候会进行具象化的补偿，也就是"视觉代偿"。

简单来说，AI会将你的提示词拆分成一个个词块去理解。AI没办法理解"背对"或者"望向"这种抽象的概念，所以要表达"背身"就只能画出更具象的背包和头盔做代偿；要表达"望向"就只能画出更具象的眼睛、面罩和正脸做代偿。当这两个词同时出现的时候怎么办呢？你会发现，恰好这些元素是可以在一个画面里被塞进去的——你看，用AI的思考方式是不是很简单地理解了为什么会得到那张图片？

所以解决办法也很简单，你只需要回避掉任何会引起视觉代偿的词汇。比如你想画宇航员看地球的背影，提示词就应该改为"一个宇航员站在月球上，背对着镜头，远方的太空中悬浮着一颗地球"——直接描述场景里能看到的视觉元素：背身的人和地球，而不是描述抽象的动作"望向"，这样就能一次性生成正确效果。

那么你还可以反向利用视觉代偿的特点去做一些更骚的操作，那就是用它去控制镜头。这是宇航员漂浮在太空的图片，如果你想拉近镜头，本质上就是希望宇航员在画面中的占比变大，那么你就可以用更多的提示词去描述头盔、面罩、宇航服细节，AI就不得不分配更多的注意力去还原这些，镜头变相地就能拉近。如果你想要特写，那就应该将描述的比重全部给宇航员的面罩和细节。

无关描述镜头的词语，诸如"特写""中景""全景""大全景"，在某些时候同样能生效，但是非常不稳定。而运用视觉代偿机制，几乎可以完全避免因为AI不听话而导致的抽卡。

这套方法同样适用于俯视角拍摄角度。如果你只写"一个宇航员俯视角拍摄"，会是这样的结果。但是如果你大量描述宇航员俯拍时画面中可以被看到的信息，诸如头顶、肩膀、氧气瓶顶部、头盔等元素，来迂回地解决拍摄角度的问题，这样生成的图片，宇航员被完全翻转了90度。

总结来说，后续生图的时候，你需要避免"先构建整个空间效果，再摆放摄像机位置"的错误理解，而应该去描述在这种场景下，你站在画面里能看到什么。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**AI生图频繁出错的根源是两类提示词陷阱——关键词联想污染和视觉代偿机制，掌握它们的规律就能告别抽卡、精准控图。**

### 2.3 核心概念精讲

#### 概念一：关键词联想污染

**是什么**：提示词中出现某个词，AI会自动激活与该词绑定的一整套刻板印象，即使这个词被否定，污染依然存在。

**为什么重要**：这是AI生图"莫名其妙出错"最常见的隐性原因，用户往往以为是AI随机抽卡，实则是提示词本身在作怪。

**详细解释**：

| 你写的词 | AI自动脑补的内容 |
|---|---|
| 护士帽 | 护士服 + 医院场景 |
| 不戴护士帽 | 依然出现医院场景 |
| 安全帽 | 工地场景 |
| 圣诞帽 | 圣诞节场景 |
| 厨师帽 | 厨房 + 做饭 |
| 游戏 | AI翻转手机方向，强行让屏幕朝向观众 |
| 修图 | AI翻转显示器，让屏幕朝向观众 |

**实际应用场景**：
- 想画打游戏的男生 → 删掉"游戏"，改为"双手横向拿着手机"
- 想画身材好的男生 → 删掉"八块腹肌"，改为"尖锐腰窄"
- 反向利用：想让显示器屏幕背对观众 → 写"男生看向显示器背面"，用AI的联想规律为己所用

---

#### 概念二：视觉代偿机制

**是什么**：AI没有完善的空间模型，无法理解"背对""望向"等抽象动作概念，会自动用更具象的视觉元素来替代表达这些概念。

**为什么重要**：这是AI生图在处理角度、朝向、动作时频繁出错的根本原因，也是控制镜头最有效的底层工具。

**详细解释**：

| 抽象描述 | AI的代偿方式 | 实际结果 |
|---|---|---|
| 背对镜头 | 画出背包、头盔（代表"背面"） | 人物依然正脸朝向镜头 |
| 望向地球 | 画出眼睛、面罩、正脸（代表"看"） | 正脸出现，无法背身 |
| 俯视角拍摄 | 给出普通视角或轻微俯角 | 角度不到位 |

**正确做法**：不描述抽象动作，改为描述"在这个场景下，画面里能看到什么视觉元素"。

> 错误写法：宇航员背对镜头望向地球
>
> 正确写法：一个宇航员站在月球上，背对着镜头，远方的太空中悬浮着一颗地球

**实际应用场景**：
- 俯视角拍摄宇航员 → 描述"头顶、肩膀、氧气瓶顶部、头盔"等俯拍时能看到的元素
- 拉近镜头 → 大量堆叠对头盔、面罩、宇航服细节的描述，迫使AI分配更多注意力到人物上
- 特写镜头 → 将描述比重全部集中在面部/局部细节，而非使用"特写"这个词

---

#### 概念三：用"描述画面"替代"描述意图"

**是什么**：生图提示词的正确思维模式——不是告诉AI你想要什么效果，而是描述一个理想画面里实际能看到的所有视觉元素。

**为什么重要**：这是解决上述两类陷阱的统一底层方法论，掌握它可以从根本上改变写提示词的思路。

**详细解释**：

| 错误思维（描述意图/动作） | 正确思维（描述视觉元素） |
|---|---|
| 打游戏 | 双手横向拿着手机 |
| 背对镜头望向地球 | 站在月球上背对镜头，远方悬浮着一颗地球 |
| 俯视角拍摄 | 描述头顶、肩膀、头盔顶部等俯拍可见元素 |
| 拉近镜头 | 大量描述头盔细节、面罩纹理、宇航服材质 |
| 八块腹肌 | 尖锐腰窄（透过衣服可见的体型效果） |

**实际应用场景**：适用于所有AI生图场景，尤其是涉及特定角度、朝向、动作、镜头距离时，优先用"画面里能看到什么"来构建提示词。

### 2.5 延伸思考

1. **关键词污染的边界在哪里？** 视频中的例子都是强关联词（护士帽→医院），那对于弱关联词，AI的联想强度是否不同？如何判断一个词是否会成为污染源，有没有系统性的测试方法？

2. **视觉代偿机制能否被完全绕过？** 视频提供的解法是"描述视觉元素而非抽象动作"，但如果场景极度复杂（比如多人物、多动作、多角度），描述所有视觉元素是否会反而引入新的污染源？如何在复杂场景下保持提示词的精准性？

3. **这套方法论对不同AI生图模型的适用性如何？** 视觉代偿和联想污染是AI的底层处理机制，但不同模型（Midjourney、Stable Diffusion、Flux等）的训练数据和架构不同，这些坑的表现形式和规避方法是否存在差异？

---

### 2.6 一句话记住它

> 别告诉AI你想要什么效果，告诉它你的画面里能看到什么——这才是AI生图真正听得懂的语言。
