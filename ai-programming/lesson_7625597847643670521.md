## 一、视频原文

上次做的小调查，提升很多新手小白，他们不知道 Vibe Coding 到底应该从哪一步开始。那今天我们就用一分钟时间来想清楚，市面上这些主流的开发工具，哪些是和非技术人员去进行 Vibe Coding 的工具。我已经准备好了一张参考表，那我们可以从两个维度去说。

第一个维度是全能型和专精型。想全能型的工具呢，它就是可以什么都做，写代码、改 bug、大范围修改都可以。如果大家什么都要去试一试的话，不妨偏全能型的工具。那像专精型的工具呢，其实它只是专注于某一方面，像我现在就是专门去做界面的，那这个 Cursor 的地方呢，它是就是专门去测试的。那如果我们是卡在某个集中的某一步不太擅长，或者说是不太好，想要精进的话，那我建议大家可以去找专精类型的工具，比较好。

第二个维度是半自动和全自动。那比如说就是助手型，就像 Cursor 就是那种我们点一下可以动一下，控制权还是在我们手里的。那全自动的 Agent 呢，就是数字员工，我们刚才说我们想干什么，它就可以自己去参加，然后自主阶段来跑通之后，最后直接交给我们产品。

那如果是纯新手的话，推荐这五个工具。第一个呢就是 Cursor，之前我已经说很多遍了，它是目前我觉得最适合小白的，因为它对网络没有任何的要求，它的配置也比较简单，并且它是完全免费的，如果大家最新手想去试一试，入门来说是完全够用的。第二个工具呢就是 Lovable，它就是最懂外行人的 UI 搭建神器，UI 在一个产品中其实也是非常重要的一个部分。然后呢就是 Bolt，如果我们想做 MVP，就是最小可行性产品，就用它就可以直接以不到为了。第四个工具呢就是 Vercel，是 Vercel 旗下出品的，就是把我们所有的想法可以变得非常高级的网页前端。第五个呢就是可能需要对它的专业知识能力有一定的要求，就是 Claude，我们用它做 Vibe Coding，你就可以把它当成另一个伙伴，我们只跟它边聊边对话，它就直接帮我们把有差别非常完美的做出来了。当然前提也是基于我们跟它的沟通中，非常有逻辑、条理清晰地把我们的需求列出来，那它给的东西基本上是不太会出错的。那大家如果有请酒画开会，就试一下吧。

### 2.3 核心概念精讲

#### Vibe Coding 工具的两大分类维度

**是什么**：用于描述和区分 AI 编程工具特性的两个核心维度：功能广度（全能 vs 专精）和自动化程度（半自动 vs 全自动）。

**为什么重要**：不同背景和需求的用户适合不同类型的工具，用错工具会导致效率低下甚至放弃，理解分类维度有助于快速找到适合自己的工具。

**详细解释**：
| 维度 | 类型 | 特点 | 适合场景 |
|---|---|---|---|
| 功能广度 | 全能型 | 写代码/改bug/大范围修改全覆盖 | 不确定自己需要什么，想全面探索 |
| 功能广度 | 专精型 | 专注某一环节 | 卡在某个步骤，想精进特定能力 |
| 自动化程度 | 半自动（助手型） | 点一下动一下，人保持控制权 | 想保持对产品的掌控感 |
| 自动化程度 | 全自动（Agent型） | AI 自主执行，最终交付产品 | 只关注结果，不想管过程 |

**实际应用场景**：新手建议从全能型+半自动的工具（如 Cursor）开始，熟悉后再尝试全自动 Agent。

---

#### 新手推荐的5款工具

**是什么**：专门针对非技术人员筛选的 Vibe Coding 入门工具清单，每款工具有其最适合的细分场景。

**为什么重要**：工具选择是新手最容易卡住的第一步，明确每款工具的定位能避免盲目尝试浪费时间。

**详细解释**：
| 工具 | 核心定位 | 最大优势 | 适合做什么 |
|---|---|---|---|
| Cursor | 新手首选 | 免费、无网络要求、配置简单 | 全能型入门，什么都可以试 |
| Lovable | UI 神器 | 最懂外行人的界面设计 | 产品界面搭建 |
| Bolt | MVP 工具 | 快速验证产品可行性 | 最小可行性产品开发 |
| Vercel (v0) | 前端生成 | 把想法变成高级网页前端 | 网页前端页面生成 |
| Claude | 对话协作 | 边聊边做，像找伙伴一起工作 | 需求清晰时的全流程协作 |

**实际应用场景**：想做一个产品 demo，可以先用 Bolt 跑通 MVP，再用 Lovable 优化界面，最后用 Claude 打磨细节。
