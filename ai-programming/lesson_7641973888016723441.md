# 教案：Agent 评测核心指标与方法

> 适合人群：想转型或深耕 AI 产品方向、需要应对面试或实际落地项目的产品经理

---

## 一、视频原文

最近在面试当中，很多 AI 产品经理都会被问到什么是 Agent 的评测。很多人一提到评测，第一个反应就是看他回答准不准确。但事实上在实际项目当中，如果你只是这么理解，那么你还是停留在传统聊天机器人的视角。

所以我今天想梳理一下，Agent 评测到底在看什么，他到底有哪些核心的指标，以及具体怎么做。对于 AI 产品经理从零到进阶的所有材料我也都准备好了，可以老地方来拿。

首先我们需要明确他到底在评什么内容。不是说一上来就看效果好不好，而是要看我们定的目标。比如说我们不同的 Agent 他会解决不同的一些问题，那么他要具体完成什么样的任务，什么样的情况下算完成，什么样的情况下算失败，这个是需要我们前置去做一个判断的。

然后进一步，我们不能只看结果，而且要看过程。因为在很多传统的 AI 问答当中，我们只去看最终的答案是不是合理。但很多时候我们会漏掉一些中间的步骤，比如说怎么去理解用户意图、拆解任务、调用接口等等。所以在评测这边，我们一定要分成两层：一层是结果的评测，看他最终有没有完成对的任务；一层是过程的评测，看他完成任务的路径是否正确。

以及我们的 Agent 评测他要哪些核心指标：首先第一个，任务完成率，看他最终的一个完成结果；其次就是意图理解准确率；还有就是工具调用的准确率；以及他最后的任务路径的正确性。

那么评测具体怎么做？首先第一步，我们应该要去拆解定义任务场景，不要一上来就看这些指标，我们应该把它列成定义的场景，然后再建立对应的测试集，再进一步去看我们整体的定义评分标准，以及说我们离线测评和在线测评做一个综合的结合。离线测评更多是去看上线前，而在线测评更多关注的是真实用户使用的过程当中，我们的 Agent 是否有效。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**Agent 评测不能只看回答准不准，要同时评"结果"和"过程"，掌握任务完成率、意图理解、工具调用、路径正确性四大核心指标，才是真正落地级的评测思维。**

### 2.3 核心概念精讲

#### 概念一：结果评测 vs 过程评测

**是什么**：Agent 评测必须分两层——看最终任务有没有完成（结果），以及完成任务的中间路径是否合理（过程）。

**为什么重要**：
传统聊天机器人只看最终答案对不对，但 Agent 要完成的是多步骤任务，中间某一步出错（比如调错接口、误解意图）即使最后"看起来"有输出，也可能是错误的完成方式，埋下风险。

**详细解释**：

| 评测层次 | 关注点 | 典型问题 |
|---|---|---|
| 结果评测 | 最终任务有没有完成 | Agent 有没有达成用户目标？ |
| 过程评测 | 中间步骤是否正确 | 意图理解对了吗？工具调用对了吗？路径合理吗？ |

**实际应用场景**：
比如一个自动订票 Agent，结果评测看"票订成功了没有"，过程评测看"它有没有正确理解出发地、有没有调对查询接口、拆解步骤顺序对不对"。

---

#### 概念二：四大核心评测指标

**是什么**：衡量 Agent 质量的四个关键量化维度——任务完成率、意图理解准确率、工具调用准确率、任务路径正确性。

**为什么重要**：
有了具体指标，评测才能量化、可比较、可迭代，而不是靠主观感受说"感觉还不错"。

**详细解释**：

| 指标 | 含义 |
|---|---|
| 任务完成率 | Agent 最终成功完成目标任务的比例 |
| 意图理解准确率 | Agent 正确理解用户真实需求的比例 |
| 工具调用准确率 | Agent 在正确时机调用正确工具/接口的比例 |
| 任务路径正确性 | Agent 执行任务的步骤顺序和逻辑是否合理 |

**实际应用场景**：
在面试或项目汇报中，能说出这四个指标并解释各自含义，体现的是真正理解 Agent 落地逻辑，而不是停留在"看答案准不准"的聊天机器人思维。

---

#### 概念三：离线测评 vs 在线测评

**是什么**：评测分两个阶段——上线前用构造好的测试集验证（离线），上线后观察真实用户使用时的表现（在线）。

**为什么重要**：
离线测评能在产品上线前发现明显问题，但覆盖不了所有真实场景；在线测评能捕捉真实用户行为带来的边缘情况，两者结合才能全面保障 Agent 质量。

**详细解释**：

| 测评类型 | 时机 | 关注点 |
|---|---|---|
| 离线测评 | 上线之前 | 用预设测试集验证指标是否达标 |
| 在线测评 | 上线之后 | 真实用户使用中 Agent 是否持续有效 |

**实际应用场景**：
产品灰度发布前跑离线测评，确认任务完成率达到阈值；上线后持续监控在线数据，发现真实用户触发了哪些离线没覆盖到的失败场景。

---

### 2.4 速查表

| 维度 | 传统聊天机器人评测 | Agent 评测 |
|---|---|---|
| 核心关注点 | 回答准不准确 | 任务有没有完成 + 过程是否正确 |
| 评测层次 | 单层（结果） | 双层（结果 + 过程） |
| 核心指标 | 答案准确率 | 任务完成率、意图理解准确率、工具调用准确率、路径正确性 |
| 评测流程 | 直接看输出 | 拆解场景 → 建测试集 → 定评分标准 → 离线+在线综合 |
| 测评时机 | 通常只在上线前 | 上线前（离线）+ 上线后（在线）持续结合 |

---

### 2.5 延伸思考

1. **工具调用准确率和任务完成率之间是什么关系？** 工具调用出错但任务最终完成了，这种情况该如何评分？是否说明某些"错误路径"在实际中是可接受的？

2. **在线测评中，如何定义一个真实用户任务"成功完成"？** 是由系统判断还是需要用户反馈？两种方式各有什么局限性？

3. **不同类型的 Agent（比如信息检索型 vs 操作执行型）在这四个指标上的权重应该一样吗？** 如何根据 Agent 的具体业务场景来调整评测重点？

---

### 2.6 一句话记住它

> 评测 Agent 不是看它"说得准不准"，而是看它"做得成不成、走得对不对"——结果和过程缺一不可。
