## 一、视频原文

大家好，我是奔走。走在各种 AI 软件神仙打架的时代，Claude 好像在慢慢被大家遗忘了，很多人对它的印象还停留在几个老问题上——记忆烂、上下文短、文件管理像个垃圾堆。

事实是，大家的认知已经配不上它的进化速度了。上期视频我们讲了 Claude 的一些使用技巧，今天我用三条干货，手把手带你打造自我进化的 Claude。来，走着学着进化。

**第一步，学会用 Standing Orders 长驻指令。**

当你发现自己每天都在对 Claude 说同一类话，这时候不要再写一次性 Prompt，你要把它升级成 Standing Orders。

举一个简单的例子，比如你想让 Claude 帮你每天收各个平台里跟 AI 有关的信息，你只需要做三件事：

第一步，给目标——告诉 Claude，我需要你帮我写一个 Standing Orders，用 OpenCLI 工具收多个平台有关 AI 的信息；第二步，给资料——去 GitHub 找到 OpenCLI 的用法，把链接发给它，Claude 会自己抓信息、自己消化，并提取关键命令和最佳实践，然后写进 Standing Order；第三步，看成品——当你让 Claude 写完一条高质量的 Standing Order 后，遇到新的指令可以让它直接参考这个模式去写。

我后面让它帮我写的"文档去 AI 化指令 Humanizer Standing Order"就是这样——先抓了 Copy.ai 的完整 Humanizer 文档，全部内化成一个美观的自动过检查清单。这条 Order 写完以后，我就不需要再提"说话别太 AI 了"，因为 Claude 已经被训练好了。

**第二步，灵活使用 Heartbeat 和 Cron 自动干活机制。**

两者的区别在于：Heartbeat 是隔一阵子看一眼，有事说事，没事继续睡，最短 30 分钟一次，能把好几件事叠到一起跑；Cron 是闹钟响了到这个点必须干活，精确到秒，一条任务一个坑。

如果一个任务要做到分钟级的响应，就用 Cron；如果要和其他事情一起跑任务，就用 Heartbeat。

你不需要自己去研究 Heartbeat 命令怎么写，不需要去背 Cron 表达式的语法，你只需要对 Claude 说一句话："帮我监控我的自媒体账号数据，每天自动查一次播放量和点赞数。"然后 Claude 自己会干这些事——去读 OpenCLI 的文档，写 Heartbeat.md，订好每条命令的间隔和步骤，剩下来它自己就能跑通了。

**第三步，对接外界专业软件，搭建低摩擦操作系统。**

Claude 自带了四种接入方式，包括 MCP、CLI、Harness 和内置工具。

第一种，MCP——可以把它理解成 AI 工具的标准接口，只要某个软件有 MCP 服务，Claude 就有机会对接它。像 Notion、GitHub、Airtable、Figma、Canva、Slack、GitLab，全线明确的软件优先用 MCP 对接。它适合查资料、管项目、改任务、写记录，或者跨上下文查数据库这类事情。

但很多平台没有这么规范的接口，比如各类自媒体平台、实时数据源。这些地方我们要的不是调用功能，而是获取平台内容——评论、字幕、热榜、创作者数据。这时候就用 CLI 工具，比如我常用的 OpenCLI，它能把原本只能手动操作的网站变成 Claude 可以调用的命令。比如做选题调研时，它能搜英文趋势、找同类视频、看高赞回答和用户痛点，再把结果统一汇总回来。

第三种，Harness——适合对接专业 Agent，比如代码任务可以交给 Codex。Claude 负责接任务、拆任务、判断逻辑、收结果，Codex 负责进仓库、改代码、跑测试，最后结果再回到 Claude。

第四种，内置工具——Claude 本身就有浏览器、文件读写、终端命令、搜索、消息、硬实任务、自 Agent 系统等功能。

什么时候开始用外接工具？很简单，只要任务不是"回答我一个问题"，而是要真的操作网页、文件、代码、表格、文档、项目系统，就该用外接工具。

最后几个例子让你一看就会。比如 Claude 帮我做这些视频——低级玩法是让模型自己瞎搞，高级玩法是：Claude 用 OpenCLI 抓平台数据，用浏览器读官方文档，用 OpusClip 存资料，用 Airtable 管理选题状态，用 Canva 做封面，需要代码 Demo 就把任务交给 Codex，用 HeyGen 生成视频动效，剩下的就是你的任务——把所有结果按脚本逻辑剪辑成片进视频，最后就成了今天你看到的这期视频内容了。

我是奔走，记得点赞关注，我们下期见。

### 2.3 核心概念精讲

#### 概念一：Standing Orders 长驻指令

**是什么**：把你反复对 Claude 说的同类 Prompt，提炼成一条持久生效的结构化指令，让 Claude 每次执行任务时自动遵守，无需重复说明。

**为什么重要**：
重复写 Prompt 是效率黑洞。Standing Orders 相当于给 Claude "预装规则"，一次配置，长期生效，把人从重复劳动中解放出来。

**详细解释**：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 第一步：给目标 | 用自然语言描述你的需求 | 例："帮我写一个 Standing Order，用 OpenCLI 收集多平台 AI 信息" |
| 第二步：给资料 | 把相关工具的文档链接发给 Claude | Claude 自动抓取、消化、提炼关键命令和最佳实践 |
| 第三步：看成品 | 审阅 Claude 生成的 Standing Order | 质量满意后，可作为模板让 Claude 参考写其他指令 |

**实际应用场景**：
- 内容创作：写一条"文风去 AI 味"的 Humanizer 指令，之后所有输出自动过滤 AI 腔
- 信息收集：写一条"每日 AI 资讯聚合"指令，自动抓取多平台内容

---

#### 概念二：Heartbeat 与 Cron 自动调度机制

**是什么**：Claude 中两种让任务按时间自动触发执行的机制，无需人工手动启动。

**为什么重要**：
有了自动调度，Claude 才能真正"自己跑"，从被动回答变成主动执行，实现无人值守的自动化工作流。

**详细解释**：

| 维度 | Heartbeat | Cron |
|------|-----------|------|
| 触发方式 | 间隔触发（轮询） | 定点触发（精确时间） |
| 最小间隔 | 30 分钟 | 精确到秒 |
| 并发能力 | 支持多任务叠加运行 | 一条任务一个坑 |
| 适用场景 | 多任务并行、对时间精度要求不高 | 分钟级响应、强时间精度需求 |
| 典型案例 | 每天自动查播放量+点赞数 | 整点准时触发某个操作 |

**实际应用场景**：
- 用 Heartbeat 每天自动监控自媒体账号数据（播放量、点赞数）
- 不需要手写配置语法，直接用自然语言告诉 Claude，它会自己读文档、写 Heartbeat.md 并跑通

---

#### 概念三：四种外接工具接入方式

**是什么**：Claude 连接外部软件和数据源的四种标准化方式，让 Claude 从"只能聊天"变成"真的能操作"。

**为什么重要**：
真实工作中大量任务需要操作文件、网页、数据库、代码仓库。没有外接工具，Claude 只是一个问答机器；有了外接工具，它才是一个能干活的 Agent。

**详细解释**：

| 接入方式 | 适用场景 | 代表软件/工具 | 核心能力 |
|----------|----------|---------------|----------|
| MCP | 有标准接口的主流软件 | Notion、GitHub、Airtable、Figma、Canva、Slack | 查资料、管项目、跨上下文操作数据库 |
| CLI（如 OpenCLI） | 无 MCP 的平台（自媒体、实时数据） | 各类自媒体平台、热榜数据源 | 抓评论、字幕、热榜、创作者数据 |
| Harness | 需要专业 Agent 协作的复杂任务 | Codex（代码执行） | Claude 拆任务，Codex 进仓库改代码跑测试 |
| 内置工具 | 通用文件/网页/系统操作 | — | 浏览器、文件读写、终端命令、搜索、自 Agent |

**实际应用场景**：
- 做选题调研：OpenCLI 搜英文趋势 → 找同类视频 → 看高赞回答和用户痛点 → 汇总结果
- 做内容视频全流程：OpenCLI 抓数据 + 浏览器读文档 + Airtable 管选题 + Codex 写代码 Demo + Canva 做封面

### 2.4 速查表

| 功能需求 | 推荐方案 | 关键特点 |
|----------|----------|----------|
| 重复 Prompt 太多 | Standing Orders | 一次配置，长期生效 |
| 任务需要定时自动执行 | Heartbeat 或 Cron | 无需人工触发 |
| 多任务并行、时间不敏感 | Heartbeat | 最短 30 分钟间隔，可叠加 |
| 精确定点触发、分钟级响应 | Cron | 精确到秒，单任务单坑 |
| 对接 Notion/GitHub/Figma 等主流软件 | MCP | 标准接口，功能调用 |
| 抓取自媒体平台数据/热榜 | CLI（OpenCLI） | 把手动操作变成可调用命令 |
| 代码任务、进仓库改代码 | Harness + Codex | Claude 拆任务，Codex 执行 |
| 操作文件/网页/终端 | 内置工具 | 无需额外配置 |

3. **当 Claude 通过 Harness 调度 Codex 这类专业 Agent 时，"Claude 负责判断逻辑、Codex 负责执行"的分工模式，是否会成为未来 AI 协作的标准范式？这种分层 Agent 架构有哪些潜在的风险和边界？**
