## 一、视频原文

换工具不换脑。

最近有人在评论区问我，到底是用 Codex 还是用 Claude Code？我以前也会特别纠结这一类的问题——哪个工具更新了我就想试一下，哪个模型变强了我就怕自己错过，别人说的某个工具效率高，我就开始怀疑自己是不是又落后了。

那我现在想明白了一件事情：很多人问的其实不是工具的问题。你问 Codex 可不可以、Claude Code 到底行不行，这些问题都属于同一类问题——是属于"手"的问题。

我现在是把自己的工作分成三块：一个叫做硬盘，一个叫做大脑，第三个叫做手。

Obsidian 是我的硬盘，它存的是我的判断、我踩过的坑、我做过的项目、我写过的稿子、梳理过的问题。

Claude Code 或者其他的大模型，他们属于大脑的层面，负责的是理解、推理、生成，把这些内容重新组织出来。

Claude Code 或者 Codex 这些，属于工具，当然最近会有越来越多的工具，他们是属于"手"——手是负责执行的，帮你写代码、帮你改文件、帮你生成内容、帮你跑任务。

所以你会发现，手是可以换的。今天我用 Claude Code，明天我就可以用 Codex，只要我的硬盘在、我的判断在、我的上下文还在，我换一个工具其实并不会从零开始。

那我今天就演示一下——这是 Claude Code 我们之前的对话，我用 Codex 重新帮我整理梳理了相关的内容，他还帮我增加了注释，帮我做整体的一个更新。这样对我来说其实是很直接的一个调用。

这也是为什么我觉得真正有复利的不是工具，工具本身没有复利。你今天花两周学会了一个工具，过几个月又会有新的工具出来，那这两周你沉淀下来的工作方法、判断标准、踩坑记录，这些是你可以直接带走的。

所以我从 Claude Code 切换到 Codex，上手是非常快的。当然未来会有各种各样的新软件，你需要的就是在你的电脑中去配置相关的信息，而不是说未来有其他的工具来了，你又是从零开始。

所以我并不会一直在问哪个工具最强，我会先问我的硬盘里面有没有东西。如果你的 Obsidian 里面什么都没有，其实差距并不会很大，因为基本上都是从零开始。但是如果你的硬盘里面已经保存了你的判断、你的经验、你的上下文，那工具换来换去，也只是换一个手，真正不变的是你的神经、你的系统。

所以我的建议其实也是：花两分钟，每天把和 AI 沟通的事情记录下来，不用写得很漂亮，就记录做了什么、干了什么事情。你有一个机内记录的过程，其实就慢慢能够让 AI 读懂你到底是什么样的风格。两个月之后，你积累的这些品味和收藏和风格，在你将来的 AI 使用中就会用得上。这我觉得才是一个工作流应该有的状态。

所以无论工具怎么迭代，核心还是你的神经和你的大脑，你的硬盘存在哪里。

今天最后一个问题就是，你自己是在怎么用这些 AI 工作流？可以评论区说一下。喜欢点赞关注。

### 2.3 核心概念精讲

#### 概念一：三层工作流框架（硬盘 · 大脑 · 手）

**是什么**：将 AI 辅助工作拆解为三个功能层，每层承担不同职责，互不混淆。

**为什么重要**：大多数人在焦虑"换不换工具"时，其实是把三层混为一谈了——以为换了"手"就换了整个系统，导致每次工具迭代都像从零开始。

**详细解释**：

| 层级 | 对应工具（举例） | 核心职责 | 可替换性 |
|------|----------------|----------|----------|
| 硬盘 | Obsidian | 存储个人判断、经验、上下文、踩坑记录 | ❌ 不可替换，是个人资产 |
| 大脑 | Claude 等大模型 | 理解、推理、生成、重新组织内容 | ⚠️ 可切换，但需重新适配 |
| 手 | Claude Code、Codex 等 | 执行：写代码、改文件、跑任务 | ✅ 随时可换，成本极低 |

**实际应用场景**：视频中演示了将 Claude Code 的历史对话直接交给 Codex 重新整理，并增加注释、做整体更新——因为上下文存在 Obsidian 里，换手几乎没有摩擦。

---

#### 概念二：工具没有复利，沉淀才有复利

**是什么**：复利在这里指"越用越值钱、越用越省力"的积累效应。工具本身不具备这一属性，但工作方法、判断标准和踩坑记录具备。

**为什么重要**：很多人花大量时间"学工具"，但工具几个月就会被新工具替代，学习投入清零。而沉淀在 Obsidian 里的内容，无论工具怎么换，都可以直接带走、直接调用。

**详细解释**：

- 花两周学会一个工具 → 几个月后新工具出现 → 两周投入归零
- 花两周同时记录工作方法和判断 → 几个月后新工具出现 → 记录直接迁移，上手极快

**实际应用场景**：博主提到，正因为 Obsidian 里有积累，从 Claude Code 切换到 Codex 时"上手非常快"，不需要从零开始配置思路。

---

#### 概念三：每日两分钟记录习惯

**是什么**：每天花约两分钟，把当天与 AI 沟通的内容、做了什么事、有什么判断，简单记录到 Obsidian 中。

**为什么重要**：这是"硬盘"能持续积累的唯一方式。没有日常输入，硬盘就是空的，换工具和不换工具没有差别。

**详细解释**：

- 不需要写得漂亮，流水账即可
- 记录内容包括：今天用 AI 做了什么、遇到什么问题、用了什么方法解决
- 积累两个月后，这些记录会形成你个人的"品味、风格、偏好"，AI 可以直接读取并匹配你的习惯

**实际应用场景**：两个月的日常记录 → Obsidian 形成个人知识库 → 新工具接入时直接读取上下文 → AI 能快速理解你的工作风格，输出质量更高。

### 2.4 速查表

| 对比维度 | 工具（手） | 个人积累（硬盘） |
|----------|-----------|----------------|
| 代表工具 | Claude Code、Codex 等 | Obsidian |
| 核心价值 | 执行效率 | 上下文与判断 |
| 有无复利 | ❌ 无，会被新工具替代 | ✅ 有，越积越值钱 |
| 迁移成本 | 低，随时可换 | 高，是个人资产不能丢 |
| 正确投入方式 | 够用即可，不必深度绑定 | 每天 2 分钟持续记录 |
| 换工具后的影响 | 几乎为零（如果硬盘有内容） | 不受影响，直接带走 |

### 2.5 延伸思考

1. **硬盘的质量比数量更重要吗？** 如果 Obsidian 里记录了大量内容，但都是流水账、没有提炼判断，这样的"硬盘"在切换工具时真的能发挥作用吗？什么样的记录才算是有效积累？

2. **"大脑层"的选择真的不重要吗？** 视频把大模型归为"大脑"，认为"手"可以随便换——但不同大模型的能力差异很大，大脑的选择会不会反过来影响硬盘里沉淀内容的质量？

3. **这套框架对新手同样适用吗？** 博主的逻辑建立在"硬盘已有积累"的前提上。对于刚开始用 AI 工具、Obsidian 里还什么都没有的人，第一步应该记录什么，才能尽快让这套系统转起来？

---

### 2.6 一句话记住它
