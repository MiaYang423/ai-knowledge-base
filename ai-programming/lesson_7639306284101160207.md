## 一、视频原文

一起来看看字节大模型 Agent 算法面，看看你能答上来几道。这套题整体难度偏难，但是前面是基础，后面才是真正区分水平的地方。我是校哲，点赞收藏加关注，我们马上开始。

**第一道题：Transformer 的结构组成和各部分作用是什么？**
这题先讲清楚主干。核心是 Embedding、位置编码、多头注意力、前馈网络、残差连接和 Layer Norm。注意力负责建模上下文关系，前馈网络做非线性变换，残差和归一化保证深层训练稳定。

**第二道题：怎么降低 Transformer 的计算复杂度？**
关键是注意力的复杂度来自序列长度的平方。工程上可以用线性注意力、局部窗口注意力、滑动窗口、低秩分解，或者 FlashAttention 这类优化。面试重点不是背名字，而是知道长文本场景怎么降延迟和显存。

**第三道题：LoRA 微调的原理是什么？秩 r 怎么选？**
LoRA 的核心是冻结原模型参数，只在部分线性层旁边加低秩增量矩阵。r 越大表达能力越强，但显存、训练成本和过拟合风险也更高。一般先从小 r 做基线，再看任务复杂度和验证集效果调整。

**第四道题：KV Cache 是什么？为什么能提升推理速度？**
KV Cache 缓存的是历史 token 计算出的 K、V 值。生成下一个 token 时就不用反复重算前面所有上下文，只算新增部分。它主要优化自回归推理延迟，但会增加显存占用，长上下文要考虑淘汰和压缩策略。

**第五道题：RAG 的完整流程怎么讲？**
流程一般是：文档切分、清洗、向量化入库；查询时做向量检索、召回、重排，拼上下文再交给大模型生成。工程上还要关注切片粒度、召回率、全链路过滤、引用溯源，以及答案不确定时的兜底策略。

**第六道题：向量数据库里时间衰减对召回有什么影响？**
这题考的不是公式，而是新旧知识怎么平衡。可以把时间作为排序特征，给新内容更高权重，但不能简单覆盖已有相关性。常见做法是向量相似度加时间分、分层召回，最后用重排模型统一打分。

**第七道题：微调训练数据怎么构建？如何保证多样性和质量？**
核心是先定义任务边界，再从真实日志、人工标注、合成数据里构造样本。质量上要去重、去噪、做安全过滤和人工抽检；多样性上要覆盖不同意图、难度边界 case，不能只堆同一种高频问法。

**第八道题：RAG 加知识图谱的 Agent 系统，图谱更新机制怎么设计？**
关键是把离线构建和在线增量分开。离线做实体抽取、关系抽取和一致性校验；在线把新增知识走消息队列或事件流增量更新。实时性靠异步刷新和版本控制，一致性靠冲突检测、回滚和可观测指标。

**第九道题：训练 LoRA 冻结哪些层？依据是什么？**
一般不会盲目全量打开，可以优先在注意力的 Q、V 或者部分 MLP 层加 LoRA。看任务是偏知识、偏格式还是偏推理能力。依据是验证集效果、训练稳定性、参数量和部署成本，最终要用实验说话。

**第十道题：高并发 Agent 系统里怎么优化召回和生成延迟？**
召回阶段可以做缓存、并行检索、分片、粗排加精排；生成阶段要用流式输出、KV Cache、批处理和超时降级。真正落地还要监控 P95 延迟，区分是模型慢、检索慢还是工具调用慢。

**第十一道题：大规模 Agent 系统的多线程和多进程资源调度怎么设计？**
核心是把任务拆成可调度单元，比如检索、工具调用、模型生成和后处理。CPU 密集型走进程池，I/O 密集型走异步或线程池，再配合队列、优先级、限流、熔断和资源隔离，避免单个任务拖垮整体系统。

好，这套面经我们就分享到这里。这套题表面看是在问 Transformer、LoRA、RAG 和 Agent 的调度，其实真正考的是 Agent 落地能力和系统设计能力。希望在评论区看到你的答案。

### 2.3 核心概念精讲

#### 概念一：LoRA（低秩适配微调）

**是什么**：一种参数高效微调方法，冻结原模型全部参数，只在部分线性层旁边插入两个小矩阵（低秩增量矩阵），通过训练这两个小矩阵来适配下游任务。

**为什么重要**：
全量微调大模型显存和计算成本极高，LoRA 用极少参数量达到接近全量微调的效果，是目前工业界最主流的大模型微调方案。

**详细解释**：

| 参数 | 含义 | 影响 |
|------|------|------|
| 秩 r（小） | 低秩矩阵的维度小 | 参数少、显存低、训练快，但表达能力弱 |
| 秩 r（大） | 低秩矩阵的维度大 | 表达能力强，但显存高、过拟合风险大 |
| 加 LoRA 的层 | Q/V 层优先，也可加部分 MLP | 取决于任务是偏知识、偏格式还是偏推理 |

**实际应用场景**：
- 选秩 r：先用小 r（如 4 或 8）跑基线，再根据验证集效果和显存预算往上调
- 选层：任务偏知识注入优先动注意力层，任务偏推理可考虑 MLP 层，最终靠实验验证

---

#### 概念二：KV Cache（键值缓存）

**是什么**：在自回归生成过程中，将已生成 token 对应的 Key 和 Value 向量缓存下来，生成下一个 token 时直接复用，不重复计算。

**为什么重要**：
没有 KV Cache，每生成一个新 token 都要重算所有历史上下文的注意力，复杂度随序列长度线性增长，推理延迟极高。KV Cache 是大模型推理提速的核心机制。

**详细解释**：

| 方面 | 说明 |
|------|------|
| 优化目标 | 自回归推理延迟 |
| 代价 | 显存占用随上下文长度增加 |
| 长上下文挑战 | 需要淘汰策略（丢弃早期 token 的缓存）或压缩策略（合并/量化缓存） |

**实际应用场景**：
高并发 Agent 系统中，KV Cache 与流式输出、批处理配合使用，是生成侧延迟优化的标配手段。

---

#### 概念三：RAG（检索增强生成）完整工程流程

**是什么**：在大模型生成前，先从外部知识库检索相关内容，将检索结果拼入上下文，再由模型生成答案，解决模型知识过时和幻觉问题。

**为什么重要**：
大模型参数内的知识是静态的，RAG 让模型能实时访问最新、私有的知识，是 Agent 系统落地的核心架构之一。

**详细解释**：

| 阶段 | 步骤 | 工程关注点 |
|------|------|------------|
| 离线建库 | 文档切分 → 清洗 → 向量化 → 入库 | 切片粒度（太大召回噪音多，太小语义不完整） |
| 在线检索 | 向量检索 → 召回 → 重排 | 召回率、全链路过滤 |
| 生成阶段 | 拼上下文 → 大模型生成 | 引用溯源、答案不确定时的兜底策略 |

**实际应用场景**：
- 加入时间衰减：对时效性强的知识库，用"向量相似度 + 时间分"做分层召回，再用重排模型统一打分
- 加入知识图谱：离线做实体/关系抽取，在线通过消息队列增量更新，用版本控制和冲突检测保证一致性

---

#### 概念四：高并发 Agent 系统的延迟优化与资源调度

**是什么**：在多用户并发场景下，通过合理的任务拆分、并发模型选择和保障机制，让 Agent 系统在高负载下仍能保持低延迟和高稳定性。

**为什么重要**：
单机 demo 和生产环境的核心差距就在系统设计能力，这是高级算法岗位真正的区分点。

**详细解释**：

| 优化维度 | 具体手段 |
|----------|----------|
| 召回侧延迟 | 结果缓存、并行检索、索引分片、粗排+精排两阶段 |
| 生成侧延迟 | 流式输出、KV Cache、批处理（batching）、超时降级 |
| 任务调度 | CPU 密集型用进程池，I/O 密集型用异步/线程池 |
| 系统稳定性 | 队列 + 优先级 + 限流 + 熔断 + 资源隔离 |
| 监控 | 监控 P95 延迟，区分瓶颈来自模型、检索还是工具调用 |

**实际应用场景**：
面试中回答此类题，要先说"把任务拆成可调度单元"，再分别说每个环节的优化策略，最后说监控和兜底，体现系统化思维。

### 2.5 延伸思考

1. **LoRA 和全量微调的边界在哪里？** 视频说 LoRA 是主流方案，但什么情况下必须做全量微调？任务复杂度、数据规模、模型结构的哪些因素会让 LoRA 失效？

2. **RAG 和长上下文大模型（如支持 100K token 的模型）之间是竞争还是互补关系？** 如果模型上下文窗口足够大，RAG 的检索和重排环节还有必要存在吗？各自的适用边界在哪里？

3. **视频最后说"真正考的是 Agent 落地能力和系统设计能力"，这意味着什么？** 一个算法工程师和一个系统工程师在 Agent 项目中的职责边界在哪里？纯算法背景的候选人如何补齐系统设计的短板？
