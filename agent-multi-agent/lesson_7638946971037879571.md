# 教案：Agent Memory 系统工程细节详解

> 来源视频：[抖音博主 · 抖音](https://www.douyin.com/video/7638946971037879571)
> 适合人群：正在准备大厂 Agent 二面、需要把 Memory 讲到工程细节的 AI 工程师

---

## 一、视频原文

美团这场 Agent 二面专门挑那种你以为懂了、其实没做过的点来问。Memory 这类题最容易答虚。你说长期记忆、短期记忆，面试官下一句就会问：怎么写入、怎么去重、冲突了谁生效？我是小哲，这期重点看你能不能把记忆系统讲到工程细节。

**第一道题：用一个例子解释 Memory 是怎么做的，以及怎么插入上下文的。**

可以举过敏信息这个例子。用户说"我对花生过敏"，系统先抽取成结构化记忆，比如 Allergy 等于 Peanut，再写入长期存储。下次用户问推荐餐厅时，检索到这条记忆，插到用户问题前面的 Memory 区，让模型在生成答案前先看到约束。

**第二道题：Memory 怎么考虑上下文长度问题？**

核心是不能把所有记忆都塞进去。工程上一般会做三步：先按用户场景和时间过滤，再用向量或者关键词召回相关记忆，最后按重要性和新鲜度排序，只放最相关的几条。上下文预算要提前分配，不能挤占工具结果和用户当前问题。

**第三道题：结构化信息和文本信息之间如何区分，会不会重复？**

结构化信息适合存稳定字段，比如过敏、偏好、地区、会员等级，方便精确过滤和规则判断。文本信息适合存长描述，比如用户的写作风格、项目背景、历史目标。会不会重复，关键看是否有同一个事实 ID，抽取时要做去重和归并。

**第四道题：记忆冲突怎么解决？比如用户前后说了不同的过敏信息。**

这题考的不是存储而是冲突策略。一般要保留历史版本，但生效时按可信度、时间和来源做判断。用户明确说"我现在不对花生过敏了"，就让新记忆覆盖旧记忆，并把旧记忆标成失效。高风险信息比如过敏，最好在关键场景再追问确认。

**第五道题：记忆是怎么写入上下文的？上下文从头到尾结构是怎样的？**

常见结构是：System Prompt 放规则，Developer 或业务指令放流程，Memory 放用户长期信息，Conversation 放最近对话，Tool Result 放工具返回，最后是当前用户问题。Memory 一般不要直接混进历史对话里，而是单独成块，标明来源、时间和约束强度。

**第六道题：上下文结构是不是 System Prompt、Memory、Tool Result、Conversation 这四个环节？**

可以这么理解，但面试里要说得更严谨。真实系统里通常还会有开发者指令、检索文档、短期摘要、工具调用记录和当前任务状态。顺序也不是固定模板，关键是高优先级规则靠前，当前问题靠后，工具结果和记忆要有清晰标注，避免模型混淆来源。

**第七道题：上下文超长了怎么办？有没有考虑过上下文压缩？**

肯定要考虑。做法不是简单截断，而是分层压缩：最近几轮对话原文保留，较早对话压成任务摘要，工具结果只保留结论、参数和关键证据，长期记忆走检索注入。这样既控制 Token 又尽量保留决策链和用户约束。

**第八道题：压缩会丢失工具调用历史，导致模型重复调用工具，这个问题怎么解决？**

工程上要把工具历史变成可读的状态，而不是只依赖原始调用记录。比如记录"已经查过什么、用了什么参数、结果是什么、是否还有效"，压缩后把它写进任务状态区。模型如果想重复调用工具，先检查状态，只有信息过期、缺失或者用户要求刷新时才再次调用。

好，这套面经我们就分享到这里。这套题表面看是在问 Memory 和上下文管理，其实真正考的是 Agent 落地能力。希望在评论区看到你的答案。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**Agent Memory 面试不能只说"长期记忆短期记忆"，你需要能讲清楚写入方式、去重策略、冲突处理、上下文结构和压缩方案这五个工程细节。**

---

### 2.2 知识地图

```
Agent Memory 系统工程细节
├── 记忆类型
│   ├── 结构化记忆（稳定字段：过敏、偏好、会员等级）
│   └── 文本记忆（长描述：写作风格、项目背景）
├── 记忆写入与检索
│   ├── 抽取 → 结构化 → 写入长期存储
│   ├── 检索：按场景/时间过滤 + 向量/关键词召回
│   └── 排序：重要性 + 新鲜度，只取最相关几条
├── 记忆冲突处理
│   ├── 保留历史版本
│   ├── 生效策略：可信度 + 时间 + 来源
│   └── 高风险信息：关键场景追问确认
├── 上下文结构
│   ├── System Prompt（规则）
│   ├── Developer 指令（流程）
│   ├── Memory 区（用户长期信息，单独成块）
│   ├── 检索文档 / 短期摘要
│   ├── Tool Result（工具返回）
│   ├── Conversation（最近对话）
│   └── 当前用户问题
└── 上下文压缩
    ├── 分层压缩（近期原文保留，早期压摘要）
    ├── 工具结果只保留结论+参数+关键证据
    └── 工具历史写入任务状态区，防止重复调用
```

---

### 2.3 核心概念精讲

#### 记忆写入与上下文注入

**是什么**：将用户信息抽取为结构化记忆并存入长期存储，在后续对话中按需检索并注入上下文的完整流程。

**为什么重要**：这是 Memory 系统最基础的工程实现，面试官最先问的就是这个流程，答不清楚直接暴露没做过。

**详细解释**：
1. 用户说"我对花生过敏" → 抽取为 `Allergy: Peanut` → 写入长期存储
2. 下次用户问推荐餐厅 → 检索到过敏记忆 → 插入上下文 Memory 区 → 模型先看约束再生成答案

**实际应用场景**：个性化推荐、用户偏好记忆、跨会话的上下文延续。

---

#### 记忆冲突处理策略

**是什么**：当用户前后提供了矛盾的信息时，系统如何决定哪条记忆生效的策略设计。

**为什么重要**：冲突处理是 Memory 系统的核心难点，答"新的覆盖旧的"太简单，面试官会追问可信度和高风险场景。

**详细解释**：

| 场景 | 处理方式 |
|------|----------|
| 用户明确说"我现在不对花生过敏了" | 新记忆覆盖，旧记忆标为失效 |
| 用户前后说法不一致，无明确纠正 | 保留历史版本，按可信度+时间+来源判断生效 |
| 高风险信息（过敏、医疗等） | 关键场景追问确认，不自动覆盖 |

**实际应用场景**：医疗健康类 Agent、个性化服务类 Agent 中，错误的记忆冲突处理可能导致严重后果。

---

#### 上下文结构设计

**是什么**：定义上下文各部分内容的排列顺序和边界，让模型能清晰区分规则、记忆、工具结果和当前对话。

**为什么重要**：上下文结构混乱会导致模型混淆来源，产生错误推理。结构清晰是 Agent 稳定运行的基础。

**详细解释**：
- 高优先级规则靠前（System Prompt、Developer 指令）
- Memory 单独成块，标明来源、时间、约束强度，不混入历史对话
- 当前用户问题靠后
- Tool Result 和记忆有清晰标注

**实际应用场景**：复杂多轮对话 Agent 的上下文管理，避免模型"忘记"关键约束或混淆不同来源的信息。

---

#### 分层压缩与工具状态管理

**是什么**：在上下文超长时，通过分层策略压缩历史信息，同时用任务状态区记录工具调用历史，防止重复调用。

**为什么重要**：简单截断会丢失关键决策链，而工具调用历史丢失会导致 Agent 反复调用同一工具，浪费算力。

**详细解释**：
- 近期对话：原文保留
- 较早对话：压成任务摘要
- 工具结果：只保留结论、参数、关键证据
- 工具调用历史：写入任务状态区（记录"查过什么、参数、结果、是否有效"），模型重复调用前先检查状态

**实际应用场景**：长任务 Agent、多步骤工具调用 Agent 的上下文管理。

### 2.4 速查表

| 面试题 | 核心答案要点 |
|--------|-------------|
| Memory 如何做？ | 抽取→结构化→写入→检索→注入上下文 Memory 区 |
| 上下文长度问题？ | 过滤+向量召回+重要性排序，预算分配不挤占工具和当前问题 |
| 结构化 vs 文本记忆？ | 稳定字段用结构化，长描述用文本；用事实 ID 去重归并 |
| 记忆冲突？ | 保留历史版本，按可信度/时间/来源判断；高风险场景追问确认 |
| 上下文结构？ | 规则靠前，Memory 单独成块，当前问题靠后，来源清晰标注 |
| 上下文压缩？ | 分层压缩：近期原文→早期摘要→工具结论；工具历史写任务状态区 |

### 2.5 延伸思考

1. 视频提到"高风险信息在关键场景追问确认"——如何定义"高风险"和"关键场景"的边界？是否应该让用户自己标注哪些记忆是高风险的？

2. 分层压缩中"较早对话压成任务摘要"——摘要本身是由 LLM 生成的，如果摘要过程中丢失了关键细节，系统应该如何检测和补救？

3. 工具状态区记录"是否还有效"——数据有效性的判断标准是什么？时间戳过期？用户主动刷新？还是需要 Agent 自己判断？

---

### 2.6 一句话记住它

> Memory 面试不是背概念，是讲清楚"怎么存、怎么取、冲突怎么判、超长怎么压"——每一步都有工程细节，答虚了就凉了。
