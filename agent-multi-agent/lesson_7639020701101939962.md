> 适合人群：对 AI 概念感到混乱、想建立系统认知框架的普通用户和 AI 入门者

## 一、视频原文

哈喽各位，相信 MCP、Agent、RAG、Claude Code、OpenClaw 这些概念本身就不难，但放在一起都懵了，原因只有一个，就是你没有去建立好你自己的一个概念坐标系——它是在哪一层，以及跟其他概念什么关系。一个人会懵掉。

所以今天我想用一家公司的结构来类比，给你理清楚。我给它分成了五层。

**第一层：大脑**

公司要运转得先有人，这个人或者说这个大脑，就是 LLM，也就是大语言模型。像我们说的 GPT、Claude 包括 Gemini，本质上都是同一个东西，叫做超级预测机器。你给它文字，它预测下一个词是什么，听起来很简单，但这个能力到了极致，能帮你写代码、做分析、做翻译、做聊天。

理解 LLM 有几个词绕不开：
- **Token**：大语言模型处理信息的最小单元，中文名叫词元。你发的每句话它都会先拆成 Token 来处理，Token 越多越贵，推理速度也越慢。
- **Context Window（上下文窗口）**：它的工作记忆。为什么有时候会忘掉前面说的？因为超出了记忆上限。窗口越大记得越多。
- **多模态（Multi-modal）**：指它不只能读文字，还能看图、听声音。发张图让 AI 分析，用的就是这个能力。
- **推理模型**：近几年出现的特殊分类，代表有 o3、DeepSeek R1。普通模型直接给答案，推理模型会先打草稿反复推敲，更擅长数学和复杂逻辑。

**第二层：沟通方式**

有了大脑，下一个问题是你用什么方式跟它沟通。有两种：
- **GUI**：图形界面，像网页、App，用鼠标点点点，比如 GPT 的网页版、Claude.ai 都是。
- **CLI**：命令行，那个黑色的框，需要打字发命令。

这也解释了很多人的困惑：Claude 和 Claude Code 是什么关系？本质上都是同一个大脑、同一个 Claude 模型，入口不同。Claude.ai 是网页版，Claude Code 是终端命令行。

沟通方式决定了你能用它做什么，但说不说得清楚同样重要。怎么保证说清楚？这就要提 **Prompt（提示词）** 和 **Prompt Engineering（提示词工程）**，研究的是怎么说话才能让模型干好活。

**第三层：工具和手册**

光会沟通还不够，模型得有称手的工具，还得有操作手册，否则每次都要重新摸索。

- **RAG（检索增强生成）**：大语言模型的知识是训练时固化的，它不知道最新的新闻。RAG 就是让模型在回答之前先去数据库查这个资料，查完再给你答案。你用 AI 问公司内部文档、查最新新闻，背后都是 RAG。
- **MCP（Model Context Protocol）**：工具接入的统一标准。以前每接一个外部工具都要单独对接，MCP 相当于统一了一个插头规格，像 USB-C 统一数据线。只要符合 MCP 协议，模型就能直接用任何工具。
- **Skills（技能/手册）**：特定任务的追加做法，把提示词写好，AI 直接调用，不用每次重新摸索。比如写短视频脚本的手册，格式、风格、注意事项都写在里面。
- **Hermes（框架）**：管理这些手册的系统。Skills 是一本本手册，Hermes 就是图书馆管理员，决定什么时候该调用哪一本。本质上是一个 Agent 的运行框架。

**第四层：自主干活的员工（Agent）**

能干活还不够，我们需要的是自主干活。这就是 Agent，也叫智能体。普通员工你得一步步指挥，Agent 不一样——你告诉它"帮我做一份竞品分析"，它自己去搜资料、整数据、写出稿，全程不用你盯着，发现问题自己修正，完了再交给你。

代表：
- **Claude Code**：Anthropic 推出的编程 Agent，跑在终端里，能读写代码、执行命令、自主调试。
- **Codex**：OpenAI 在 2025 年推出的编程 Agent，逻辑类似。
- **OpenClaw（小龙虾）**：今年最火的开源 Agent，短短几个月 GitHub Star 长到 25 万，普通人可以用它自动清理收件箱、整理日历、规划三餐，还可以用微信发条消息，它在后台把事情办好。
- **Hermes**：也是一个开源 Agent，最大特点是有学习循环，从每次使用中积累经验，自动更新记忆，用得越久越聪明。

Agent 的公式：**Agent = LLM + 工具 + 自主循环**，不是一个新的模型，而是一种组合方式。

**第五层：办公室（IDE / 工作环境）**

员工最终得在一个地方办公。大模型的办公室就是 IDE。像 VS Code 是 IDE，Cursor 也是 IDE，很多 IDE 都加上了 AI 功能，在 IDE 里用自然语言指挥 AI 改代码，不用切换工具，可以换很多模型：Claude、GPT、DeepSeek、Kimi 都可以换。

IDE 是办公室，Claude 或其他模型是员工，两个维度的东西，不在同一层。国内的 Alibaba 的 CodeWork、腾讯的 Maris、MiniMax 各种 Agent，这些都是办公室里的，你可以有很多大语言模型的员工。

还有一个概念叫 **Vibe Coding**：不用写代码，只要用自然语言告诉 AI 你要什么，AI 全程生成、修改、调试，人负责提需求和审核，AI 负责执行。这不是把 AI 当工具了，而是一种做事的新方式。

所以 AI 工具会越来越多，新名词也会越来越多，但你只要理解这五层——大脑、沟通方式、工具手册、自主决策、办公室——你就能有自己的 AI 概念坐标系，每次看到新概念，先想想它属于哪一层。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**用"公司结构"类比把 LLM、MCP、RAG、Agent、IDE 等 AI 概念分成五层，帮你建立系统的 AI 概念坐标系，再也不会被新名词搞懵。**

### 2.3 核心概念精讲

#### LLM（大语言模型）

**是什么**：超级预测机器，接收文字输入，预测下一个词，能力到了极致可以写代码、做分析、做翻译。

**为什么重要**：这是所有 AI 能力的地基，后面所有层都建立在这一层上面。

**详细解释**：

| 子概念 | 含义 |
|--------|------|
| Token | 模型处理信息的最小单元，Token 越多越贵越慢 |
| Context Window | 工作记忆，超出上限就会"忘记"前面的内容 |
| 多模态 | 能处理文字、图片、声音 |
| 推理模型 | 先打草稿推敲再给答案，擅长数学和复杂逻辑 |

**实际应用场景**：选模型时要考虑 Token 成本、Context Window 大小、是否需要多模态或推理能力。

---

#### MCP（Model Context Protocol）

**是什么**：工具接入的统一标准协议，像 USB-C 统一数据线，让模型能直接使用任何符合协议的外部工具。

**为什么重要**：解决了以前每接一个工具都要单独对接的问题，大幅降低工具集成成本。

**详细解释**：
- 以前：每个工具各自对接，重复开发，接口不统一
- MCP 之后：只要工具符合 MCP 协议，模型就能直接调用，生态快速扩展

**实际应用场景**：Agent 调用外部工具（搜索、数据库、日历、邮件等）时，通过 MCP 统一接入。

---

#### Agent（智能体）

**是什么**：LLM + 工具 + 自主循环的组合，能自主规划、执行、纠错，完成复杂任务，无需人工逐步指挥。

**为什么重要**：Agent 是 AI 从"工具"升级为"员工"的关键，代表了 AI 应用的最高形态。

**详细解释**：
- 普通 LLM：你问一句，它答一句
- Agent：你说目标，它自主拆解任务、调用工具、执行、纠错、交付结果

**实际应用场景**：编程 Agent（Claude Code、Codex）、生活助手 Agent（OpenClaw）、有学习能力的 Agent（Hermes）。

### 2.4 速查表

| 层级 | 代表概念 | 类比 |
|------|----------|------|
| 第一层：大脑 | LLM（GPT、Claude、Gemini） | 公司员工本人 |
| 第二层：沟通方式 | GUI、CLI、Prompt | 说话方式 |
| 第三层：工具手册 | RAG、MCP、Skills、Hermes | 工具箱 + 操作手册 |
| 第四层：自主干活 | Agent（OpenClaw、Claude Code） | 能自主完成任务的员工 |
| 第五层：办公室 | IDE（Cursor、VS Code）、Vibe Coding | 办公室 |

### 2.5 延伸思考

1. 视频说"每次看到新概念先想想它属于哪一层"——Sora（视频生成）、Midjourney（图片生成）这类生成式 AI 工具，应该归入五层框架的哪一层？它们和 LLM 的关系是什么？

2. 视频提到 Hermes 有"学习循环，从每次使用中积累经验"——这和传统 LLM 的微调有什么本质区别？学习循环在工程上是如何实现的？

3. Vibe Coding 被视频定义为"新的工作方式"而不是工具——这个定性意味着什么？它对传统软件开发流程（需求→设计→编码→测试）的冲击体现在哪些环节？

---

### 2.6 一句话记住它

> AI 概念不乱，是因为你有坐标系：大脑（LLM）→ 沟通方式 → 工具手册 → 自主干活（Agent）→ 办公室，新名词出来先找它在哪一层。
