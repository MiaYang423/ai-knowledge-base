## 一、视频原文

如果你经历过去年内部的 DeepSeek 热潮，也被今年铺天盖地的大模型洗脑过，那你一定会和我有同样的疑惑：就是怎么才短短一年的时间，AI 就从一个全能配料变成了能自动干活的数字员工？我这才刚刚弄明白大模型是什么，怎么又冒出来一个智能体？没关系，今天这期视频我就跟大家好好聊聊 AI 到底是怎么帮人干活的。

智能体，也就是常说的 Agent，它和豆包这种 AI 聊天工具最大的区别就是，它能够自主拆解各种任务并自动执行，直到取得具体的结果。我在之前的视频里仔细讲过，大家可以去看一看。

想让 Agent 顺畅工作，其核心还是要依赖大模型。而目前市面上主流的 Agent 工具都是自带大模型的。比如公认世界最强的 Claude Code 代理，就是自家的 Claude Sonnet 模型。虽然它的模型也是世界顶线水平，但价格也同样顶线。像我这种用不起它们家模型，但是有缘分用上 Claude Code 这个 Agent 的，怎么办呢？最一般的方法就是选择国产模型替代。那怎么样才能用上国产模型呢？这里又涉及到一个名叫 API 的技术。

API 全称应用程序接口，你可以把它理解为一条跨场景的模型直连通道。而连接这个通道的东西，是一串名叫 API Key 的代码。只要你购买了国产模型的 Coding Plan，拿到专属的 API Key，把它们填到特定的位置，就能够让 Agent 调用国内模型了。但是不同的工具的配置方法是不一样的，有的时候会提供一个设置面板，你填进去就完事了；而像 Claude Code 这种就需要你去修改它的设置文件。一般情况下，每一个国产模型的 Coding Plan 后台都会提供对应的文件修改方式。但这对新手小白来说还是有点困难，像我当初准备这个都花了小半天。不过现在 GitHub 上面有一个名叫 CC Switch 的项目，能够非常方便地进行 API Key 的供款和各种模型的切换。

那搞定了 Agent 和模型的匹配，是不是就能愉快地起飞了呢？也不是。因为你会发现几个非常现实的问题：比如你想让它连网搜索东西，它会告诉你没这个能力；截个图给它看吧，它能回出一堆驴头不对马嘴的内容；扔个链接给它分析，它也根本分析不了。如果想解决这个问题，就必须要到我们今天的第三个核心概念了——MCP。

MCP 全称是 Model Context Protocol，翻译过来就是模型上下文协议。大家千万别被"协议"这两个字吓到了，简单来说，它就像 AI 界共同打造的一个万能接口。如果不懂这个协议，你的 Agent 其实是个偏科生。比如你让它去扒一个网页，网页底层用的是复杂的 HTTP 代码，而 Agent 的脑子里只能处理纯文本字符。为了让它俩交流，以前必须针对这一个网页苦哈哈地手写一套定制化的数据转换代码，换一个网站还得重写一套。而 MCP 的出现直接颠覆了这种落后的模式。

当你把 MCP 配置到你的 Agent 头里之后，后台会发生这三个极其智能的步骤：第一步，MCP 工具会主动给你的 Agent 递上一份技能清单，比如清单上会写明"我们先掌握了三个技能：提取网页正文、调用联网搜索、读取数据库"。第二步，当你在聊天框下达了"帮我搜索一下最新新闻"的指令，Agent 会自动去查内奋清单，发现技能二正好对口，于是它会按照协议规定的统一格式把你的需求打成一个标准包，发送给 MCP 工具。第三步，MCP 工具拿到需求后，自己转身去外部网络里执行抓取动作，完成之后会把乱七八糟的网页代码全部翻译成大模型能看懂的结构化纯文本，直接塞进它的上下文里。

那 MCP 工具去哪里获取呢？GitHub 上面一搜一大把。

好，现在咱们搞定了大模型的接入，配置进去了外部工具，咱们 Agent 的底层架构就算彻底搭建完毕了。但架构再好，如果它每次干活就得像带新人一样，把复杂的业务逻辑从头到尾事无巨细地交代一遍，不仅你累得够呛，跑出来的 Token 账单也能让你肉疼。那怎么才能让它把这些复杂的流程记住，变成一个高效稳定的自动化功能？这就涉及到了我们今天的最后一个核心概念——SOP。

简单来说，SOP 就是针对高频复杂任务的标准作业流程。它的核心作用就是把你日常重复的、专业要求极高的工作流程封装成一个固定的模块。等你下次再需要处理同类任务时，Agent 就可以直接调用这个现成的 SOP 去自动执行，再也不需要你每次苦口婆心地输入几千字的提示词了。这不仅把自动化执行的效率拿到了极致，还能把你大模型的 Token 消耗成本很狠地降下来。

那关于 SOP 在实际工作中到底有多强悍，不同人应该怎么去玩，我之前专门做过一期视频，非常详细地讲过。为了不占用大家的时间，这里就不再展开了。强烈建议刚入门的小伙伴去翻一翻我之前的那期视频，绝对能把你打开高效工作流的新大门。

OK，以上就是本期视频的全部内容了。我是林营，咱们下期视频再续前缘。

### 2.1 这个视频在讲什么（一句话）

**从零搭建一个可用的 AI Agent，需要掌握四个核心概念：Agent 本体、API（模型接入）、MCP（工具扩展）、SOP（流程封装），四者缺一不可，共同构成 Agent 的完整底层架构。**

### 2.3 核心概念精讲

#### API（应用程序接口）

**是什么**：一条让 Agent 与大模型之间建立直连的通道，通过 API Key 进行身份验证和调用授权。

**为什么重要**：顶级 Agent 工具（如 Claude Code）自带的模型价格昂贵，通过 API 可以将 Agent 对接到性价比更高的国产模型，大幅降低使用成本。

**详细解释**：
- 购买国产模型的 Coding Plan → 获取 API Key
- 将 API Key 填入 Agent 的配置位置（设置面板或配置文件）
- 辅助工具：CC Switch（GitHub 开源项目，便捷管理多个 API Key 和模型切换）

**实际应用场景**：用国产大模型替代 Claude Sonnet，在保持 Agent 能力的同时大幅降低 Token 成本。

---

#### MCP（Model Context Protocol）

**是什么**：模型上下文协议，一个让 Agent 能够调用外部工具（网页抓取、联网搜索、数据库读取等）的标准化接口协议。

**为什么重要**：没有 MCP，Agent 只是个"偏科生"，无法联网、无法读取文件、无法与外部系统交互；有了 MCP，Agent 的能力边界几乎可以无限扩展。

**详细解释**：MCP 的三步工作流程：

| 步骤 | 发生了什么 |
|-----|---------|
| 第一步 | MCP 工具向 Agent 递交技能清单（我能做什么） |
| 第二步 | Agent 按协议格式将用户需求打包成标准请求发给 MCP |
| 第三步 | MCP 执行外部操作，将结果翻译成结构化文本返回给 Agent |

**实际应用场景**：让 Agent 能够联网搜索最新新闻、抓取网页正文、读取数据库数据。

---

#### SOP（标准作业流程）

**是什么**：将高频、复杂的工作流程封装成固定模块，让 Agent 在处理同类任务时可以直接调用，无需每次重新输入长提示词。

**为什么重要**：重复性工作每次都要输入大量提示词，不仅耗费精力，还会产生大量 Token 成本；SOP 将流程固化，实现真正的自动化。

**详细解释**：
- 把复杂的业务流程（如"每天早上整理行业报告"）封装成一个 SOP 模块
- 下次触发时，Agent 直接调用 SOP，无需重新描述流程
- 效果：自动化效率最大化 + Token 成本显著降低

**实际应用场景**：日报生成、竞品分析、数据整理等高频重复任务的自动化封装。

### 2.4 速查表

| 核心概念 | 解决的问题 | 类比 |
|---------|---------|-----|
| Agent | 从聊天到自主执行任务 | 从顾问到员工 |
| API | 接入性价比更高的大模型 | 换个更便宜的大脑 |
| MCP | 让 Agent 能使用外部工具 | 给员工配备工具箱 |
| SOP | 固化重复流程，降低成本 | 给员工一本操作手册 |

---

### 2.5 延伸思考

1. 视频中提到 API Key 的配置对新手有门槛，CC Switch 等工具降低了这个门槛——但 API Key 泄露的安全风险如何防范？普通用户应该注意什么？

2. MCP 的"技能清单"机制意味着 Agent 需要先了解自己有哪些工具才能使用，当 MCP 工具数量很多时，Agent 如何高效选择正确的工具？会不会产生选择混乱？

3. SOP 封装了固定的工作流程，但业务场景会随时变化——如何设计 SOP 的版本管理和更新机制，让封装的流程保持有效性？

---

### 2.6 一句话记住它

> 搭建 Agent 就像装修一套房：Agent 是毛坯房，API 是接通水电，MCP 是安装各种家电，SOP 是制定家居使用规范——四步缺一，这房子都住不舒服。
