## 一、视频原文

2026年了，写代码的方式层出不穷，今天从好到差锐评一下主流的开发方式。

首先出场的是我们的老大哥——纯手写代码。这种编程方式使用传统写代码工具，开发者纯手写代码进行构建程序。因为模型能力逐渐变强，在2024、2025年还有大佬说自己坚持不使用AI，但如今已经成为过去，只有密码学和密码领域的大佬才会使用，出于情怀，给到NPC。

其次是网页对话式编程，通过网页端对话让AI生成代码。因为AI无法感知到本地环境，只能靠用户用语言描述或截图给AI，所以很多时候无法快速定位上下文内容，需要手动复制，也是个问题。写扫雷、贪吃蛇什么的程序还好，但无法应用于实际生产，直接给到垃圾。

接下来是第三种——IDE 集成编程，比如市面上比较火的 Cursor，也可以使用 VS Code 的插件来实现。IDE 集成编程能读取本地环境，AI 自己跑命令验证代码有没有问题，还有很多便捷的功能。因为能看到项目文件，是大部分老程序员会使用的方式，给到顶级。

接下来是命令行编程，主流的有 Claude Code、Cursor 等。该方式凭借大模型强大的能力，不注重代码而注重想法，想要什么功能自己描述一下就好。但因为无法看到项目文件，失去了对代码的掌控感，所以是没有代码基础的人用得多，拿来写大模型和中小项目都合适。擅长使用的人能做得很好，不擅长使用的人一个功能可能需要多次修改，也是给到顶级。

接下来看手机编程，通过手机下达命令，让电脑的 Claude Code、Codex 这类命令行工具写代码。适合灵感突发，就拿出手机让AI去做，无论你在健身房、在走路、在吃饭，都可以拿出手机让AI进行编程，完成你的想法。但手机屏幕小、信息少，请先用来做一些小 demo 验证灵感，因此给到一般。

接下来要出场的是命令行自动编程，给AI装很多 MCP、Cursor 让AI能自己打开浏览器、程序，自己截图找问题。你写好详细的计划让AI自己执行，但这只适合编程大佬，新人提出来的计划可能需求是相冲的，导致AI难以实现。但可以预料到这是未来的编码方式，因此给到一般。

最后一种是多 Agent 编程，比较流行的概念。使用一个命令行进行写代码时，你需要花几分钟等待模型的输出；而多 Agent 没有这个问题，你有了想法就可以快速验证，几个 Agent 写代码，几个 Agent 进行测试，几个 Agent 进行合并冲突，是目前最新潮的编码方式，也是效率最高的，直接给行。

### 2.3 核心概念精讲

#### 概念一：IDE 集成编程

**是什么**：在本地开发环境（如 Cursor、VS Code）中直接嵌入 AI 能力，让 AI 读取项目文件并辅助写代码。

**为什么重要**：AI 能"看到"你的整个项目，理解上下文，自动运行命令验证代码是否正确，极大减少手动操作。

**详细解释**：

| 特性 | 说明 |
|------|------|
| 本地环境感知 | AI 可读取项目文件结构，无需手动粘贴代码 |
| 命令自动执行 | AI 可自行跑测试、编译命令，验证结果 |
| 代表工具 | Cursor、VS Code + AI 插件 |
| 主要用户 | 有一定代码基础的老程序员 |

**实际应用场景**：在已有项目中添加新功能、排查 bug、重构代码，适合日常生产开发。

---

#### 概念二：命令行编程

**是什么**：在终端通过自然语言描述需求，由 AI（如 Claude Code）直接生成并执行代码，开发者不需要深度介入代码细节。

**为什么重要**：降低了编程门槛，让没有深厚代码基础的人也能构建中小型项目，同时让有经验的人专注于想法而非实现。

**详细解释**：

- 核心理念：**注重想法，不注重代码**
- 优点：上手快，描述需求即可
- 缺点：无法直观看到项目文件，对代码掌控感弱
- 效果差异大：擅长者事半功倍，不擅长者反复修改

**实际应用场景**：开发大模型相关工具、中小型项目的快速原型搭建。

---

#### 概念三：命令行自动编程（MCP 自动化）

**是什么**：为 AI 配置大量 MCP（Model Context Protocol）工具，使其能自主打开浏览器、截图、定位问题，按照预先写好的计划自动完成开发任务。

**为什么重要**：代表了 AI 编程从"辅助"走向"自主执行"的趋势，是未来编码方式的雏形。

**详细解释**：

| 维度 | 内容 |
|------|------|
| 核心能力 | AI 自主操作浏览器、截图、调试 |
| 前提条件 | 需要写好详细、无冲突的计划文档 |
| 适合人群 | 有经验的编程大佬 |
| 新人风险 | 需求描述不清或相互冲突，AI 难以推进 |
| 未来潜力 | 视频认为这是未来主流方向 |

**实际应用场景**：复杂项目的自动化开发流程，适合能写清晰技术规格文档的资深开发者。

---

#### 概念四：多 Agent 编程

**是什么**：同时启动多个 AI Agent，分别承担写代码、测试、合并冲突等不同角色，并行推进开发任务。

**为什么重要**：解决了单 Agent 串行等待慢的问题，是目前效率最高的 AI 编程方式。

**详细解释**：

| 单 Agent 模式 | 多 Agent 模式 |
|--------------|--------------|
| 串行执行，需等待输出 | 多个 Agent 并行工作 |
| 写完代码才能测试 | 写代码、测试、合并同步进行 |
| 等待时间以分钟计 | 快速验证想法 |
| 当前主流 | 当前最新潮、效率最高 |

**实际应用场景**：需要快速迭代的项目，一个 Agent 写功能，一个 Agent 跑测试，一个 Agent 处理代码合并冲突。

### 2.5 延伸思考

1. **IDE 集成编程和命令行编程都被评为"顶级"，但适合人群不同——对于一个从零开始学编程的人，应该直接跳过 IDE 集成，从命令行编程入手吗？这样做有什么潜在风险？**

2. **命令行自动编程（MCP 自动化）被认为是"未来的编码方式"，但目前只适合大佬，瓶颈在于"写好详细计划"的能力——这说明未来程序员的核心竞争力会从"写代码"转变成什么？**

3. **多 Agent 编程效率最高，但多个 Agent 并行工作时如何保证代码风格一致、逻辑不冲突？这个问题目前是否有成熟的解决方案？**
