## 一、视频原文

换工具不换脑。最近有人在评论区问我，到底是用 Codex 还是用 Claude Code？我以前也会特别纠结这一类问题——哪个工具更新了我就想试一下，哪个模型变强了我就怕自己错过，别人说某个工具效率高，我就开始怀疑自己是不是又落后了。

那我现在想明白了一件事情。很多人问的其实不是工具的问题，你问 Codex 可不可以、Claude Code 到底行不行，这些问题都属于同一类问题——属于"手"的问题。

我现在是把自己的工作分成三块：一个叫做硬盘，一个叫做大脑，第三个叫做手。

Obsidian 是我的硬盘，它存的是我的判断、我踩过的坑、我做过的项目、我写过的稿子、梳理过的问题。Claude Code 或者其他的大模型，属于大脑的层面，它负责理解、推理、生成，把这些内容重新组织出来。Claude Code 或者 Codex 这些属于工具，当然最近会有越来越多的工具出来，它们属于"手"，手负责执行——帮你写代码、帮你改文件、帮你生成内容、帮你跑任务。

所以你会发现，手是可以换的。今天我用 Claude Code，明天我可以用 Codex，只要我的硬盘在、我的判断在、我的上下文还在，换一个工具其实并不会从零开始。

我今天就演示一下，这是 Claude Code 我们之前的对话，我用 Codex 重新帮我整理梳理了相关的内容，它还帮我增加了注音、帮我做整体的更新，这对我来说其实是很直接的一个调用。

这也是为什么我觉得真正有复利的不是工具。工具本身没有复利——你今天花两周学会了一个工具，过几个月又会有新的工具出来，但这两周你沉淀下来的工作方法、判断标准、踩坑记录，这些是你可以直接带走的。所以我从 Claude Code 切换到 Codex，上手是非常快的。

当然未来会有各种各样的软件，你需要的就是在你的系统里配置好相关的信息，而不是每来一个新工具你又从零开始。

所以我并不会一直问哪个工具最强，我会先问：我的硬盘里面有没有东西？如果你的 Obsidian 里面什么都没有，其实差距并不会很大，因为基本上都是从零开始。但如果你的硬盘里面已经保存了你的判断、你的经验、你的上下文，那工具换来换去，也只是换一个手，真正不变的是你的审美、你的系统。

所以我的建议是，花两分钟，每天把和 AI 沟通的事情记录下来，不用写得很漂亮，就记一下做了什么、干了什么事情。有一个机内记录的过程，慢慢就能让 AI 读懂你是什么样的风格。两个月之后，你积累的这些品味、收藏和风格，在你将来的 AI 使用中就会用得上。这才是一个工作流应该有的状态。

所以无论工具怎么变，核心还是你的审美和你的大脑，你的硬盘存在哪里。

### 2.3 核心概念精讲

#### 概念一：硬盘—大脑—手 三层框架

**是什么**：把 AI 工作流拆解为三个角色——Obsidian 作为个人知识硬盘，大模型作为推理大脑，执行工具（Claude Code / Codex 等）作为执行的手。

**为什么重要**：
大多数人纠结"换不换工具"，是因为没有区分这三层的职责。一旦分清楚，就会发现"手"是最不重要的那一层，可以随时替换。

**详细解释**：

| 层级 | 对应工具 | 职责 | 可替换性 |
|------|----------|------|----------|
| 硬盘 | Obsidian | 存储判断、经验、上下文、踩坑记录 | ❌ 不换，是核心资产 |
| 大脑 | Claude 等大模型 | 理解、推理、重新组织内容 | ⚠️ 可换，但有切换成本 |
| 手 | Claude Code / Codex 等 | 写代码、改文件、跑任务、执行 | ✅ 随时可换 |

**实际应用场景**：
视频中演示了把 Claude Code 里的对话记录迁移到 Codex，由于 Obsidian 里已有完整上下文，Codex 可以直接接手整理内容、增加注音、做整体更新，几乎没有重新上手的成本。

---

#### 概念二：工具没有复利，上下文才有复利

**是什么**：花时间学一个工具，工具本身不会随时间增值；但花时间积累的工作方法、判断标准、踩坑记录，会随时间持续发挥价值。

**为什么重要**：
AI 工具迭代极快，几个月就会出现新工具。如果把精力全押在"学工具"上，每次迭代都要从零开始；如果把精力押在"建硬盘"上，换工具只是换一个手。

**详细解释**：
- 学会一个工具 ≠ 沉淀可迁移的能力
- 踩坑记录、判断标准、个人风格 = 可以带走的资产
- Obsidian 硬盘越丰富，新工具的上手速度越快，因为 AI 已经"认识你"

**实际应用场景**：
作者从 Claude Code 切换到 Codex，因为 Obsidian 里已有积累，上手非常快，不需要重新向工具解释自己的背景和偏好。

---

#### 概念三：每日两分钟记录习惯

**是什么**：每天花两分钟，把和 AI 沟通做了什么事情简单记录下来，不需要写得漂亮，只要留下痕迹。

**为什么重要**：
这是建立"硬盘"的最低成本方式。两个月的积累，会让 AI 逐渐读懂你的风格、品味和偏好，形成真正个性化的工作流。

**详细解释**：
- 不要求格式，只要记录"做了什么、干了什么"
- 目标是让 AI 能读懂你的风格
- 时间成本极低，但复利效应在两个月后开始显现

**实际应用场景**：
把每次和 AI 协作的结果、调整过程、踩过的坑，简单记在 Obsidian 里，形成个人的"AI 使用日志"，这些内容未来可以作为上下文喂给任何新工具。

### 2.5 延伸思考

1. **"硬盘"的质量如何衡量？** 记录在 Obsidian 里的内容，什么样的才算真正有价值的上下文，什么样的只是噪音？有没有办法在记录时就做好筛选？

2. **三层框架是否适用于非编程场景？** 如果你是设计师、写作者或产品经理，"手"对应的工具是什么？"硬盘"里应该存什么类型的内容？

3. **当大模型本身也在快速迭代时，"大脑"层是否也面临同样的替换焦虑？** 如何判断什么时候值得切换大模型，而不只是切换执行工具？
