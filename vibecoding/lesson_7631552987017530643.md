## 一、视频原文

如果我重新来过，重新开始学AI，那么我一定会做这三件事。其实之前我在星球内都有反复说过，那本期视频我来完整表述一遍。

**第一，使用Claude Code。即使没用两天就被封号，也要用。**

今天我们已经进入到Agentic AI阶段，那在这个阶段，最先进最有代表性的工具就是Claude Code。它跟之前的一些聊天机器人是完全不一样的，完全是天壤之别。聊天机器人输出的是文字和图片，这个很好理解。而Claude Code这种工具，通过代码创造更多的东西，通过命令去控制更多的东西。那这个，光看别人的描述，或者你看了几个视频，那是根本想象不出来的。所以如果我今天要重新来过的话，那第一件事一定是亲自上手使用Claude Code，只有上手了才有真正的体感。

AI正在飞速发展，这话听起来确实特别烂大街，但是我真的找不出比飞速发展更准确的词来描述了。在这种情况下，一定不能做井底之蛙，比如觉得AI不过如此。一定要亲自探出脑袋去看一看，亲眼看了才知道天究竟有多大。这个就是我前面说即使被封号了也要用的原因。不管之后你是否还继续使用Claude Code，反正在最初是一定要体验，一定要摸一摸的。

多说两句，之前我出视频介绍过Google Stitch，当时我就判断接下来要失业的很可能是设计师。很多人不服气，觉得我就是在传播焦虑。结果前两天Claude Code站出来了。你以为设计就是天马行空，就是所谓的感觉？你就看着吧，从最标准化的工作开始，AI会一步一步吃掉设计师的份额。那扯远了，咱们继续下一条。

**第二，彻底解决基础设施的问题。**

最先进的AI在国外，国内大概落后一年，这个就是事实，跟爱不爱国、是否支持国产AI没有关系。那由于众所周知的原因，要稳定使用国外的AI，真的需要一套基础设施。这个也是我们在星球里老生常谈的话题，我不断地被问到，我也不断地强调。所以如果我今天重新来过，那我一定会专门花时间去优先解决，否则刚才说的Claude Code、Gemini AI我也用不了。

比如支付的问题，最简单的做法是办一张国内的Visa或者Mastercard信用卡。说是国内的卡，但还是挺有用的。比如我一直觉得，用招商的Mastercard订阅Gemini，用建行的Visa订阅Copilot。至于那些没法用国内卡订阅的AI，还可以通过美区iOS账户加礼品卡的方式来完成支付。那更进一步，还可以办一张Capital One的实体信用卡，这个是国外的卡，可能是因为我申请了ITIN，所以办这些都非常顺利。

说实话，我看到最多的情况就是因为基础设施的原因导致没法使用最先进的AI。不知道是懒还是什么原因，反正就是不愿意彻底解决，然后一次又一次又一次地被卡住。我真的都无法理解了。

**第三，资料Markdown化，能力Skill化。**

所有人都在说要学AI，那么要学到什么程度呢？你总得有一个衡量标准吧。我之前出过关于学习AI的教程，之后又在星球内特意发了这么一个帖子，就是关于标准的。我认为在现阶段，衡量标准就有一个：个体和AI能否成为认知共同体。

一方面，个体清楚知道AI的边界，这个边界包括AI能力和支持的界限、不同模型工具的差异等等。另一方面，AI能够基于对个体的了解进行动态的预判。那要做到这一点，你得让AI了解你。

所以如果我今天重新来过，那我一定会把我所有沉淀下来的资料都变成Markdown的格式，因为这个格式AI能读，咱们人类也能看得懂。至于什么Word、PDF、PPT，我肯定是不会再搞了，那些都太复杂了，AI看不懂。既然AI看不懂，那这些格式一定会越来越被边缘化。

另外，我还会把我所有的能力都Skill化。说实话，这个过程挺有难度的。能把脑子里的东西变成Skill，变成明明白白的SOP，变成AI也能理解的文字，这其实就是一个自我提升的过程。这也是我今年一直在星球内建议大家做三个Skill的原因，不仅可以赋能自己的AI系统，还能充分梳理自己的工作流。

以上这三条建议，都是我过去这两三年的经验总结，也是我在星球内反复强调的。至于具体怎么做，你可以去搜索，可以去问AI，别来问我。如果你搞不定的话，那就别想着学AI了。

OK，以上就是本期内容。想了解AI，想成为超级个体，想找到志同道合的人，就来我们Newtype社群。那咱们下期见。

### 2.3 核心概念精讲

#### 概念一：Agentic AI 与 Claude Code

**是什么**：Agentic AI 是指能够通过代码和命令主动创造、控制事物的AI形态，Claude Code 是作者认为当前阶段最具代表性的工具。

**为什么重要**：
它和聊天机器人有本质区别，仅靠看视频或听描述根本无法建立真实认知，必须亲自上手才能理解其能力边界。

**详细解释**：

| 维度 | 聊天机器人（如早期 ChatGPT） | Agentic AI（如 Claude Code） |
|------|----------------------------|-----------------------------|
| 输出形式 | 文字、图片 | 代码、可执行操作、控制指令 |
| 交互方式 | 对话式问答 | 命令式任务执行 |
| 能力边界 | 生成内容 | 创造和控制更多事物 |
| 体感获取方式 | 看描述可理解 | 必须亲自上手 |

**实际应用场景**：
作者举例，Google Stitch 和 Claude Code 已经开始进入设计领域，从最标准化的设计工作开始替代人工。作者认为这一趋势会持续扩展。

---

#### 概念二：学AI的基础设施

**是什么**：指稳定使用国外先进AI工具所需的一整套支付、账户、网络等配套条件。

**为什么重要**：
基础设施问题是作者观察到阻止大多数人真正用上先进AI的最主要障碍，不解决这个问题，其他学习计划都是空谈。

**详细解释**：

| 层级 | 方案 | 适用场景 |
|------|------|---------|
| 初级 | 国内 Visa / Mastercard（如招商、建行） | 订阅 Gemini、Copilot 等 |
| 中级 | 美区 iOS 账户 + 礼品卡 | 无法用国内卡订阅的 AI 服务 |
| 进阶 | Capital One 实体信用卡（需 ITIN） | 更广泛的国外服务订阅 |

**实际应用场景**：
作者自己用招商 Mastercard 订阅 Gemini，用建行 Visa 订阅 Copilot，并申请了 ITIN 以便办理国外信用卡。

---

#### 概念三：认知共同体——学AI的衡量标准

**是什么**：作者提出的判断AI学习是否到位的标准——个体与AI能否形成"认知共同体"，即双方能够相互理解、协同工作。

**为什么重要**：
"要学AI"这个目标太模糊，没有衡量标准就不知道学到哪里算够。这个标准给出了一个具体的终点。

**详细解释**：

认知共同体需要满足两个条件：

- **个体侧**：清楚知道AI的边界，包括不同模型的能力差异、各工具的适用范围和限制
- **AI侧**：AI能基于对个体的充分了解，进行动态预判，而不是每次都从零开始

要让AI了解你，前提是你得把自己的信息、资料、能力整理成AI能读懂的形式。

**实际应用场景**：
这是作者建议做"资料Markdown化"和"能力Skill化"的根本原因——只有把自己结构化，AI才能真正了解你，双方才能形成认知共同体。

---

#### 概念四：资料 Markdown 化与能力 Skill 化

**是什么**：将个人积累的资料转为 Markdown 格式，将个人能力整理为明确的 SOP（标准操作流程），使AI能够读取和理解。

**为什么重要**：
AI无法有效解析 Word、PDF、PPT 等复杂格式，这些格式会越来越被边缘化。只有结构化的信息，才能真正赋能AI系统。

**详细解释**：

| 类型 | 做什么 | 为什么 | 难度 |
|------|--------|--------|------|
| 资料 Markdown 化 | 把存量资料转为 .md 格式 | AI 可读，人类也可读 | 相对容易 |
| 能力 Skill 化 | 把脑子里的经验写成明确 SOP | 让 AI 理解并复现你的工作方式 | 有一定难度，但本身是自我提升 |

**实际应用场景**：
作者在自己的社群中建议成员今年完成三个 Skill 的整理，目的是既赋能自己的AI系统，又借此机会梳理和优化自身工作流。

### 2.5 延伸思考

1. **"认知共同体"的标准是否适用于所有人？** 作者提出个体和AI相互了解、能够动态预判，但不同职业、不同使用深度的人，达到这个标准所需的投入差异巨大——对于普通用户来说，这个标准是否过高，还是说本身就应该有门槛？

2. **基础设施问题是一次性解决还是持续挑战？** 作者建议优先彻底解决基础设施，但国外AI服务的支付规则、封号风险、政策变化都在持续变动——"彻底解决"究竟意味着什么，这套方案的稳定性有多高？

3. **能力 Skill 化的过程本身就是学习——那AI在这个过程中扮演什么角色？** 作者说把经验写成SOP是"自我提升的过程"，那么在整理Skill的过程中，是否可以反过来让AI参与提问、追问、帮助结构化？这个双向过程会带来什么不同的效果？

### 2.6 一句话记住它
