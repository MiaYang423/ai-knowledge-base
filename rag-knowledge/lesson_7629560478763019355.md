## 一、视频原文

这几分钟十分钟教你搭建自己的个人知识库。前段时间OpenAI的创始人Karpathy提出了一个个人知识库的理念，这个观点在推特上非常火，获得了1900多万的浏览量，各大科技博主也都纷纷下场，网上的教程铺天盖地，但是我完全看不明白他们为什么要弄得那么复杂。今天也带大家在十分钟之内完成自己的个人知识库。

首先大家了解，传统的知识库往往是类似于Notion这样的树状结构，做知识的梳理和整理。这样的知识库不合理的地方在于，你必须要通过手动对所有的内容进行分类、注释、整理，这种分类的工作量是非常大的。记住，AI时代任何一种不直接产生工作价值的事都应该把它外包化——比如扫地交给扫地机器人，洗衣服交给洗衣机，烘干交给烘干机，对不对？

另外，这种分类方式一篇内容只放在一个分类里面，但一篇内容往往有多个维度。比如有的内容既涉及技术与数据分析，又涉及上班经验，还涉及心理学，还涉及超级个体，那把它放在哪里？而当你积累的数据量过大的时候，你往往很难找到所有对你有价值的内容，检索的成本非常高。

而Karpathy的知识库最大的便利就在于，所有你收集到的内容完全不用你自己整理。你只需要把你收集到的所有信息放到一个文件夹里，接下来让模型给你整理好就可以了。最后怎么在十分钟之内完成搭建呢？你在Obsidian里面创建一个叫做"原始数据"的文件夹，把你所有的资料放进去，然后到Claude Code的界面里面去告诉它：按照Karpathy的知识库帮我整理我所有的Obsidian的文件。接下来吃点零食等它执行完成就好了。

那你可以看一下我输入给Claude Code的信息，然后它的一些分析以及它输出的结果。我们来看一看，在Concept这个已经整理好的总结果的概念内容里面，这些内容其实就已经可以为我的创作提供灵感了。看看它整个的知识图谱是怎么样的，它把大语言模型、知识库这个概念给哪些内容进行了关联。大家可以看到，这也是为什么我们说用大语言模型搭建的知识图谱会比我们传统文件型的知识库更直观好用的原因。至于我这篇视频的口播文案，主要的框架也是由它输出的，可以看到在它的分析框架里面它采用了哪些知识点，然后它是怎么思考的，它写了哪些内容。

所以搭建个人知识库这件事情，第一，它没有网上博主所说的那么复杂，它只需要让Claude Code有操作Obsidian的权限就够了。这又怎么打通呢？你仍然可以问它说，我现在需要打通我的Obsidian，应该怎么操作，它会告诉你。

但到最后我们也不能过度吹捧，这个形式的知识库就是所有状态下的最优解吗？并不是。只有当你的文本量达到40万字以上的时候，你才会有比较好的体验。比如说你的文件夹里面只收藏了两三篇的内容，那你就不用考虑搭建知识库这件事情，没有意义。当你收集了足够多的内容的时候才有价值。

这也从侧面说明了，你从现在开始就应该要为你自己的基础信息和框架知识做整理、做收集。不一定现在就要把知识库完成，但你未来想要做哪些内容，你对哪些方向比较感兴趣——心理学、经济学、商业、个人成长——你现在就可以收集那些不会过时的经典的内容。

那么一年之后，别人的收藏夹变成了一片废墟，尘寂无人访问，而你的收藏夹在一年之后可以变成一座只属于你自己的图书馆，成为你的灵感来源、文案输出的好帮手。

在AI时代，大家在工具面前其实是平权的，但是能否给AI提供优质、明确的信息源，才是大家对AI使用结果产生差异的关键。所以请尽早开始积累自己的信息库。关注我，带你高效学习AI时代的前沿知识。

### 2.3 核心概念精讲

#### 概念一：Karpathy个人知识库理念

**是什么**：由OpenAI创始人Karpathy提出的知识管理方式——不手动分类，只管收集，交给AI模型自动整理。

**为什么重要**：
这个理念在推特上获得1900万浏览，触动了大量知识工作者的痛点：我们花了大量时间整理笔记，却几乎没时间真正使用它们。

**详细解释**：
Karpathy的核心逻辑是把"整理"这件不直接产生价值的事外包给AI，就像把扫地外包给扫地机器人、把洗衣外包给洗衣机一样。你只需要专注于"收集"这件事本身。

**实际应用场景**：
你平时收藏的文章、截图、笔记，全部丢进Obsidian的一个文件夹，不需要打标签、不需要分类，定期让Claude Code跑一次整理即可。

---

#### 概念二：传统树状知识库的结构性缺陷

**是什么**：以Notion为代表的树状分类结构，要求每条内容归属于唯一的一个类别。

**为什么重要**：
理解这个缺陷，才能理解为什么AI驱动的知识图谱是更优解。

**详细解释**：

| 问题维度 | 传统树状知识库 | AI知识图谱 |
|---|---|---|
| 分类方式 | 手动，唯一归属 | 自动，多维度关联 |
| 整理成本 | 高，需持续维护 | 低，模型自动处理 |
| 跨主题检索 | 困难，容易遗漏 | 自动建立关联节点 |
| 数据量扩大后 | 检索成本急剧上升 | 关联反而更丰富 |

**实际应用场景**：
一篇同时涉及"数据分析+心理学+个人成长"的文章，在Notion里你不知道放哪个文件夹；在AI知识图谱里，它会同时出现在三个维度的关联节点中。

---

#### 概念三：Claude Code + Obsidian 的工具组合

**是什么**：用Obsidian作为本地文件存储容器，用Claude Code作为AI执行引擎，自动读取并整理文件夹内容。

**为什么重要**：
这是视频给出的具体可操作路径，把抽象理念变成了十分钟可完成的动作。

**详细解释**：

| 工具 | 角色 | 做什么 |
|---|---|---|
| Obsidian | 存储层 | 存放所有原始收藏内容 |
| Claude Code | 执行层 | 读取文件、分析内容、输出整理结果 |
| 指令 | 连接层 | 告诉Claude Code按Karpathy方式整理 |

操作步骤：
1. 在Obsidian中新建文件夹，命名为"原始数据"
2. 把所有资料放入该文件夹
3. 打开Claude Code，授权其操作Obsidian文件
4. 输入指令：按照Karpathy的知识库帮我整理我所有的Obsidian文件
5. 等待执行完成

**实际应用场景**：
整理完成后，Claude Code会输出Concept概念总结文档和知识图谱，可以直接用于创作选题、灵感发散和文案框架生成。

---

#### 概念四：知识积累的时间复利

**是什么**：个人知识库的价值不是即时的，而是随着内容积累量的增长呈指数级放大。

**为什么重要**：
视频明确给出了一个门槛：文本量需达到40万字以上才有较好体验。这意味着"现在开始收集"比"现在搭建知识库"更重要。

**详细解释**：
- 内容量少（如只有两三篇）：不需要搭建知识库，没有意义
- 内容量达到40万字以上：知识图谱的关联价值才开始显现
- 坚持收集一年后：别人的收藏夹是废墟，你的收藏夹是图书馆

**实际应用场景**：
现在就开始收集你感兴趣领域（心理学、经济学、商业、个人成长等）中不会过时的经典内容，不要求立刻整理，只要求持续收集。
