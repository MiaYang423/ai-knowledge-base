# 教案：Claude 从入门到自动化四阶段路径

> 来源视频：[抖音博主 · 抖音](https://www.douyin.com/video/7632686453138836410)
> 适合人群：刚接触 Claude 或 AI 工具、想系统上手但不知从何开始的普通用户

---

## 一、视频原文

我觉得很多人学 Claude 走了弯路，不是因为贪难，而是顺序搞反了。正确的顺序是先把它用起来，再把它配好，最后让它自动跑。

所以第一阶段是去认识它。你不要一上来就去研究什么高阶玩法，先去完成最简单的三件事：账号要注册，找几个真实任务体验一下，它和搜索引擎到底有什么本质区别。Claude 不是去回答问题的工具，它是在交付结果。这个认知转变越早建立越好。模型选择也可以在这个阶段确立，小任务可以用 Haiku 这个模型，日常用 Sonnet，难题用 Opus，别一把就说用最贵的。

第二阶段是你要去学会提需求了。这也是最容易被跳过的，也最影响体验的一步。Claude 感觉没用，90% 的原因还是在你的提示词，不是在模型本身。核心方法是三段式提示词：背景，就是你是谁、你在做什么；目标，就是你要它做什么，越具体越好；规则，就是格式、语气、字数、结构，告诉它怎么去交付。掌握之后输出的质量会立刻从"能看"变成"能用"。你这一步做扎实，后面所有阶段都会更顺利。

第三阶段是配置你的工作环境。这阶段的目标是把 Claude 从孤立的聊天框变成一个生产力系统的一部分。核心有三个功能：第一个是 Projects，这是体验质变的关键，把背景资料和规则一次性放进去，后续在同一个项目里开对话，它就自动带上你的上下文，不需要每次重新解释，也不用每次在提示词里重复格式要求。Connected，Claude 接入了 Google Drive、Slack 等工具，它能真正参与到你的工作流，不是一个独立存在。另外要养成一个习惯：对话变长了就压缩上下文并开新的对话继续，越聊越乱不是 Claude 的问题，是信息噪音把它给淹没了。

第四阶段是进阶自动化。当你发现自己在重复做同一样事情的时候，就是进入这一层的时机。Prompts，把高频的提示词做成可复用的工作流；Claude 把整个重复的流程交给 Claude 在后台自动执行；Claude Code 是写代码的人用了就回不去的体验；Research Mode 是需要深度使用的生存研究报告。这一层不用急着全部上手，按需来就可以。

所以 Claude 的正确用法不是去问问题，而是交任务。学校也是一样，先跑起来，然后学会下任务，然后配好环境，然后自动化地重复、替代你现在的工作。

### 2.3 核心概念精讲

#### 认知转变：Claude 是结果交付者，不是问题回答者

**是什么**：把 Claude 当成"执行任务、交付成果"的助手，而不是"搜索答案"的工具。

**为什么重要**：带着"问问题"的心态用 Claude，你会觉得它没用；带着"交任务"的心态，你才能发挥它的真实价值。

**详细解释**：
- 搜索引擎：你问 → 它给链接 → 你自己读
- Claude：你描述任务 → 它直接交付可用的结果（文档、代码、分析报告等）

**实际应用场景**：不要问"什么是市场分析"，而是说"帮我写一份针对 XX 产品的竞品分析报告，格式是表格，涵盖价格、功能、用户口碑三个维度"。

---

#### 三段式提示词

**是什么**：由背景、目标、规则三部分构成的结构化提示词写法。

**为什么重要**：视频明确指出，Claude 感觉没用 90% 的原因在提示词，不在模型本身。三段式是最快提升输出质量的方法。

**详细解释**：

| 部分 | 内容 | 示例 |
|------|------|------|
| 背景 | 你是谁、在做什么 | "我是一名产品经理，正在准备季度汇报" |
| 目标 | 要做什么，越具体越好 | "帮我写一份 500 字的用户增长分析" |
| 规则 | 格式、语气、字数、结构 | "用数据说话，分三段，结尾给出建议" |

**实际应用场景**：写方案、做分析、生成内容，任何需要 Claude 输出的场景都适用。

---

#### Projects 功能

**是什么**：Claude 的项目空间，可以一次性存入背景资料和规则，后续同项目内的对话自动带上这些上下文。

**为什么重要**：解决了"每次都要重新解释背景"的痛点，是体验从普通到质变的关键功能。

**详细解释**：
- 把常用背景（公司介绍、产品资料）和规则（输出格式、语气要求）存入项目
- 同一项目内开新对话，Claude 自动读取，无需重复粘贴

**实际应用场景**：长期项目协作、固定风格的内容创作、团队共用的工作流模板。

### 2.4 速查表

| 阶段 | 核心目标 | 关键动作 |
|------|----------|----------|
| 第一阶段：认识 | 建立正确认知 | 注册账号、跑真实任务、选对模型 |
| 第二阶段：提需求 | 提升输出质量 | 掌握三段式提示词（背景+目标+规则） |
| 第三阶段：配环境 | 融入工作流 | 用 Projects、接入外部工具、管理上下文 |
| 第四阶段：自动化 | 替代重复工作 | Prompts 工作流、Claude Code、Research Mode |

| 模型 | 适用场景 |
|------|----------|
| Haiku | 小任务、快速响应 |
| Sonnet | 日常使用 |
| Opus | 复杂难题 |

### 2.5 延伸思考

1. 视频说"顺序搞反了"是最大弯路——如果你直接跳到第三、四阶段会发生什么？为什么基础认知和提示词能力必须先建立？

2. 三段式提示词的"规则"部分（格式、语气、字数）在实际工作中有多少人会认真写？跳过这一步对输出质量的影响具体体现在哪里？

3. 视频提到"对话变长了要压缩上下文开新对话"——这说明 Claude 的记忆机制有什么局限？在长期项目中如何设计上下文管理策略？

---

### 2.6 一句话记住它

> Claude 不是搜索引擎，它是你的任务执行者——先把它用起来，再学会下命令，最后让它替你干活。
