# 教案：Agent评测核心指标与方法

> 来源视频：[抖音博主 · 抖音](https://www.douyin.com/video/7641973888016723441)
> 适合人群：有志于AI产品经理岗位、正在准备大模型面试的求职者

---

## 一、视频原文

最近在面试当中很多AI产品经理都会被问到什么是Agent的评测。很多人一提到评测，第一个反应就是看他回答准不准。但事实上在实际项目当中，如果你只是这么理解，那么你还是停留在传统聊天机器人的视角。所以我今天想梳理一下，Agent评测到底在看什么，他到底有哪些核心的指标以及具体怎么做。对于AI产品经理从零到进阶的所有材料我也都准备好了，可以老地方来拿。

首先我们需要明确他到底在评什么内容，不是说一上来就看效果好不好，而是要看我们对的目标。比如说我们不同的Agent他会解决不同的一些问题，那么他要具体完成什么样的任务，什么样的情况下算完成，什么样的情况下算失败，这个是需要我们前置去做一个判断的。

然后进一步，我们不能只看结果，而且要看过程。因为在很多传统的AI问答当中，我们只去看最终的答案是不是合理。但很多时候我们会漏掉一些中间的步骤，比如说怎么去理解用户意图、拆解任务、调用接口等等。所以在评测这边，我们一定要分成两层：一层是结果的评测，看他最终有没有完成对的任务；一个是过程的评测，看他完成任务的路径是否正确。

以及我们的Agent评测他要哪些核心指标。首先第一个任务完成率，看他最终的完成结果；其次就是意图理解准确率；还有就是工具调用的准确率；以及他最后的任务路径的正确性。

那么评测具体怎么做？首先第一步我们应该要去拆解对应任务场景，不要一上来就看这些指标，我们应该把它列成对应的场景，然后再建立对应的测试集，再进一步去看我们整体的定义评分标准，以及说我们离线测评和在线测评做一个综合的结合。离线测评更多是去看上线前，而在线测评更多关注真实用户使用过程当中我们的Agent是否有效。

## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**Agent评测不能只看答案准不准，必须同时评估任务完成率、意图理解、工具调用准确率和路径正确性，并结合离线与在线两种测评方式。**

---

### 2.2 知识地图

```
Agent评测体系
├── 评什么
│   ├── 明确任务目标（完成/失败判断标准）
│   └── 两层评测
│       ├── 结果评测（最终是否完成任务）
│       └── 过程评测（路径是否正确）
├── 核心指标
│   ├── 任务完成率
│   ├── 意图理解准确率
│   ├── 工具调用准确率
│   └── 任务路径正确性
└── 怎么做
    ├── 拆解任务场景
    ├── 建立测试集
    ├── 定义评分标准
    └── 离线测评 + 在线测评结合
```

---

### 2.3 核心概念精讲

#### 两层评测框架

**是什么**：将Agent评测分为"结果评测"和"过程评测"两个维度。

**为什么重要**：传统聊天机器人只看最终答案，但Agent需要完成多步骤任务，中间过程（意图理解、任务拆解、接口调用）同样关键，只看结果会漏掉大量问题。

**详细解释**：
- **结果评测**：Agent最终有没有完成对的任务，结果是否符合预期
- **过程评测**：完成任务的路径是否正确，包括意图理解、任务拆解、工具调用等中间步骤

**实际应用场景**：评测一个订票Agent时，不仅要看票订成了没，还要看它有没有走弯路、调用了哪些接口、有没有理解对用户意图。

---

#### 四大核心指标

**是什么**：衡量Agent性能的四个关键量化指标。

**为什么重要**：有了具体指标才能客观比较不同Agent的优劣，也能在迭代时有明确的优化方向。

**详细解释**：

| 指标 | 含义 |
|------|------|
| 任务完成率 | Agent最终完成任务的比例 |
| 意图理解准确率 | 正确理解用户意图的比例 |
| 工具调用准确率 | 选对工具、传对参数的比例 |
| 任务路径正确性 | 完成任务所走路径是否合理、无冗余 |

**实际应用场景**：在面试中被问到"如何量化Agent性能"时，用这四个维度系统作答，远比只说"准确率"更有说服力。

---

#### 离线测评 vs 在线测评

**是什么**：两种不同阶段的评测方式。

**为什么重要**：两者关注点不同，需要结合使用才能全面评估Agent的实际效果。

**详细解释**：
- **离线测评**：上线前，基于测试集进行评估，关注功能正确性
- **在线测评**：上线后，基于真实用户使用数据，关注实际有效性

**实际应用场景**：产品上线前用离线测评验证基本能力，上线后用在线测评持续监控真实场景下的表现。

---

### 2.4 速查表

| 维度 | 关键点 |
|------|--------|
| 评什么 | 先明确任务目标和完成/失败标准 |
| 结果评测 | 最终任务是否完成 |
| 过程评测 | 路径是否正确（意图→拆解→调用） |
| 指标1 | 任务完成率 |
| 指标2 | 意图理解准确率 |
| 指标3 | 工具调用准确率 |
| 指标4 | 任务路径正确性 |
| 测评方式 | 离线测评（上线前）+ 在线测评（上线后）|
| 操作步骤 | 拆场景→建测试集→定评分标准→综合测评 |

---

### 2.5 延伸思考

1. 对于同一个任务，Agent A用3步完成、Agent B用50步完成，任务完成率相同，但路径效率差异巨大——在实际产品中，你会如何设计"路径效率"这个指标的具体评分规则？

2. 离线测试集的构建往往依赖人工标注，而Agent处理的任务场景可能非常多样——如何设计一套高覆盖率、低成本的测试集构建方案？

3. 在线测评中，如果用户没有明确反馈（没有点"踩"或"赞"），如何间接判断Agent是否真正完成了任务？

---

### 2.6 一句话记住它

> 评测Agent不是给聊天机器人打分——结果和过程都要看，四个指标缺一不可，离线+在线双管齐下才算真正的评测体系。
