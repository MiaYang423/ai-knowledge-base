## 一、视频原文

一起来看看字节的大模型 Agent 算法面经，看看你能答上来几道。这套题整体难度偏难，但是前面是基础，后面才是真正区分水平的地方。我是小哲，点赞收藏加关注，我们马上开始。

**第一道题：Transformer 的结构组成和各部分作用是什么？**

这题先讲清楚主干。核心是 Embedding、位置编码、多头注意力、前馈网络、残差连接和 Layer Norm。注意力负责建模上下文关系，前馈网络做非线性变换，残差和归一化保证深层训练稳定。

**第二道题：怎么降低 Transformer 的计算复杂度？**

关键是注意力的复杂度来自序列长度的平方。工程上可以用稀疏注意力、局部窗口注意力、滑动窗口、低秩分解或者 Flash Attention 这类优化。面试重点不是背名字，而是知道长文本场景怎么降延迟和显存。

**第三道题：LoRA 微调的原理是什么？Rank 怎么选？**

LoRA 的核心是冻结原模型参数，只在部分线性层旁边加低秩增量矩阵。Rank 越大表达能力越强，但显存、训练成本和过拟合风险也更高。一般先从小 Rank 做基线，再看任务复杂度和验证集效果调整。

**第四道题：KV Cache 是什么？为什么能提升推理速度？**

KV Cache 缓存的是历史 Token 计算出的 K、V 值。生成下一个 Token 时就不用反复重算前面所有上下文，只算新增部分。它主要优化自回归推理延迟，但会增加显存占用，长上下文要考虑淘汰和压缩策略。

**第五道题：RAG 的完整流程怎么讲？**

流程一般是：文档切分→清洗→向量化→入库，查询时做向量召回→重排→拼上下文→交给大模型生成。工程上还要关注切片粒度、召回率、权限过滤、引用溯源，以及答案不确定时的兜底策略。

**第六道题：向量检索库时间衰减对召回有什么影响？**

这题考的不是公式，而是新旧知识怎么平衡。可以把时间作为排序特征，给新内容更高权重，但不能简单覆盖与语义相关性。常见做法是向量相似度加时间分、分层召回，最后用重排模型统一打分。

**第七道题：微调训练数据怎么构建？如何保证多样性和质量？**

核心是先定义任务边界，再从真实日志、人工标注、合成数据里构造样本。质量上要去重、去噪、做安全过滤和人工抽检。多样性上要覆盖不同意图、难度、边界 case，不能只堆同一种高频问法。

**第八道题：RAG 加知识图谱的 Agent 系统，图谱更新机制怎么设计？**

关键是把离线构建和在线增量分开。离线做实体抽取、关系抽取和一致性校验；在线把新增知识走消息队列或事件流增量更新。实时性靠异步刷新和版本控制，一致性靠冲突检测、回滚和可观测指标。

**第九道题：训练 LoRA 冻结哪些层？依据是什么？**

一般不会盲目全量打开，可以优先在注意力的 Q、V 或者部分 MLP 层加 LoRA。看任务是偏知识、偏格式还是偏推理能力。依据是验证集效果、训练稳定性、参数量和部署成本，最终要用实验说话。

**第十道题：高并发 Agent 系统里怎么优化召回和生成延迟？**

召回阶段可以做缓存、并行检索、索引分片、粗排加精排。生成阶段要用流式输出、KV Cache、批处理和超时降级。真正落地还要监控 P95 延迟，区分是模型慢、检索慢还是工具调用慢。

**第十一道题：大规模 Agent 系统的多线程和多进程资源调度怎么设计？**

核心是把任务拆成可调度单元，比如检索、工具调用、模型生成和后处理。CPU 密集型走进程池，IO 密集型走异步或线程池，再配合队列、优先级、限流、熔断和资源隔离，避免单个任务拖垮整体系统。

### 2.3 核心概念精讲

#### Transformer 核心结构

**是什么**：由 Embedding、位置编码、多头注意力、前馈网络、残差连接和 Layer Norm 组成的深度学习架构，是现代大语言模型的基础。

**为什么重要**：所有大模型面试的起点，必须能清晰讲出每个组件的作用，不能只背名字。

**详细解释**：

| 组件 | 作用 |
|------|------|
| Embedding + 位置编码 | 将 Token 转为向量，加入位置信息 |
| 多头注意力 | 建模 Token 之间的上下文关系 |
| 前馈网络（FFN） | 做非线性变换，增强表达能力 |
| 残差连接 | 缓解梯度消失，保证深层训练稳定 |
| Layer Norm | 归一化，稳定训练过程 |

**实际应用场景**：理解 Transformer 结构是分析模型推理瓶颈、选择优化方案的基础。

---

#### LoRA 微调

**是什么**：冻结原模型参数，只在部分线性层旁边添加低秩增量矩阵（A×B）进行微调的参数高效训练方法。

**为什么重要**：LoRA 是目前最常用的大模型微调方案，面试必考，且需要讲到 Rank 选择和层选择的工程依据。

**详细解释**：
- Rank 越大：表达能力越强，但显存/成本/过拟合风险越高
- Rank 选择策略：先小 Rank 做基线，根据验证集效果和任务复杂度调整
- 冻结层选择：优先在注意力的 Q、V 层或部分 MLP 层加 LoRA，依据是任务类型和实验效果

**实际应用场景**：在资源有限的情况下，对大模型做领域适配或指令微调。

---

#### 高并发 Agent 系统延迟优化

**是什么**：在高并发场景下，通过多层次优化（缓存、并行、流式输出、监控）降低 Agent 召回和生成延迟的系统设计方案。

**为什么重要**：这是算法面后半段的区分题，考察的是系统设计能力，不只是算法理解。

**详细解释**：

| 阶段 | 优化手段 |
|------|----------|
| 召回阶段 | 缓存、并行检索、索引分片、粗排+精排 |
| 生成阶段 | 流式输出、KV Cache、批处理、超时降级 |
| 监控 | P95 延迟监控，区分模型慢/检索慢/工具调用慢 |

**实际应用场景**：线上 Agent 服务的性能调优，保证高并发下的响应时间 SLA。

1. 视频说"Flash Attention"可以降低 Transformer 的计算复杂度——Flash Attention 的核心思路是什么？它解决的是计算复杂度问题还是显存访问问题？两者有什么区别？

2. RAG 流程中"重排（Rerank）"这一步的作用是什么？如果去掉重排，只用向量召回，在什么场景下会明显影响答案质量？

3. LoRA 微调时"先从小 Rank 做基线"——但如果任务本身很复杂，小 Rank 的基线效果很差，如何判断是 Rank 不够还是数据质量的问题？有没有系统性的排查方法？
