## 一、视频原文

一起来看看阿里的 AI Agent 解决方案面题，看看你能答上来几道。这套题整体难度偏难，但是核心不是背概念，而是能不能讲清楚落地方案。我是小哲，点赞收藏加关注，我们马上开始。

第一道题：AI Agent 和 Function Calling 的区别是什么？Function Calling 是模型调用工具的一种能力，AI Agent 是围绕目标完成任务的系统，可能包含规划、记忆、工具调用、知识库和反馈循环。简单说，Function Calling 是零件，AI Agent 是整套执行流程。

第二道题：AI Coding 工具如何辅助开发 AI Agent？它可以帮你生成工具 Schema、写编排样例、补测试、解释框架代码和排查错误。但它不理解完整业务逻辑，所以核心流程、鉴权、安全和异常处理仍然要人工把关。提效可以，甩锅不行。

第三道题：LangChain、LangGraph 等 Agent 框架适合什么场景？LangChain 组件丰富，适合快速搭 RAG 和工具调用；LangGraph 更适合有状态、可分支、可回放的 Agent 工作流。真正落地时要看团队是否需要框架生态，还是更需要可控和可观测。

第四道题：AI Agent 落地时常用哪些云服务能力？常用模型服务、向量数据库、对象存储、消息队列、函数计算、日志监控、鉴权管理和内容安全。Agent 不是一个模型接口，线上还要有弹性、审计、告警、灰度和成本控制。

第五道题：常见大模型推理引擎有哪些，推理关注哪些指标？常见有 vLLM、TensorRT-LLM、TGI 和 SGLang。推理指标主要看吞吐、首 token 延迟、总延迟、显存占用、并发能力和模型兼容性。不同业务对吞吐和延迟的权重不一样。

第六道题：为什么需要 PD 分离，业界方案通常如何实现？PD 分离就是把 Prefill 和 Decode 阶段拆开优化。Prefill 更耗算力和带宽，Decode 更关注持续生成和缓存管理。分离后可以独立扩缩容、提高资源利用率。常见方案是分层调度和 KV Cache 管理。

第七道题：AI Agent 单链落地的技术挑战和发展趋势是什么？挑战是稳定性、成本、可解释性、鉴权安全、评测困难和长任务可靠性。趋势是更强的工具生态、更标准的协议、更细的模型路由，以及从聊天框走向业务流程自动化。

好，这套面题我们就分享到这里。这套题表面看是在问 Agent 的框架、云服务和推理引擎，其实真正考的是解决方案落地能力。希望在评论区看到你的答案。

### 2.3 核心概念精讲

#### Function Calling vs AI Agent

**是什么**：Function Calling 是模型调用单个外部函数的能力；AI Agent 是围绕目标、包含规划/记忆/工具调用/反馈循环的完整任务执行系统。

**为什么重要**：面试中两者经常被混淆，能清晰区分说明候选人理解了 Agent 的架构层次。

**详细解释**：

| 维度 | Function Calling | AI Agent |
|---|---|---|
| 定位 | 单次工具调用能力（零件） | 完整任务执行系统（整套流程） |
| 包含要素 | 参数结构化调用 | 规划 + 记忆 + 工具调用 + 知识库 + 反馈循环 |
| 典型场景 | 查天气、查数据库 | 自动完成多步骤复杂任务 |

**实际应用场景**：客服 Agent 需要先理解用户意图（规划），调用订单查询接口（Function Calling），再根据结果决定下一步（反馈循环），这整套才是 Agent。

---

#### PD 分离（Prefill-Decode 分离）

**是什么**：将大模型推理的 Prefill（输入处理）阶段和 Decode（逐 token 生成）阶段拆分到不同资源上分别优化。

**为什么重要**：两个阶段的资源瓶颈完全不同，混在一起会导致资源浪费；分离后可独立扩缩容，大幅提升资源利用率。

**详细解释**：

| 阶段 | 资源特点 | 优化方向 |
|---|---|---|
| Prefill | 耗算力 + 带宽（并行处理输入） | 高算力 GPU / 批处理 |
| Decode | 耗缓存（逐 token 生成） | KV Cache 管理 / 低延迟 |

**实际应用场景**：高并发在线推理服务中，Prefill 集群和 Decode 集群分开部署，根据流量分别扩缩容，降低整体推理成本。

---

#### Agent 落地云服务矩阵

**是什么**：AI Agent 线上运行所需的完整云服务组合，远不止一个模型 API 接口。

**为什么重要**：很多候选人只知道调模型接口，不了解生产级 Agent 需要的工程基础设施，这是面试中的高频失分点。

**详细解释**：

| 服务类型 | 作用 |
|---|---|
| 模型服务 | 核心推理能力 |
| 向量数据库 | 长期记忆 / RAG 检索 |
| 对象存储 | 文件 / 知识库存储 |
| 消息队列 | 异步任务解耦 |
| 函数计算 | 工具执行弹性伸缩 |
| 日志监控 | 可观测性 / 告警 |
| 鉴权管理 | 安全边界控制 |
| 内容安全 | 输入输出过滤 |

**实际应用场景**：企业内部知识问答 Agent 需要向量数据库存知识库、对象存储放文档、日志监控追踪每次调用链路、内容安全过滤敏感信息。
