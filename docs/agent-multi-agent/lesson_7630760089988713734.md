### 2.3 核心概念精讲

#### 内置本地 MCP 服务

**是什么**：Pencil 在工具内部直接集成了 MCP（模型上下文协议）服务器，无需额外配置，Agent 可以直接识别并调用 Pencil 的设计能力。

**为什么重要**：传统设计工具与 AI 之间存在交互壁垒，Agent 无法直接操作设计软件；内置 MCP 服务打通了这个壁垒，让 AI 能真正"看懂"和"操作"设计工具。

**详细解释**：
- 本地 MCP 服务：在用户本机运行，低延迟，数据安全
- Agent 直接识别：无需手动描述工具能力，Agent 自动获取 Pencil 的技能清单
- Agent 直接调用：可通过自然语言指令让 Agent 操作 Pencil 完成设计任务

**实际应用场景**：用 Agent 自动生成 UI 组件、调整设计参数、批量处理设计资产。

---

#### 全 JSON 结构设计

**是什么**：Pencil 将所有设计数据以 JSON 格式存储和表达，而非传统设计工具的私有二进制格式。

**为什么重要**：JSON 是大模型最容易理解和处理的数据格式之一，全 JSON 结构意味着 AI 可以直接读取、理解和修改设计内容，大幅提升 AI 协同设计的效率和准确性。

**详细解释**：
- 模型友好：大模型天然擅长处理 JSON 结构化数据
- 理解效率高：AI 无需额外的格式转换，直接理解设计意图
- 对比传统工具：Figma 等工具的文件格式对 AI 不友好，需要额外的解析层

**实际应用场景**：Agent 读取 JSON 格式的设计文件，直接理解组件结构、样式参数、布局关系，并进行精准修改。

### 2.5 延伸思考

1. 全 JSON 结构的设计工具在 AI 协同上有优势，但在渲染性能和文件体积上是否会有劣势？这种 tradeoff 在实际产品设计中如何权衡？

2. Pencil 内置本地 MCP 服务意味着设计数据留在本地，这在隐私保护上有优势，但如果需要团队协作，本地 MCP 如何支持多人协同场景？

3. 随着越来越多的设计工具开始支持 MCP，设计工具生态会如何演变？未来的 UI 设计工作流中，人类设计师和 Agent 的分工边界会在哪里？
