## 一、视频原文

建议所有人都去学Claude Skills，真的太香了。之前我们写篇文章，从折腾选题到收集资料，再到写出稿、改稿、校对，就算是专业的创作者，至少也要两小时起步。现在我将大博主都在用的写作流程提炼成SOP，再让Claude帮我写成Skill，就可以让AI帮你自动化写作和归档一篇高质量的文稿，从之前的两小时起步到20分钟就能搞定。今天手把手带你过一遍整个流程，看看Skill到底有多爽。

前置的Claude Code安装、模型接入切换、Skill如何安装的基础知识，我这里不展开讲，相关的操作我会整理好知识库放在评论区。

装好Claude Code，建议大家第一步先安装Skill Creator，这个官方创建Skill的技能。它可以交互式引导你创建Skill，只需通过一系列问题帮你梳理需求，就能自主帮你生成Skill。

在开始创建我们的写作Skill之前，我们需要拆解一下我们的工作流程，分为：理解需求、信息搜集、选题确认、创建任务、拆解学习我的风格、出稿创作，将AI分为这几个步骤。

第一步，理解需求，按照格式自动保存项目BRIEF文件夹。
第二步，信息搜集与知识管理。因为涉及到科技产品测评，所以多渠道的信息搜集是必须要做的，搜索完了自动保存到对应文件夹，内容必须包含信息搜集时间和信息来源。
第三步，选题讨论是必做的，不要让AI直接写文章，而是要先和你讨论选题，让AI根据需求提供3-4个选题方向，包含核心内容方向、工作量评估、内容大纲等。
第四步，如果选题要真实测试或配图，AI会创建文档到协作文件夹，帮你做一个任务销项清单。
第五步，学习风格，参考文件夹中历史文章的风格，提取开头方式、结构偏好、语言特征、经济风格等内容。
第六步，根据我的风格加入时间案例进行出稿创作。出稿无需特别完美，咱们让它先把框架线搭出来。
第七步，让它进行三重审校。就像在媒体工作一样，出稿是小编写的，发布前要经过主编的一层层审核，从数据校对到去AI味，再到细节打磨，让出稿成为直接能用的高质量中稿文档。

我同样整理在知识库了，直接把文档上传到Claude让它理解内容，自己执行，你只等十几分钟就创建好了。

创建好的Skill文档存放目录在这个文件夹，你可以打开确认文件是否符合Skill的标准。Skill.md是必须有的，重点关注Name、Description这两个元数据，里面包含了AI自动触发的要求。其他的reference是参考文档，Script是运行脚本，Template是模板。

大家可以看一下我的写作Agent的目录结构：风格参考，放我之前写的几个平台数据都不错的爆文；写作技巧，放我自己整理的Markdown文件；规范指南，目前放一些防线流指南；运行细节；用户人设，关于我想让它知道的我的信息和写作需求与倾向。

我的项目是按照文档用途分类，让Claude Code自动写入对应文件夹。这样这个写作Agent会随着我沉淀的资料越来越多，会变得越来越专业。你只需自然语言就可以触发调用自动化写作Skill，暗自加载，高效的同时还十分节省token，效果很好，真的很香。

### 2.3 核心概念精讲

#### Claude Skills

**是什么**：Claude Code中可自定义的自动化能力单元，通过Skill.md文件定义触发条件和执行逻辑，实现特定工作流的自动化。

**为什么重要**：让AI不再是"一次性对话工具"，而是能持续复用、越用越专业的个人Agent。

**详细解释**：
- **Skill.md**：必须有，包含Name（技能名称）和Description（触发条件描述）两个核心元数据
- **reference/**：参考文档（如风格样本、规范指南）
- **scripts/**：运行脚本
- **template/**：模板文件
- 用自然语言触发，Claude自动识别并调用对应Skill

**实际应用场景**：写作自动化、代码审查、数据整理、邮件生成等重复性工作流。

---

#### 写作SOP的七步拆解

**是什么**：将内容创作流程拆解为7个明确步骤，让AI能按步骤自动执行。

**为什么重要**：SOP的质量直接决定Skill的质量，拆解越清晰，AI执行越准确。

**详细解释**：
| 步骤 | 内容 | 关键要求 |
|------|------|---------|
| 1.理解需求 | 保存项目BRIEF | 按格式自动归档 |
| 2.信息搜集 | 多渠道搜集 | 必须含时间+来源 |
| 3.选题讨论 | 提供3-4个方向 | 含大纲+工作量评估 |
| 4.任务清单 | 测试/配图需求 | 创建销项清单 |
| 5.学习风格 | 分析历史文章 | 提取开头/结构/语言特征 |
| 6.出稿创作 | 先搭框架 | 无需完美，先出结构 |
| 7.三重审校 | 数据校对→去AI味→细节打磨 | 直接可用的高质量中稿 |

**实际应用场景**：科技测评、内容营销、自媒体创作。

---

#### Skill Creator（官方Skill创建工具）

**是什么**：Claude Code内置的官方Skill，通过交互式问答引导用户创建自定义Skill。

**为什么重要**：降低Skill创建门槛，无需了解Skill开发标准，用自然语言描述需求即可生成。

**详细解释**：
- 安装后通过对话描述需求
- 可以把工作流文档直接上传给Claude让它理解并生成Skill
- 创建完成后重启Claude Code即可使用

**实际应用场景**：快速将个人工作流封装成可复用的Skill。

### 2.5 延伸思考

1. 视频中的写作Skill依赖"历史文章风格参考"来学习个人风格，随着时间推移风格会演变，Skill的知识库应该如何定期更新和维护？

2. "三重审校"（数据校对→去AI味→细节打磨）是让输出更接近人类写作的关键，如何在Skill中设计具体的审校规则，让AI真正能识别并修改"AI味"？

3. Skill的Name和Description是AI自动触发的依据，如果创作者有多个不同类型的写作Skill（如科技测评、产品分析、生活随笔），如何设计Description避免触发混淆？
