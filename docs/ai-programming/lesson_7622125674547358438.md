## 一、视频原文

今天想跟大家分享一下 Vibe Coding 是如何帮我这一个没有开发背景的人打开了三个技术界的新世界大门的。

第一个是关于 GitHub 和开源。虽然这两个词我以前就听过，但是我从来都没有用过，因为我觉得它们离我的世界真的是太远了，只有很专业的那种开发人员才会去用。但是当我开始 Vibe Coding 之后，我必须去学如何用 GitHub 的时候，我才发现这个东西没有我想象的复杂。并且因为我学过了一些基础的 GitHub，包括如何去控制一个 repo，所以现在我看到网上一些开源的好玩的东西，我可以直接去用。例如最近很火的 G-Stack，还有 Mirofish，还有 OpenClaw，这些都是人人都可以去 GitHub 上克隆下来并且尝试一下的好东西。GitHub 上有这么多好玩的东西，感觉我以前大概是错过了一个宇宙吧。当然，当我开始了解开源社区之后，我也很开心地去参与，并且也把我的一些项目放到了 GitHub 上来开源给大家。

第二个就是 Markdown 文档，也就是所谓的 `.md` 文件。在 Vibe Coding 之前我从来没有听说过 Markdown 或者 `.md` 文档，但是现在我知道了，它就是一个文字文档加上一点点格式而已，并没有什么复杂的地方。当我们开启一个新项目的时候，我们可能会写一个 `project.md` 或者是 `CLAUDE.md`，把我们所有的产品需求都写在里面。我们现在听说的各种 Skills，它们最主要最核心的文档也是一个 `.md`。还有如果你有 OpenClaw 的话，你知道它有 `SOUL.md`、`AGENTS.md`、`MEMORY.md`，这些都是你可以打开直接读的。而且还有像 OpenCD 这样的工具，它所有的文档都是以 `.md` 的方式来写的。

第三个就是听得懂并且可以参与技术型对话了。以前学东西我一直摸不透我具体都学到了什么、什么知识，感觉就是一边在尝试一边在玩。但是我最近发现我不知不觉还是学到了很多技术知识。有时候最近我跟一个团队来讨论知识管理，我就想到了 OpenClaw 的 `MEMORY.md` 还有下面的每天的 Daily Log；当我们讨论到语音助理不用保存录音的时候，就想到了我用了好多个月的 Gemini，只是把当下的信息都提取出来，然后不会保留任何录音；再或者当我们讨论如何保存一些敏感信息的时候，我听得懂 `.gitignore`，听得懂 Environment Variable。所以虽然我没有意识到我具体都学到了哪些内容和单词或者是概念，但是在跟别人的沟通和交流中，我发现我在前进过程中确实积累了很多新的知识。所以这让我更坚定地想继续一直不停地用新的 AI 产品、新的 AI 科技，和尝试自己的 Vibe Coding 项目。

### 2.3 核心概念精讲

#### 概念一：GitHub 与开源

**是什么**：GitHub 是一个代码托管平台，开源是指将代码公开给所有人免费使用和贡献。

**为什么重要**：GitHub 上有海量现成的好工具可以直接克隆使用，不需要从零开始，参与开源也是技术人融入社区的方式。

**详细解释**：
- 克隆（clone）一个 repo：把别人的项目下载到本地，直接运行
- 典型开源项目：G-Stack、Mirofish、OpenClaw——人人可用
- 从"消费者"到"贡献者"：把自己的项目也放上去开源

**实际应用场景**：看到网上推荐的 AI 工具，不用等别人打包，直接去 GitHub 克隆下来自己跑。

---

#### 概念二：Markdown 文档（.md）

**是什么**：一种轻量级文本格式，用简单符号表示标题、列表、加粗等格式，文件后缀为 `.md`。

**为什么重要**：AI 编程工具（OpenClaw、Claude 等）的核心配置文件都是 `.md` 格式，读懂它等于读懂工具的"大脑"。

**详细解释**：

| 文件名 | 用途 |
|--------|------|
| `project.md` / `CLAUDE.md` | 写产品需求，告诉 AI 项目背景 |
| `SOUL.md` | OpenClaw 的人格设定 |
| `AGENTS.md` | OpenClaw 的行为规则 |
| `MEMORY.md` | OpenClaw 的长期记忆 |
| `SKILL.md` | 各类技能的核心说明文档 |

**实际应用场景**：开新项目时写一个 `CLAUDE.md`，把需求、规则都写进去，AI 每次都能按你的意图行事。

---

#### 概念三：技术型对话能力

**是什么**：在日常使用 AI 工具过程中，潜移默化地积累技术词汇和概念，能够参与并理解技术讨论。

**为什么重要**：技术能力不只是写代码，能听懂、能聊、能类比，就已经是跨越了认知门槛。

**详细解释**：
- 知识管理讨论 → 联想到 `MEMORY.md` + Daily Log
- 隐私/录音讨论 → 联想到 Gemini 不保存录音的机制
- 敏感信息讨论 → 听懂 `.gitignore`、Environment Variable

**实际应用场景**：和技术团队开会时，不再一脸懵，能用自己的 Vibe Coding 经验类比和理解技术方案。
