## 一、视频原文

这几分钟十分钟帮大家搭建自己的个人知识库。前段时间 OpenAI 的创始人 Karpathy 提出了一个个人知识库的理念，这个观点在推特上非常火，获得了 1900 多万的浏览量，各大科技博主也都纷纷下场，网上的教程铺天盖地，但是我完全看不明白他们为什么要弄得那么复杂。今天也帮大家在十分钟之内完成自己的个人知识库。

首先大家了解，传统的知识库往往是类似于这样的树状结构，做知识的梳理和整理。这样的知识库不合理的地方在于，你必须要通过手动对所有的内容进行分类、注释、整理，这种分类的工作量是非常大的。记住，AI 时代，任何一种不直接产生工作价值的事都应该把它外包化，比如扫地交给扫地机器人，洗衣服交给洗衣机，烘干交给烘干机，对不对？

另外，这种分类方式一篇内容只放在一个分类里面，但一篇内容往往有多个维度。有的时候技术与数据分析相关，又和个人经验相关，又和心理学相关，又和超级个体相关，那把它放在哪里？而当你积累的数据量过大的时候，你往往很难找到所有对你有价值的内容，检索的成本非常高。

而 Karpathy 的知识库最大的便利就在于，所有你收集到的内容完全不用你自己整理，你只需要把你收集到的所有信息放到一个文件夹里，接下来让模型给你整理好就可以了。最后怎么在十分钟之内你就可以把它搭建完成呢？你在 Obsidian 里面创建一个存放原始数据的文件夹，你所有的资料放进去，然后到 Claude Code 的界面里面去告诉他，按照 Karpathy 的知识库帮我整理我所有的 Obsidian 的文件。接下来吃点零食等他完成执行就好了。

那你可以看一下我在 Claude Code 输入的信息，然后他的一些分析以及他输出的结果。我们看一看，在 Concept 这个已经整理好的总结概念内容里面，这些内容其实就已经可以为我的创作提供灵感了。看看他整个的知识图谱是怎样的，他把大语言模型知识库这个概念和哪些内容进行了关联，大家可以看到，这也是为什么我们说用大语言模型搭建的知识图谱会比我们传统文件型的知识库更直观好用的原因。至于我这篇知识库的口播文案，主要的内容也是由他输出的，可以看到在他的分析框架里面他采用了哪些知识点，然后他是怎么思考的，他写了哪些内容。

所以搭建个人知识库这件事情，第一，他没有网上博主所说的那么复杂，他只需要让 Claude Code 有操作 Obsidian 的全权就够了。这又怎么打通？你仍然可以问他说，我现在需要打通我的 Obsidian，应该怎么操作，他会告诉你。

但到最后我们也不能无脑吹，不是说这个形式的知识库就是所有状态下的最优解，并不是。只有当你的文本量达到 40 万字以上的时候，你才会有比较好的体验。比如说你的文件夹里面只收藏了两三篇内容，那你就不用考虑搭建知识库这件事情，对吧，没有意义。当你收集了足够多的内容的时候才有价值。

这也从侧面说明了，你从现在开始就应该要为你自己的基础信息和框架知识做整理、做收集，不一定现在就要把知识库完成。你未来想要做哪些内容，你对哪些方向比较感兴趣，心理学、经济学、商业、个人成长，你现在就可以收集那些不会过时的经典的内容。那么一年之后，别人的收藏夹变成了一片废墟，沉寂无人访问，而你的收藏夹在一年之后可以变成一座只属于你自己的图书馆，成为你的灵感来源、文案输出的好帮手。

在 AI 时代，大家在工具面前其实是平权的，但是能否给 AI 提供优质、明确的信息源，才是大家对 AI 的使用结果产生差异的原因。所以请尽早开始积累自己的信息库，关注我，带你高效学习 AI 时代的前沿知识。

### 2.3 核心概念精讲

#### 概念一：Karpathy 式个人知识库

**是什么**：由 OpenAI 创始人 Karpathy 提出的一种知识管理理念，核心是把所有收集的原始信息交给 AI 模型自动整理，而非人工分类。

**为什么重要**：
传统知识管理耗费大量时间在"分类整理"这件本身不产生价值的事情上，Karpathy 式知识库把这部分工作完全外包给 AI，让人只需专注于"收集"和"使用"。

**详细解释**：

| 对比维度 | 传统树状知识库 | Karpathy 式知识库 |
|----------|--------------|-----------------|
| 整理方式 | 人工手动分类 | AI 自动整理 |
| 内容归属 | 一篇只能放一个分类 | 多维度关联，不受限 |
| 检索方式 | 按目录翻找 | 语义检索，知识图谱 |
| 使用门槛 | 随时可用 | 需积累 40 万字以上 |
| 维护成本 | 高（持续整理） | 低（丢进去即可） |

**实际应用场景**：
收集了大量关于"商业 + 心理学 + 个人成长"的文章，传统方式不知道放哪个分类，Karpathy 式知识库可以让 AI 自动建立多维关联，检索时从任意角度都能找到相关内容。

---

#### 概念二：AI 外包化原则

**是什么**：视频中提出的一个判断原则——任何不直接产生工作价值的事情，都应该交给工具或 AI 去做。

**为什么重要**：
这是理解为什么要用 AI 搭建知识库的底层逻辑。手动分类本身不产生创作价值，是典型的"应该被外包"的工作。

**详细解释**：

视频用了一个生活类比来说明：

- 扫地 → 扫地机器人
- 洗衣服 → 洗衣机
- 烘干 → 烘干机
- 知识分类整理 → AI（Claude Code）

**实际应用场景**：
判断一件事值不值得自己做的标准：这件事本身能否直接产生输出价值？如果不能，优先找工具替代。

---

#### 概念三：Claude Code + Obsidian 的搭建组合

**是什么**：视频推荐的具体工具组合——用 Obsidian 存放原始资料，用 Claude Code 执行自动整理任务。

**为什么重要**：
把抽象的"AI 整理知识库"理念落地成具体可操作的步骤，10 分钟内可完成初始搭建。

**详细解释**：

操作步骤：

1. 在 Obsidian 中新建一个文件夹，专门存放原始数据
2. 把所有收藏的资料（文章、笔记等）放入该文件夹
3. 打开 Claude Code，输入指令：「按照 Karpathy 的知识库帮我整理我所有的 Obsidian 文件」
4. 等待执行完成（期间可以休息）

如果不知道如何打通 Obsidian 与 Claude Code 的权限，可以直接问 Claude Code：「我现在需要打通我的 Obsidian，应该怎么操作」，它会给出具体指引。

**实际应用场景**：
整理完成后，Claude Code 会在 Obsidian 中生成 Concept 概念总结文件和知识图谱，可直接用于创作灵感提取和文案输出。

---

#### 概念四：信息积累的复利效应

**是什么**：知识库的价值不是立刻显现的，而是随着积累量的增长呈现出复利式增长——内容越多，知识库越有用。

**为什么重要**：
视频明确指出文本量需达到 40 万字以上才有较好体验，这说明"现在开始积累"比"现在就搭建"更重要。

**详细解释**：

- 内容少（两三篇）：搭建知识库没有意义
- 内容达到 40 万字以上：才能体验到知识图谱的真正价值
- 一年持续积累：收藏夹从"废墟"变成"个人图书馆"

**实际应用场景**：
现在就确定自己感兴趣的方向（心理学 / 经济学 / 商业 / 个人成长等），开始系统收集该领域不会过时的经典内容，为未来搭建知识库打基础。
