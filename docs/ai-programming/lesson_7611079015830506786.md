## 一、视频原文

你让AI写代码，写完发现一跑就崩；你让AI改个小功能，结果整个项目全废了。别急，全世界最懂AI编程的人之一，Django创始人Simon Willison给出了一个杀手级方法，四个字：红绿灯法。今天我把他的整套Agentic Engineering方法论给你拆透。

Simon Willison是Django框架的联合创始人，开源圈的老炮，全球写AI编程实践最深的人之一。他最近启动了一个新项目叫Agentic Engineering Patterns。注意，这不是Vibe Coding。Vibe Coding是你随便说两句话AI就给你写代码，能不能用另说。Agentic Engineering是专业工程师用AI做工具，你控制它，不是它控制你。

这个项目的第一篇就是Red Green TDD。TDD是测试驱动开发，在软件行业存在了20多年，但Simon说AI编程时代它不再是可选项，而是必须的。为什么？因为AI写的代码你没法逐行检查，你让它改个按钮颜色，它可能顺手把你的登录逻辑删了，你根本看不过来。但测试，测试就是你给AI定的及格线：红灯，没达标；绿灯，通过。AI写得再花，过不了测试就是不行。

具体怎么操作，三步。第一步，先写测试。你不用写代码，你写"我要什么"。比如你跟AI说，帮我写一个函数，输入一篇文章，输出所有标题。先别写实现，先用Red Green TDD写测试，AI就会先给你一组测试用例。第二步，跑测试，确认是红灯。这一步很关键，为什么？因为AI有时候会写出永远通过的测试，那跟没测试一样。你必须确认测试先是失败的，才能证明它是有效的。第三步，让AI写实现代码，然后跑测试，绿灯就过，红灯就重写。就这么简单。

还有一个配套技巧叫First Run Test。什么意思？每次你开一个新的AI编程会话，第一句话就说"先跑测试"。三个好处：第一，AI知道项目有测试套件，以后每次改代码都会自动想到跑测试；第二，测试数量告诉AI项目有多大，它会更谨慎；第三，AI看到你重视测试，它会主动给你写测试。四个字：拉高一个台阶。

你可能觉得这不就是写写测试吗，有什么大不了的。Simon在另一篇文章里说了一句很有意思的话：写代码，现在几乎不要钱了。以前一个工程师一天写几百行代码，现在AI几分钟写几千行。Cloudflare，一个工程师加一个AI，一周重写了整个Next.js框架，花了1100美元。代码产量爆炸了，但质量怎么保证？

这就是Red Green TDD的真正价值，它不是一个小技巧，是代码免费时代的核心竞争力。你想想看，当每个人都能让AI写代码，什么变成了稀缺资源？不是写代码的能力，而是定义"什么叫对"的能力。谁能写出好的测试，谁能定义清晰的验收标准，谁就能让AI持续产出高质量的成果。

Simon说这套思路和1994年的设计模式一样，那本书定义了软件工程的上一个时代，Agentic Engineering Patterns可能定义下一个。不管你用Claude Code还是Codex，这个方法都管用。记住一个原则：别只告诉AI"帮我做"，要告诉它"做成什么样算对"。这一个习惯，质量翻一倍。

总结一下：AI编程时代，写代码不值钱了，能定义标准的人才值钱。Red Green TDD，先写测试，红灯确认，绿灯通过，就这么简单。关注我，每天带你掌握最新AI资讯和前沿工具，我们下次见。

### 2.3 核心概念精讲

#### Red Green TDD（红绿灯测试驱动开发）

**是什么**：一种软件开发方法，核心是"先写测试，再写实现代码"。红灯 = 测试失败（代码还没写），绿灯 = 测试通过（代码实现正确）。

**为什么重要**：AI写代码速度极快但你无法逐行审查，测试是你给AI设定的及格线，确保AI无论怎么改代码，都必须通过你定义的验收标准。

**详细解释**：

三步操作流程：

| 步骤 | 操作 | 要点 |
|---|---|---|
| 第1步 | 先写测试，描述"我要什么" | 不写实现代码，让AI生成测试用例 |
| 第2步 | 跑测试，确认红灯 | 必须先失败，证明测试是有效的 |
| 第3步 | 让AI写实现，跑测试 | 绿灯通过，红灯重写 |

**实际应用场景**：使用Claude Code、Codex等AI编程工具开发任何功能时，先让AI写测试再写实现，防止AI"顺手"破坏其他功能。

---

#### First Run Test（首次跑测试技巧）

**是什么**：每次开启新的AI编程会话时，第一句话固定说"先跑测试"，让AI立即了解项目现状。

**为什么重要**：建立AI对项目测试体系的认知，从根源上引导AI在每次修改代码后都自觉跑测试。

**详细解释**：三个具体好处：
1. AI知道项目有测试套件，以后改代码时会自动想到跑测试
2. 测试数量让AI判断项目规模，更加谨慎操作
3. 你重视测试的态度会引导AI主动给你写测试

**实际应用场景**：每次打开Claude Code、Cursor等工具开始新对话时，作为固定习惯执行。

---

#### Agentic Engineering vs Vibe Coding

**是什么**：两种截然不同的AI编程方式。Vibe Coding是随便说两句话让AI写代码，结果能不能用另说；Agentic Engineering是专业工程师把AI当工具，保持对代码的控制权。

**为什么重要**：区分这两种模式，决定了你在AI时代是"被AI控制"还是"用AI作为工具"。

**详细解释**：
- **Vibe Coding**：门槛低，随意，输出质量不稳定，适合快速原型
- **Agentic Engineering**：系统化，有测试保障，适合生产级代码

**实际应用场景**：个人玩具项目可以Vibe Coding，商业项目或团队协作必须用Agentic Engineering思维。

### 2.5 延伸思考

1. 视频说"AI写代码几乎不要钱了，定义标准的能力才值钱"——那么写出高质量测试用例本身需要什么能力？这种能力是否也会被AI替代？

2. "First Run Test"这个技巧的本质是通过固定习惯来影响AI的行为模式——这背后反映了AI编程工具怎样的工作机制？还有哪些类似的"会话初始化技巧"？

3. Simon Willison将Agentic Engineering Patterns类比为1994年的设计模式——设计模式解决了面向对象时代的代码组织问题，Agentic Engineering Patterns要解决的核心问题是什么？它是否真的有可能成为下一个时代的软件工程范式？
