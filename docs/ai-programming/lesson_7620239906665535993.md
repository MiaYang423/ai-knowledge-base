### 2.3 核心概念精讲

#### 概念一：Skill（Agent 技能插件）

**是什么**：可以安装到 AI Agent 上的功能模块，类似手机 App，装上之后 Agent 就具备对应的能力。

**为什么重要**：不需要自己写代码或配置复杂流程，直接安装现成 Skill 就能扩展 Agent 的能力边界。

**详细解释**：视频中提到的 Skill 名为"Follow Builders not Influencers"，安装后 Agent 就能自动去抓取指定推特账号的内容并整理推送给用户。

**实际应用场景**：在 Claude 或其他 Agent 平台上找到该 Skill 并一键安装，Agent 即可执行信息抓取任务。

---

#### 概念二：Follow Builders not Influencers（关注建造者而非网红）

**是什么**：一个筛选信息源的策略，专门追踪 25 位真正在做 AI 产品和技术的一线建造者，而非泛泛传播 AI 资讯的网红博主。

**为什么重要**：信息源的质量决定认知的质量。建造者分享的是一手实践经验和真实观点，信息密度和价值远高于二手转述内容。

**详细解释**：

| 对比维度 | Influencer（网红） | Builder（建造者） |
|---|---|---|
| 内容来源 | 二手转述、热点追踪 | 一手实践、真实思考 |
| 信息密度 | 低，掺杂大量噪音 | 高，关键动态为主 |
| 适合目的 | 娱乐、泛了解 | 深度学习、跟进前沿 |

**实际应用场景**：用这 25 个账号替代自己漫无目的地刷推特，把有限注意力集中在最有价值的信息上。

---

#### 概念三：定时自动推送（Agent 定时任务）

**是什么**：让 Agent 在固定时间自动执行抓取和整理任务，无需手动触发。

**为什么重要**：把"主动刷信息"变成"被动接收精华"，从根本上减少信息焦虑和时间浪费。

**详细解释**：视频中的具体配置是每天早上 10 点让 Agent 自动抓取完毕，用户起来直接看摘要即可，整个信息消费流程变得可控且高效。

**实际应用场景**：配置好时间后，每天早上花几分钟看 Agent 整理好的推文摘要，替代原本可能花费半小时以上的无序浏览。

### 2.5 延伸思考

1. **信息源的选择本身就是一种能力**：这 25 位 Builders 是谁选出来的、依据什么标准？如果你来构建自己的"精选信息源"列表，你会用什么标准筛选？

2. **从"刷内容"到"管理信息流"**：让 Agent 帮你过滤信息，本质上是把注意力管理外包给了 AI。这种方式会不会带来新的风险，比如错过 Agent 判断为"不重要"但实际对你有价值的内容？

3. **Skill 生态的潜力**：如果 Agent 可以通过安装 Skill 不断扩展能力，未来"给 Agent 配置合适的 Skill 组合"会不会成为一项重要的个人生产力技能？

---

### 2.6 一句话记住它

> 与其每天被 AI 信息淹没，不如让 Agent 帮你只盯 25 个最值得看的人，每天早上送到你面前。
