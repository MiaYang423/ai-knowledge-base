# 教案：用 AI 工具制作游戏概念 Demo

> 来源视频：[抖音博主 · 抖音](https://www.douyin.com/video/7636749070652367016)
> 适合人群：想用 AI 工具快速制作游戏概念演示视频的独立开发者或创意设计师

---

## 一、视频原文

这条 AI 游戏 Demo 发出来之后很多人都在问我是怎么做的，今天就简单拆一下流程，教大家怎么用 Image 2 和 Kling 2.0 制作类似这种游戏画面。

整个流程主要分四步。

第一步，用 Image/MidJourney 创建 2D 角色。这里要把人物外形、服装、纹身、机械手臂这些关键细节写清楚，因为后面的游戏界面和视频画面都会围绕这个角色继续延展。

第二步，用某工具增加 3D 风格和纹理，把 3D 参考图的质感转移到刚才的 2D 角色上。这样角色变得更像游戏里的 3D 角色，细节和质感都更真实。

第三步，用 GPT Image 2 生成游戏概念界面。不是生成单独角色图，而是让 AI 模拟一整套游戏 UI。我一共做了 5 张关键画面，这样画面会更像一个完整的游戏项目，整个 Demo 的世界观也会更真实。

第四步，用 Kling 2.0 做成完整视频。我用的是 Kling 2.0 模型，提示词先写清楚角色设定，让视频里的主角场景和视觉质感保持一致，然后详细地写清楚每秒出现的画面，比如 0 到 2 秒出现开始页面，2 到 4 秒是人物介绍页，要写清楚每个页面的镜头动作等细节。这样生成出来的视频更像一只真正的游戏预告片。

如果对生成的视频很满意，还可以再做一步视频增强和高清放大，这样能提升画面清晰度和细节质感。

这套流程很适合做游戏概念 Demo、角色展示、游戏 UI 测试，或者虚拟的项目预告片。

### 2.3 核心概念精讲

#### 概念一：角色一致性设定

**是什么**：在流程最开始就把角色的视觉细节定义清楚，并在后续所有步骤中保持统一。

**为什么重要**：游戏 Demo 的可信度来自"世界观统一"，角色外形、服装、纹身、机械手臂等细节如果前后不一致，整个 Demo 会显得零散、不像真实项目。

**详细解释**：
- 第一步生成 2D 角色时就要把所有关键细节写进提示词
- 第三步生成 UI 界面时，角色要出现在界面里，保持视觉延续
- 第四步生成视频时，提示词开头先写角色设定，再写分镜

**实际应用场景**：制作角色卡、游戏人物介绍页、预告片主角出场镜头，都需要依赖这个统一的角色设定。

---

#### 概念二：整套 UI 生成（而非单图）

**是什么**：用 GPT Image 2 一次性模拟出完整的游戏界面系统，包括开始页、人物介绍页等多个关键画面，而不是只生成一张角色图。

**为什么重要**：单张图只能展示角色，一套 UI 才能让人相信"这是一个真实的游戏项目"，大幅提升 Demo 的说服力和专业感。

**详细解释**：

| 做法 | 效果 |
|------|------|
| 只生成角色图 | 像概念艺术，不像游戏项目 |
| 生成完整 UI 套图（5 张） | 像真实游戏的截图，世界观完整 |

**实际应用场景**：向投资人或团队展示游戏概念、做游戏 UI 风格测试、制作项目提案素材。

---

#### 概念三：逐秒分镜提示词

**是什么**：在用 Kling 2.0 生成视频时，把提示词写成"时间轴 + 画面内容 + 镜头动作"的分镜格式，精确控制每一秒的画面。

**为什么重要**：AI 视频生成如果只给模糊描述，输出结果随机性很高；逐秒分镜能让生成结果更接近预期，视频节奏感更像真正的预告片。

**详细解释**：
- 格式示例：
  - `0–2 秒`：出现游戏开始页面，镜头缓慢推进
  - `2–4 秒`：切换到人物介绍页，角色从左侧滑入
- 提示词结构建议：
  1. 先写角色设定（外形、风格、世界观）
  2. 再按时间段逐一描述画面和镜头动作

**实际应用场景**：制作游戏预告片、产品概念视频、虚拟项目宣传片。

---

#### 概念四：视频增强与高清放大

**是什么**：在视频生成完成后，额外做一步画质提升处理，提高清晰度和细节质感。

**为什么重要**：AI 生成视频的原始分辨率和细节有时不够精细，增强处理后更适合对外展示或投放。

**详细解释**：这是一个可选步骤，适合对最终画质有较高要求的场景，比如对外发布的宣传视频或需要全屏展示的演示场合。

**实际应用场景**：Demo 视频需要在大屏幕展示、发布到社交媒体追求高画质时使用。

### 2.4 速查表

| 步骤 | 工具 | 核心操作 | 关键注意点 |
|------|------|----------|------------|
| ① 创建 2D 角色 | 图像生成工具 | 生成角色基础形象 | 细节要全：外形、服装、纹身、机械手臂 |
| ② 增加 3D 质感 | 3D 风格化工具 | 质感迁移 | 参考 3D 图要与角色风格匹配 |
| ③ 生成游戏 UI | GPT Image 2 | 生成完整 UI 套图 | 至少 5 张关键画面，模拟完整游戏界面 |
| ④ 合成预告视频 | Kling 2.0 | 逐秒分镜生成视频 | 先写角色设定，再写时间轴分镜 |
| ⑤ 画质增强（可选） | 视频增强工具 | 高清放大处理 | 对外展示时建议做此步骤 |

### 2.5 延伸思考

1. **角色一致性的边界在哪里？** 当 Demo 需要多个角色时，如何在提示词层面管理多角色的视觉统一性，避免不同角色之间风格割裂？

2. **逐秒分镜的粒度怎么把握？** 分镜写得太细 AI 可能无法完全执行，写得太粗结果又难以控制——对于不同时长的视频，最优的分镜粒度应该是怎样的？

3. **这套流程能否反向用于真实游戏开发？** 用 AI Demo 验证了概念之后，这些生成的 UI 图和角色图能否直接作为真实开发的参考资产，还是必须重新制作？
