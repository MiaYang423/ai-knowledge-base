## 一、视频原文

Skill 装多了，真的非常容易乱。我自己呢，现在常用的 Skill 是 11 个，按场景分成了三类。

我先说第一类，它是属于效率工具。

首先是 Agent 的位置。以前我们搜资料需要一个平台一个平台去切换，现在你只需要一句话说清楚你要什么，这个 Skill 它能帮你跨平台去找——YouTube、小红书、推特、公众号都能支持，还能直接提取字幕，不花 API 的费用。

第二个呢，是 NotebookLM Skill。这个不是 NotebookLM 本身，而是在 Claude 里面操控它的 Skill，把抓进来的资料批量导入，让它去做归纳分析，你不用打开软件，在另一个界面里直接搞定。

第三个是 BrowserBase，这个是让 AI 直接接管你的 Claude，打开网页、点按钮、填表单、截图、爬数据，跟你自己坐在电脑前面操作没有任何区别。每天你要重复的流程，你丢给它就可以。

接下来是 Skill Play Store，Claude 的官方出品，写 Skill 的 Skill。把你每天重复的工作流程用一句话描述出来，它能帮你封装成可以反复调用的 Skill，让 Claude 按照你的方式工作。我觉得这个 Skill 长期的价值会非常非常高。

接下来是 Find Skill，相当于 Skill 的一个搜索引擎。你不知道该用哪个 Skill 的时候，把你的需求告诉它，它根据上下文帮你找到最合适的，不用自己一个一个去翻。

好，接下来讲第二类，是产品开发。

JustStack，这个是从创业圈出来的开源技能库，里面的 Offers 是我用得最多的。说任何事之前，先被 AI 追问 6 个问题，把目标和边界条件逼出来。跳过这一步，做出来的东西返工率高很多。

Discipline Powers，这个是给 Claude Code 装的一套工程师的工作习惯——拆任务、写测试、代码审查，这些流程它默认就会走，不用你每次都提醒。本质上就是把 AI 编程从"能跑就行"推到"可以交付"。

然后 Framer Design 和 UI/UX Pro Max，这两个是做前端设计的 Skill。你不懂设计没关系，排版配色、细节它自己去判断，一个负责从零去生成页面，一个负责在现有的基础上做优化。

然后是 Playwright MCP，自动化测试产品。做完以后，让它模拟用户的操作去跑一遍，哪里点不动、哪里报错了，全部报出来，你不用自己手动一个一个点。

OK，再讲最后一类，就是自媒体，只有一个，但我几乎每天都在用——Humanizer。这个是 Guy 出品的，AI 写的东西通顺归通顺，就是读起来不像人话，这个 Skill 专门去处理这个问题，过一遍之后自然度明显不一样，做内容的基本上都用得上。

所以你看，三条线，11 个 Skill，基本上覆盖了我 90% 的日常场景：提效、做产品、做内容。

### 2.3 核心概念精讲

#### 概念一：Skill（Claude 技能）

**是什么**：安装在 Claude 中的功能扩展模块，类似手机 App，用于增强或定制 Claude 的能力。

**为什么重要**：
原生的 Claude 是通用的，而 Skill 让它能按照特定场景、特定工作流去运作，大幅减少重复提示词的编写成本。

**详细解释**：
- Skill 可以由官方出品（如 Skill Play Store），也可以来自开源社区（如 JustStack）
- Skill 装多了容易混乱，作者建议聚焦在真正高频的场景，精选常用的即可
- 有专门的 Skill 帮你找 Skill（Find Skill），说明 Skill 生态已经有一定规模

**实际应用场景**：
用 Skill Play Store 把自己每天重复的工作流描述一遍，让它自动封装成专属 Skill，之后一句话调用。

---

#### 概念二：AI 接管浏览器（BrowserBase）

**是什么**：让 AI 直接操控浏览器完成打开网页、点击按钮、填写表单、截图、爬数据等操作的 Skill。

**为什么重要**：
把人工重复点击的操作完全外包给 AI，解放双手，尤其适合每天有固定操作流程的场景。

**详细解释**：
- 效果等同于"你自己坐在电脑前操作"
- 不需要写脚本或懂自动化技术，直接用自然语言描述任务即可
- 适合数据采集、表单批量填写、定期截图存档等重复性工作

**实际应用场景**：
每天需要登录多个平台查看数据、截图汇报的工作，丢给 BrowserBase 自动跑。

---

#### 概念三：需求澄清前置（JustStack Offers）

**是什么**：在开始任何开发任务之前，强制让 AI 先追问 6 个问题，把目标和边界条件逼清楚的 Skill。

**为什么重要**：
作者明确指出，跳过这一步，做出来的东西返工率高很多。需求不清是 AI 辅助开发中最常见的浪费来源。

**详细解释**：
- 来自创业圈的开源技能库 JustStack
- 核心逻辑：AI 生成内容之前，先通过提问帮用户想清楚"要做什么、不做什么"
- 把模糊的需求变成有边界的任务，再交给 AI 执行

**实际应用场景**：
启动一个新功能开发时，先过一遍 Offers 的 6 个问题，确认目标、用户、约束条件之后再让 Claude Code 动手写。

---

#### 概念四：AI 内容人性化（Humanizer）

**是什么**：专门用于降低 AI 生成内容"机器味"、提升自然度的 Skill。

**为什么重要**：
AI 写的内容语法通顺，但读起来不像人说的话，在自媒体场景下会影响读者体验和内容传播效果。

**详细解释**：
- 由 Guy 出品
- 处理的核心问题：AI 文本"通顺但不自然"
- 过一遍之后，内容的自然度有明显提升
- 作者本人几乎每天都在用

**实际应用场景**：
用 AI 起草好公众号文章、视频脚本或社交媒体文案后，过一遍 Humanizer 再发布。

### 2.5 延伸思考

1. **Skill Play Store 的长期价值在哪里？** 作者说这个 Skill"长期价值非常高"，它本质上是在做什么——是在帮用户把个人经验和工作流"编程化"，这和传统的 SOP 文档有什么本质区别？

2. **Find Skill 的存在说明了什么问题？** 当 Skill 多到需要一个专门的搜索引擎来管理时，AI 工具的使用门槛是在降低还是在升高？普通用户和重度用户之间的使用差距会怎么变化？

3. **Discipline Powers 把 AI 编程从"能跑就行"推到"可以交付"，这个跨越意味着什么？** 工程质量标准（测试、代码审查、任务拆解）被内置进 Skill 之后，AI 辅助编程是否正在重新定义"初级程序员"的工作边界？
