### 2.3 核心概念精讲

#### Claude Code Agent Teams（多智能体团队）

**是什么**：Claude Code内置的功能，允许用户同时启动多个AI角色，每个角色承担不同职责，互相讨论后给出最终方案。

**为什么重要**：单个AI的注意力和视角有限，多角色分工可以从架构、业务、技术多维度审视方案，减少遗漏，模拟真实团队协作。

**详细解释**：作者的典型团队配置：

| 角色 | 职责 |
|---|---|
| 企业架构师 | 整体技术架构设计 |
| 前端工程师 | 前端代码实现 |
| 后端工程师 | 后端代码实现 |
| 商业分析师 | 市场和业务分析 |
| 项目管理人/高层 | 整体协调和决策 |
| 投资人（可选） | 商业可行性视角 |

团队规模：大项目6人，小项目3到4人。

**实际应用场景**：开发中大型项目时，先用Agent Teams讨论出架构方案，再进入具体开发阶段。

---

#### Claude Code X vs Claude Code 的分工

**是什么**：两个不同的Claude编程工具，在项目不同阶段各有优势。

**为什么重要**：选错工具会导致项目起步遗漏多、后期Bug难调，合理分工能提升整体开发质量。

**详细解释**：

| 工具 | 优势 | 适用阶段 |
|---|---|---|
| Claude Code X | 项目起步成型好，遗漏少，注意力机制更完整 | 项目初始化、架构搭建 |
| Claude Code | 日常开发主力，结合Agent Teams使用 | 功能开发、迭代 |

作者观察：Claude Code有时因注意力机制导致小需求被遗忘，Claude Code X在这点上表现更好。

**实际应用场景**：新项目先用Claude Code X做起步，后续日常开发切换到Claude Code + Agent Teams。

---

#### 三窗口并行开发工作流

**是什么**：作者独创的多窗口开发布局：左边Claude Code X、中间Claude Code、右边IDE，三个工具同时运行。

**为什么重要**：让不同工具并行工作，最大化开发效率，同时保留IDE作为最终审查和收尾的工具。

**详细解释**：
- 左（Claude Code X）：项目起步和架构阶段
- 中（Claude Code）：主力开发，结合Agent Teams
- 右（IDE）：代码审查、验收、收尾

**实际应用场景**：适合长期维护的中大型项目，通过布局固化工作流，减少切换成本。

### 2.5 延伸思考

1. 作者说"一个人开发中大型项目，不懂具体技术很容易出很多Bug"——那么Agent Teams的多角色讨论能在多大程度上弥补这个问题？哪些类型的Bug是AI团队仍然难以发现的？

2. Claude Code X和Claude Code的"注意力机制差异"导致了不同的使用场景——这说明在选择AI工具时，除了模型能力，还需要关注哪些维度的差异？

3. 这种"AI虚拟团队"的工作方式和传统的"单AI对话"有什么本质区别？在什么规模的项目上，多角色分工会开始带来真正的价值，而不只是增加复杂度？
