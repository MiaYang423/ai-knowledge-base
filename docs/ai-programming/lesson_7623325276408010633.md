### 2.3 核心概念精讲

#### Codex Plugin CC

**是什么**：OpenAI 官方发布的 Claude Code 插件，允许在 Claude Code 环境中直接调用 Codex 的代码能力。

**为什么重要**：此前在 Claude Code 中使用 Codex 需要通过 subagent 绕道实现，官方插件让调用更直接、更稳定，降低了集成成本。

**详细解释**：
- 插件名称：Codex Plugin CC
- 发布方：OpenAI 官方
- 使用场景：在 Claude Code 工作流中无缝接入 Codex 的代码生成与审查能力

**实际应用场景**：日常用 Claude Code 写代码时，直接唤起 Codex 进行 bug 修复或代码审查，无需切换工具。

---

#### 标准代码生成

**是什么**：Codex Plugin 的基础功能，根据需求生成代码或修复已有 bug。

**为什么重要**：这是最高频的使用场景，能直接提升编码效率。

**详细解释**：输入问题描述或有 bug 的代码，Codex 生成修复方案或新代码片段。

**实际应用场景**：发现某个函数有逻辑 bug，调用此功能让 Codex 直接给出修复版本。

---

#### 对抗性测试

**是什么**：在修复 bug 的同时，对代码的架构设计进行审查，发现潜在的设计问题。

**为什么重要**：不只是"能跑"，还要"设计合理"；对抗性测试帮助开发者在早期发现架构隐患，避免技术债积累。

**详细解释**：
- 功能 1（标准生成）：解决 bug
- 功能 2（对抗性测试）：同时审查架构是否合理，提出改进建议

**实际应用场景**：新功能开发完成后，用对抗性测试模式检查整体架构设计是否有明显缺陷。

---

#### 救援模式（Rescue Mode）

**是什么**：当 Claude Code 无法解决某个 bug 时，启用 Rescue 模式，Codex 接管整个代码上下文，完成后续所有工作。

**为什么重要**：解决了"Claude Code 卡壳"的痛点，相当于请来了一个更强的队友接手烂摊子。

**详细解释**：
- 触发条件：Claude Code 反复尝试仍无法解决 bug
- 工作方式：Codex 读取整个代码上下文，自主完成剩余修复和开发工作

**实际应用场景**：遇到复杂的多文件 bug，Claude Code 来回折腾解决不了，直接切换 Rescue 模式交给 Codex 接管。
