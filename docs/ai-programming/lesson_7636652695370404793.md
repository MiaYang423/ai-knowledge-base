## 一、视频原文

我 Vibe Coding 了一个自己的心舰。这是一个基于 Obsidian 搭建的属于我自己的个人心舰操作系统，主要的用途就是记录和管理我们自己的生活。

其实最近又重温了一下《星际迷航》，哇，这个心舰实在太帅了。这个就是我们的人生心舰，我把它分成了几个核心舱段：指令舱、气录舱、自产舱、生命舱、行行舱、记忆舱。

指令舱记录今天要做什么，自产舱记录我用了的东西，生命舱记录健康和身体的数据，行行舱记录我们的旅行地点，记忆舱则负责读书记录和一些自我的 reflection。

先来看看指令舱。这里我做了一个交互效果，就像是我们真正进入了这艘心舰。指令舱其实就是我每天的 To-do List。在这个舱室内我们可以添加每天的任务，我给它做了一个和首页的联动，当我们点击圆圈按钮的时候，首页也会帮我们划掉已完成的任务。

然后是自产舱。这里我会把自己的一些物品录入进来，比如我的电子设备，然后我们可以选择它的状态是使用中还是闲置。当我们输入价格后，我们的装备卡片就会自动计算我们持有的天数和入手成本，更加直观地判断我们物品的价值。

接着是生命舱。生命舱主要记录我们的身体数据，比如体重趋势。这里的数据我用的是小米体质秤的导出，然后我们也可以自己添加。还可以添加我们的体检报告，让 AI agent 帮我们自动分析身体的异常指标。

然后就是行行舱。这个模块是我最喜欢的。接下来是一个小熟博主的看项目，它用的是一个地球仪来记录我们去过的地方。在添加地点后，地球仪就会根据坐标渲染出地点，点进某个地点后就可以打开旅行笔记，记录我们的旅游体验。比较有意思的是这个地球仪它是可以放得很大的，最大可以放到这么大，可以看到非常精细的细节。

最后是记忆舱。记忆舱主要是用来记录我们读过的一些书和一些自我的复盘 reflection。可以给读过的书籍放上封面、摘抄好句、写一些读后感，也可以记录下我们的随笔和思考。

那么这就是我们的心舰 Space OS。我个人觉得工具的审美很重要，如果一个工具每天看起来都让我觉得很无聊，那我真的很难长期使用。但如果它像一艘我自己的心舰，那我会更愿意每天打开它。

OK，那今天的分享我就先到这里，谢谢大家。

### 2.3 核心概念精讲

#### 概念一：Vibe Coding

**是什么**：一种借助 AI 辅助生成代码的编程方式，不需要深厚编程基础，通过描述需求让 AI 帮你写出功能实现。

**为什么重要**：让没有系统编程经验的普通用户也能定制属于自己的个性化工具，大幅降低了"做自己的工具"的门槛。

**详细解释**：视频作者用 Vibe Coding 在 Obsidian 基础上实现了交互动效、地球仪渲染、AI 联动分析等较复杂的功能，这些在传统意义上需要一定前端开发能力才能完成。

**实际应用场景**：想定制 Obsidian 插件或页面交互效果，但不会写 JavaScript 的用户，可以用 Vibe Coding 描述需求来生成代码。

---

#### 概念二：Space OS（个人操作系统）

**是什么**：以"操作系统"为隐喻，把个人生活的不同维度（任务、物品、健康、旅行、阅读）分门别类管理的个人知识与生活管理系统。

**为什么重要**：传统的生活管理工具往往只有表格和清单，缺乏沉浸感和美感，导致用户难以坚持使用；Space OS 通过主题化设计提升了使用意愿。

**详细解释**：

| 舱段 | 管理内容 | 核心功能亮点 |
|------|----------|--------------|
| 指令舱 | 每日任务 | 任务与首页联动，完成即划掉 |
| 自产舱 | 个人物品 | 自动计算持有天数与入手成本 |
| 生命舱 | 身体健康 | 接入小米体质秤数据 + AI 分析体检报告 |
| 行行舱 | 旅行记录 | 3D 地球仪按坐标标注地点 |
| 记忆舱 | 阅读与复盘 | 书籍封面 + 摘抄 + 读后感 + 随笔 |

**实际应用场景**：希望把生活各个维度整合在一个地方统一管理，而不是在多个 App 之间反复切换的用户。

---

#### 概念三：工具审美与长期使用意愿

**是什么**：作者提出的一个观点——工具的视觉美感和沉浸感，直接影响用户是否能坚持长期使用它。

**为什么重要**：很多人买了笔记本、下了 App，用了两周就放弃，原因之一就是工具本身不够吸引人，打开它变成了一种负担而不是享受。

**详细解释**：作者以《星际迷航》心舰为灵感，把整个系统设计成"驾驶一艘宇宙飞船"的沉浸体验，让每次打开工具都有仪式感和期待感。

**实际应用场景**：在选择或定制个人管理工具时，不只考虑功能是否齐全，也要考虑界面是否让自己每天都想打开它。

### 2.4 速查表

| 对比维度 | 传统生活管理工具 | 作者的 Space OS |
|----------|-----------------|----------------|
| 形态 | 表格、清单、待办 App | 主题化沉浸式操作系统 |

| 地理记录 | 文字或图片 | 3D 地球仪 + 坐标渲染 |
| 实现方式 | 直接使用现成工具 | Obsidian + Vibe Coding 自定义开发 |
| 坚持使用动力 | 依赖自律 | 审美吸引力驱动内在动机 |

---

### 2.5 延伸思考

1. **审美驱动 vs 功能驱动**：作者认为工具审美影响长期使用意愿，但高度个性化的工具往往需要较高的搭建成本——对于大多数人来说，这个投入产出比合理吗？什么时候值得自己做工具，什么时候直接用现成的更好？

2. **AI agent 接管数据分析的边界在哪里**：视频中提到用 AI agent 自动分析体检报告的异常指标，这类健康数据的 AI 分析结果应该如何使用？它能替代医生判断吗？用户需要具备什么认知才能正确解读这些结果？

3. **个人操作系统的可持续性**：Space OS 的核心魅力部分来自于它的新鲜感和沉浸体验，但随着时间推移，这种新鲜感会消退吗？如何设计一个系统，让它在"好看"之外也能真正沉淀有价值的个人数据和习惯？
