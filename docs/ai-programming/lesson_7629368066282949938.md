## 一、视频原文

哎呀，AI 变成工具越来越像是个超级个体和一人公司了。十分钟不到，我就用 AI 做出了一个简历网站生成器。上传任意一个 PDF 简历，这个工具就会把简历做成一个单独好看的简历网站，并且支持切换风格和分享，非常适合在现在精简隐私招聘季帮助用户成为 offer 收割机。

在之前，完整的项目主要配置：产品经理、设计师、架构师、程序员、测试。但在 AI 时代，我们完全可以一人搞定。

我这次是怎么做的呢？用的是 TRAE Solo 新版本的 Builder 模式。Solo 的界面非常简洁，核心就是一个对话框，也符合 AI 时代自然语言交互搞定一切的趋势。

除了 Builder 模式之外，新版本还有一个 MTC 模式，也就是 Multi-task Code 模式，它不只是做开发，各种办公场景都能渗透，比如整理文件、创建 PPT、生成报告等等。

但回到 Builder 模式，新建一个项目，然后在对话框输入我的需求。在理解需求之后，Solo 会自行担任产品经理角色，帮我们完成一份 PRD，也就是产品需求文档，各种样式都会先帮我们定义出来。Solo 还会担任架构师角色，整体架构、技术栈、路由树与模型也会全部梳理出来。作为老板，我们觉得 OK 的话，跟 Solo 确认就可以正式入场了。

它会开始安装依赖、写代码，而且干活的过程中，我们已经可以通过右侧的预览窗口看到这个网站一点点成形了。

项目第一版出来之后，整体页面很现代，没关系，我们来演示一下功能。上传一份简历，一巴掌来了——简历上所有内容都被解析成标题了。虽然布局风格能切换，但该填入信息的位置都没填进去，下面还残留一些示例内容。好消息是主题颜色和字体切换没问题。那么只需要解决内容解析这一个问题就可以了。

在输入框里直接指示这个问题，让 Solo 考虑结合大模型做解析。然后稍等片刻，第二版搞定后，页面就多出了一个大模型配置区，用来进一步提升简历解析的准确性。我们还是直接上传简历试试，这次信息解析一点毛病没有，一个电子简历网站已经成形了。

再看其他几个功能：布局风格切换 OK，主题颜色切换 OK，字体设置切换 OK。最后是分享功能，复制链接直接粘贴到 Solo 的内置浏览器，成功打开了对应的简历网站，整体流程已经跑通了。

不过我觉得，两句提示词就做出来的网站还是有一定的迭代空间的。这时候就可以用到 Solo 的评论功能——直接点选想修改的元素，然后留下评论。比如我想要这个标题用更酷炫的渐变色，然后把评论添加到对话里。评论功能一个很大的好处是，我们不用很抽象地描述，比如"最上面那个区域里最大的标题"，直接选中对应区域，AI 就能很精准地找到要修改哪里，不会跑偏。

下面这个按钮我也想让它的动效更丰富一点，那还是点击标题、评论、添加到对话，然后回车让 Solo 修改。等修改完成之后再看，渐变色有了，更酷炫的点击动效也加上了。

除了手动提出优化点，我们还可以借助一些成熟的 Skill 继续提升。点击 Skill 技能市场，里面已经内置了一系列常用的先进 Skill。比如这个 Frontend Design，就是 Anthropic 官方出的前端设计技能包，它的作用是让 AI 做出更有审美方向、细节更完整的页面。废话不多说，肯定得按照来看看效果。

按照完之后回到项目，在输入框里用斜杠调用这个功能，要求增加更多简历风格样式。等它跑完之后，还是上传简历，布局风格里面多出了三个新选项：

- **粗野**：整体风格更张扬，有那种美式漫画的感觉
- **优雅**：确实优雅，整体更简约，走的是轻奢方向
- **深色赛博风**：科技感很强

所以这次这个项目，我大概只用了十几分钟左右，就已经拿到了一个成熟的网站。对于超级个体或一人公司来说，意义其实很大，因为拿到的不只是一个 demo，而是一个可以持续往业务方向推进的出发产品。比如你可以继续加付费功能、付费模式、分享解锁，把它一步步做成一个真正能对外收费的产品。

而 TRAE Solo 新版本的 Builder 模式，已经不只是帮你生成几个代码文件了，而是在参与整个从零到一的交付流程。目前这个升级版的 Solo 还在内测中，欢迎在评论区留言，我那边会送出一些官方邀请码，等你亲自体验。我们下期视频见。

### 2.3 核心概念精讲

#### 概念一：Builder 模式

**是什么**：TRAE Solo 新版本的核心开发模式，用自然语言对话驱动，让 AI 全程参与从需求分析到代码交付的完整流程。

**为什么重要**：它把过去需要产品经理、架构师、程序员多人协作的工作，压缩成一个人与 AI 的对话，大幅降低独立开发的门槛和时间成本。

**详细解释**：

| 阶段 | AI 扮演的角色 | 输出内容 |
|------|-------------|---------|
| 需求理解 | 产品经理 | PRD（产品需求文档）、功能定义 |
| 技术规划 | 架构师 | 技术栈选型、路由树、数据模型 |
| 编码实现 | 程序员 | 安装依赖、写代码、实时预览 |
| 迭代优化 | 执行者 | 根据反馈修复问题、调整样式 |

**实际应用场景**：视频中，作者输入"做一个简历网站生成器"的需求后，Solo 依次完成了 PRD 撰写、技术架构规划，再到代码生成，全程无需手动切换工具或角色。

---

#### 概念二：评论功能（可视化定点修改）

**是什么**：在预览页面中，直接点选想修改的 UI 元素并留下评论，再将评论附加到对话中让 AI 执行修改。

**为什么重要**：传统方式需要用文字描述页面位置（如"最上面区域最大的那个标题"），容易产生歧义导致 AI 修改跑偏。评论功能通过"所见即所指"，让 AI 精准定位目标元素，减少沟通损耗。

**详细解释**：
- 操作步骤：点选元素 → 输入评论 → 添加到对话 → 回车执行
- 适用场景：样式调整（渐变色、动效）、内容修改、布局微调

**实际应用场景**：作者想改标题渐变色和按钮动效，直接点选对应元素留言，AI 精准修改，两处改动均一次到位。

---

#### 概念三：Skill 技能市场

**是什么**：TRAE Solo 内置的能力扩展市场，提供可直接调用的专业 Skill 包，用斜杠命令在对话中激活。

**为什么重要**：引入经过专业打磨的 Skill，可以让 AI 的输出质量跃升到更高水准，而不必自己反复调教提示词。

**详细解释**：

| Skill 名称 | 出品方 | 作用 |
|-----------|--------|------|
| Frontend Design | Anthropic 官方 | 让 AI 输出更有审美方向、细节更完整的前端页面 |

调用方式：在输入框输入 `/Frontend Design`，附上具体需求（如"增加更多简历风格样式"）即可。

**实际应用场景**：调用 Frontend Design 后，简历网站新增了"粗野""优雅""深色赛博风"三种风格，视觉完成度明显提升。

---

#### 概念四：超级个体与"出发产品"

**是什么**：超级个体指一个人借助 AI 工具承担完整项目团队的职能；"出发产品"指 AI 交付的不只是 demo，而是可以持续迭代、推向商业化的初始版本。

**为什么重要**：改变了独立开发者对 AI 工具价值的认知——AI 不只是"写几个代码文件的助手"，而是"从零到一交付流程的参与者"。

**详细解释**：
- 传统项目团队：产品经理 + 设计师 + 架构师 + 程序员 + 测试
- AI 时代：一人 + TRAE Solo Builder 模式 ≈ 完整团队
- 出发产品的延伸路径：简历网站 → 加入付费功能 → 加入分享解锁 → 对外收费产品

**实际应用场景**：视频中十几分钟产出的简历网站，作者明确指出可以继续叠加付费模式、分享解锁等商业功能，逐步演化为真正的商业产品。

| 交互方式 | 自然语言对话 | 自然语言对话 |
| 典型场景 | 做网站、做工具产品 | 整理文件、生成文档 |

| 功能 | 解决的问题 | 使用方式 |

| 对话框指示 | 描述需求 / 反馈问题 | 直接输入文字 |
| 评论功能 | 精准定位 UI 修改位置 | 点选元素 → 留言 → 添加到对话 |

### 2.5 延伸思考

1. **Builder 模式中 AI 自动生成 PRD 和架构，如果方向判断出现偏差，用户在哪个环节介入纠正最高效？确认环节的"把关"能力对最终产品质量影响有多大？**

2. **评论功能解决了"描述不清导致 AI 跑偏"的问题，但对于逻辑层面的 Bug（而非视觉层面），这种可视化定点方式是否还适用？复杂业务逻辑的问题该如何高效反馈给 AI？**

3. **视频提到从 demo 到"可对外收费产品"只需持续叠加功能，但产品的安全性、稳定性、数据隐私等工程质量问题，在纯 AI 驱动的开发流程中如何保障？**
