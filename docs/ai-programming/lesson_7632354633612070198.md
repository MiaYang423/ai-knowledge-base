## 一、视频原文

我最近高强度用了两周 Claude Code，两个 Max 账号都不够用的那种。我现在最大的感受就是它真的特别强，但你没用好就随时会白白浪费掉。我今天不会去讲教程，我想讲三个我现在最真实的感受吧。

第一个感受就是一定要保存上下文。我最崩溃的一次是跟 Claude Code 聊了差不多一两个小时，我跟他聊清楚了所有我关于这个项目的一些想法，我把项目目标、背景资料、好的标准、想要做成什么样全跟他聊清楚了，而且我当时是用语音调的，越调越进入状态，最后他真的帮我做出来了一个还不错的一个产品 scale。结果第二天我准备继续调的时候，你猜怎么着，账号被封了。那一刻我真的是太崩溃了，又崩溃又无力又无语。我崩溃不是因为没了一个账号，而是前面所有的上下文、所有的共识、所有已经铺垫好的状态都没了。

所以如果你现在正在用 Claude Code，像我这种今天没有把东西做出来，一定要去不断地保存你们聊到哪了、当前的方案是什么、下一步要怎么做、判断的标准是什么。我希望大家都可以保存下来。AI 真的很强，但是他一旦失去上下文，我觉得就像一个突然失忆的队友。

OK，第二个感受就是不要什么都自己想，一定要让 AI 来主导。我以前用 AI 也会把步骤拆好，第一步该怎么着，第二步该怎么着，第三步怎么着。后来我发现，当 Claude Code、当 AI 足够聪明的时候，你其实不需要把他怎么做都想好。人更重要的是想清楚：我要做什么、为什么做、做到什么程度算好。你就可以跟他说一句话，这句话太重要了，这句话我也经常跟我朋友分享：你可以问我任何问题，直到你觉得你可以把这件事情做好为止。这句话真的特别好用，所有用过的朋友都给我非常不错的反馈。你让他反过来问你各种问题，他做好这件事需要哪些信息，直到他能做好这件事。这时候你跟 AI 的关系就变了，不是你一句一句地指挥，而是你负责方向，他负责推进。想清楚做什么是你的事，怎么把这个事做出来是 Claude 的事。

所以第三个感受就来了，Claude Code 真正强的地方不是单点能力，而是组合能力。当他只是一个聊天工具的时候，他确实已经很聪明了，但当你把各种 skill、MCP 工具链串起来之后，你的感觉绝对会完全不一样。他可以联网帮你写文、可以游览网页帮你查资料、可以读公众号抓信息然后整理很多信息结构，他可以围绕一个目标持续地帮你推进。这个感觉你真的会超级爽。

这个感觉就有点像英雄联盟，我们都会给英雄出装备，这个装备不是你有就好，或者越多越好，而是怎么搭配这件事更重要，怎么搭配让他进入到更强的一个状态。我觉得工具也是一样，不是装一堆 skill 就厉害，真正厉害的是你怎么组合他们，让他们进入到最适合当前任务的状态。当这套装备跑通的时候，你会明显感觉到他不再只是一个工具，更像是一个能跟你一起干活的队友。

所以这两周我用下来最大的一个判断就是：Claude Code 不单是一个代码工具，他更像是一个依赖上下文、任务定义和工具组合的执行系统。现在来说，你能想到的事他应该都可以尝试去做。如果你还只是一问一答，那你可能真的只用了他很小的部分能力。但如果你能把目标讲清楚、把上下文保存好，再把工具链搭起来，他真的能帮你把很多事情往前推。这就是我这两周最强烈的一个感受。后面我也会继续分享关于 skill、MCP 这套装备该怎么组合才能发挥到最大的效用。OK，先这样。

### 2.3 核心概念精讲

#### 概念一：上下文管理

**是什么**：在与 AI 的长时间协作中，主动保存对话进展、共识和状态，防止因意外中断导致信息全部丢失。

**为什么重要**：
作者花了一两个小时与 Claude Code 对齐项目目标、背景、标准，最终做出了满意的产品雏形。但账号被封后，所有铺垫全部归零——崩溃的不是账号没了，而是那些"共识和状态"没了。AI 一旦失去上下文，就像一个突然失忆的队友，之前建立的默契完全消失。

**需要保存的四类信息**：

| 保存项 | 说明 |
|--------|------|
| 聊到哪了 | 当前进展和阶段 |
| 当前方案 | 已确定的解决思路 |
| 下一步计划 | 接下来要做什么 |
| 判断标准 | 什么算做好、做完 |

**实际应用场景**：每隔一段时间或每完成一个阶段，把以上四项整理成文档或笔记，下次开新对话时直接粘贴给 AI 作为背景。

---

#### 概念二：让 AI 主导流程（反向提问法）

**是什么**：不再由人来拆解执行步骤，而是把"做什么、为什么做、做到什么程度算好"交代清楚，然后让 AI 反过来问你它需要的信息，直到它具备独立推进的条件。

**为什么重要**：
当 AI 足够聪明时，人去逐步拆解"第一步、第二步、第三步"反而是一种浪费——你在用人脑做 AI 擅长的事。真正有价值的人的工作是定义方向和标准，执行路径交给 AI 来规划。

**关键提示词**：
> 你可以问我任何问题，直到你觉得你可以把这件事情做好为止。

**使用方式**：

| 旧方式 | 新方式 |
|--------|--------|
| 人拆步骤，AI 执行 | 人定方向，AI 反向提问补全信息 |
| 人主导节奏 | AI 主导推进，人把关方向 |
| 容易遗漏关键信息 | AI 会主动索取它需要的信息 |

**实际应用场景**：开始一个新任务时，先告诉 Claude 你想做什么、为什么做、好的标准是什么，然后加上这句关键提示词，让 Claude 来问你问题，直到双方都准备好。

---

#### 概念三：工具组合能力

**是什么**：将 Claude Code 与各种外部工具（skill、MCP 等）串联起来，形成一套能围绕目标持续推进的工具链，而不是单独使用 Claude 作为聊天工具。

**为什么重要**：
Claude Code 单独使用时已经很聪明，但串联工具链之后，它能做的事情发生质变——从"回答问题"变成"持续推进任务"。作者用英雄联盟出装备来类比：装备不是越多越好，搭配对了才能发挥最大战力。

**工具组合能做的事**：

| 能力 | 说明 |
|------|------|
| 联网搜索 | 帮你查资料、获取最新信息 |
| 浏览网页 | 读取指定页面内容 |
| 读取公众号 | 抓取文章信息并整理结构 |
| 持续推进 | 围绕一个目标不断迭代执行 |

**实际应用场景**：根据当前任务类型，选择合适的 MCP 工具组合接入 Claude Code，而不是把所有工具都装上——关键是"适合当前任务的搭配"。

### 2.5 延伸思考

1. **上下文管理的边界在哪里？** 随着项目越来越复杂，需要保存的上下文会越来越长，AI 的上下文窗口也有限制——如何设计一套可持续的上下文管理方案，既不遗漏关键信息，又不让每次对话的启动成本过高？

2. **"让 AI 主导"是否有风险？** 当你把执行路径完全交给 Claude 规划时，AI 可能走向你没预料到的方向。如何在"放权给 AI"和"保持对项目的掌控感"之间找到平衡点？

3. **工具组合的"最优解"是动态的吗？** 作者提到搭配比数量更重要，但不同任务阶段（调研、开发、测试、迭代）所需的工具组合可能完全不同——是否应该为不同阶段预设不同的"装备方案"？
