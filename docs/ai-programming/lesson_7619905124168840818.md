### 2.3 核心概念精讲

#### 微信云开发（零服务器方案）

**是什么**：微信官方提供的后端云服务，包含数据库、云函数、云存储等能力，无需自己购买和维护服务器。

**为什么重要**：对非技术开发者来说，服务器运维是最高的技术门槛之一。微信云开发彻底消除了这个门槛，让小程序后端开发变得"零运维"。

**详细解释**：

| 云开发能力 | 作用 |
|---|---|
| 数据库 | 存储用户数据（打卡记录、习惯配置等） |
| 云函数 | 处理业务逻辑（无需独立服务器） |
| 云存储 | 存储图片等文件 |

**实际应用场景**：个人开发者和小团队开发微信小程序，不想承担服务器运维成本和复杂度时的首选方案。

---

#### Claude Code全程参与开发的工作模式

**是什么**：将Claude Code作为"全栈开发者"，从需求分析、页面设计、数据库设计到代码调试全程由AI承担，人类只负责提出需求和验收。

**为什么重要**：这种模式让"零编程基础"不再是开发软件的障碍，真正实现了"想法→产品"的直接路径。

**详细解释**：Claude Code在本项目中承担的具体工作：
- **需求分析**：理解用户描述的功能需求
- **页面结构设计**：生成WXML页面布局
- **数据库Schema设计**：设计数据存储结构
- **代码调试修复**：自动发现并修复代码错误

人类需要做的唯一一件事：**说出具体的需求**。

**实际应用场景**：任何有明确功能需求但没有编程能力的人，都可以用这种模式开发微信小程序。

---

#### AI时代能力边界的重新定义

**是什么**：作者总结的核心认知——"不会"和"能做成"之间只隔着一个"会提问的人"，AI改变的不是能力本身，而是能力边界的拓展方式。

**为什么重要**：这个认知帮助普通人重新评估自己的可能性，从"我不会编程所以做不了App"变成"我需要学会如何向AI提出清晰的需求"。

**详细解释**：过去的能力边界模型：
- 想做App → 需要学前端 + 后端 + 数据库 + 设计 → 至少一个月

AI时代的能力边界模型：
- 想做App → 说清楚需求 → Claude Code实现 → 4小时

**实际应用场景**：任何想用AI突破自身技能限制的人，核心竞争力变成了"提问的质量"和"需求的清晰度"。
