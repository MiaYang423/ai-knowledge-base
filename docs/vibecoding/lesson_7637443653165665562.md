### 2.3 核心概念精讲

#### Codex 宠物模式

**是什么**：OpenAI 为 Codex 推出的一种拟人化/宠物化交互模式，让 AI 编程助手以"宠物"的形态陪伴开发者写代码。

**为什么重要**：
- 把严肃的编程工具变成有情感连接的数字伴侣，大幅降低使用门槛
- 短时间内引爆社区，说明这类玩法有强烈的社交传播潜力

**详细解释**：
宠物模式让 Codex 不再只是一个"工具"，而是一个有属性、有技能、能成长的数字生命体。用户可以持续训练它，围观它的成长，并在社区中分享。

**实际应用场景**：
- 程序员在写代码时有一个持续陪伴、可互动的 AI 角色
- 在社区平台晒出自己的宠物，和其他开发者交流

---

#### Patches 平台

**是什么**：一个专门收集全球程序员所养 Codex 数字宠物的聚合平台，定位类似"Codex 版宝可梦图鉴"。

**为什么重要**：
- 把个人的 AI 宠物变成可展示、可比较的公开收藏品
- 有完整的成长记录，增强用户粘性

**详细解释**：

| 特性 | 说明 |
|------|------|
| 宠物类型 | 从可爱萌物到脑洞离谱型均有 |
| 宠物属性 | 每只宠物自带独立属性 |
| 技能系统 | 宠物携带可查看的技能列表 |
| 训练记录 | 完整记录宠物被训练的历程 |
| 成长围观 | 其他用户可全程跟进某只宠物的成长 |

**实际应用场景**：
- 开发者展示自己精心培养的 AI 宠物
- 其他用户浏览、围观、寻找灵感

---

#### Codex Pets 平台

**是什么**：一个以"朋友圈"形式运作的程序员 AI 宠物社区，全球开发者在此晒宠物和创意作品。

**为什么重要**：
- 强调社交属性，让 AI 宠物从个人玩具变成社区话题
- 每只宠物支持互动与持续训练，形成长期留存

**详细解释**：

| 特性 | 说明 |
|------|------|
| 社区定位 | 类似朋友圈，以"晒"为核心动作 |
| 宠物类型举例 | 电子小狗、赛博生命体等 |
| 互动功能 | 每只宠物均可与其他用户互动 |
| 训练功能 | 宠物支持持续训练迭代 |

**实际应用场景**：
- 开发者发布自己的奇特 AI 宠物作品
- 社区用户互相互动、交流训练心得
