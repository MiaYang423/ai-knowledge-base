### 2.3 核心概念精讲

#### 概念一：Vibe Coding

**是什么**：不依赖系统性编程知识，主要靠 AI 工具驱动完成代码编写的开发方式。

**为什么重要**：
让非开发背景的人也能独立完成中大型项目，大幅降低了软件开发的门槛。

**详细解释**：
视频中的博主明确表示自己"不是开发背景"，但通过长期使用 Vibe Coding，脑子里也积累了多种语言和技术的基本认知，甚至能自己排查部分错误。Vibe Coding 不等于完全不懂技术，而是以 AI 为主力，人负责方向和验收。

**实际应用场景**：
- 独立开发者做产品原型
- 产品经理直接实现自己的想法，不依赖工程团队

---

#### 概念二：Cursor Agent Teams 角色分工

**是什么**：Cursor 编辑器内置的 Agent Teams 功能，允许用户给 AI 设置多个不同职能角色，让它们互相讨论后输出最终方案。

**为什么重要**：
单一 AI 视角容易有盲区；多角色分工能从架构、商业、管理等多维度审视方案，减少遗漏。

**详细解释**：

| 角色 | 职责 |
|------|------|
| 架构师 | 设计整体技术架构 |
| 前端工程师 | 负责前端实现 |
| 后端工程师 | 负责后端实现 |
| 商业分析师 | 分析商业可行性和案例 |
| 高层管理人 | 统筹视角，把控方向 |
| 投资人（可选）| 从投资回报角度审视方案 |

团队规模灵活，可以是 3-4 人小团队，也可以是 6 人完整团队，根据项目复杂度调整。

**实际应用场景**：
- 项目立项阶段，用多角色讨论快速得出技术 + 商业综合方案
- 复杂功能设计时，避免只从单一技术视角思考

---

#### 概念三：三工具并行工作流

**是什么**：同时打开 Claude、Cursor、IDE Copilot 三个工具，在开发的不同阶段各司其职。

**为什么重要**：
不同工具有不同的优势区间，组合使用能弥补各自的短板，提升整体开发质量和效率。

**详细解释**：

| 位置 | 工具 | 使用时机 | 优势 |
|------|------|----------|------|
| 左侧 | Claude | 项目冷启动 | 架构储型完整，遗漏少，注意力机制更稳定 |
| 中间 | Cursor | 主力开发全程 | Agent Teams 功能强，适合迭代开发 |
| 右侧 | IDE Copilot | 开发中后期收尾 | 在编辑器内嵌入式补全 |

博主特别指出：Cursor 因注意力机制的问题，有时会遗漏一些小需求；而 Claude 在项目起步阶段的完整性表现更好，所以用 Claude 打底再切换到 Cursor 是更稳的策略。

**实际应用场景**：
- 新项目启动：先用 Claude 生成完整骨架，再导入 Cursor 继续开发
- 日常迭代：以 Cursor 为主，Copilot 辅助收尾

### 2.5 延伸思考

1. **角色设置的边界在哪里？** Agent Teams 的角色越多讨论越充分，但也可能引入噪音和矛盾意见——什么情况下该用 3 人小团队，什么情况下才需要 6 人完整团队？

2. **Vibe Coding 的技术天花板是什么？** 博主提到非开发背景做中大型项目容易出 Bug 且难以 Debug，那么 Vibe Coding 是否有项目复杂度的边界？超过这个边界后，人需要补充哪些最低限度的技术能力？

3. **工具切换的成本如何控制？** 三工具并行意味着上下文需要在不同工具间同步，这个同步成本如何最小化？是否有更高效的单工具替代方案？

---
