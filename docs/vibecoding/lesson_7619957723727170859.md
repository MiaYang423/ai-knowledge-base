## 一、视频原文

我现在每天用 Claude Code 超过10个小时，总结出了3个提效技巧，能够让它越用越懂你，干活不返工。

第一个：先商量再放手。复杂的任务先跟他说"别动手，给我方案"，他会自动切换为 Plan 模式，然后询问咱们细节、列步骤、给方案，等我们确认之后再开启狂编模式，把全线都给它——不管是上网查资料、写文件还是改代码，他都能给你库库干了，你就只需要去泡杯咖啡，回来就干完了。那如果你是新手，应该是找不到这个狂编模式的，需要在设置里把这个选项给勾上。

第二个：给他定规矩。在项目里建一个 `claude.md` 的文档，把你的偏好规则全都写进去。用的过程当中如果有需要让他记住的事情，跟他说"记住这一点"，他是真的能够记住，下次不会再犯同样的错误。那如果你的项目特别大，可以再开一个 Memory——跟项目有关的东西放 `claude.md`，跟你自己个人偏好有关的放在 Memory。

最后：用嘴说，不要用手打。按住 `H` 键快捷键就能调出语音输入，普通人说话的速度是打字的三倍，用这种方式你会不自觉地说得更详细，AI 做出来的东西质量更高、也更周全。

关注我，后面还有更多的 AI 使用技巧分享。

### 2.3 核心概念精讲

#### 概念一：Plan 模式（先商量再放手）

**是什么**：在让 Claude Code 动手之前，先让它进入"只出方案、不写代码"的规划模式。

**为什么重要**：
复杂任务直接开干容易跑偏，返工成本极高；先对齐思路再执行，能大幅减少无效工作。

**详细解释**：
- 触发词：在任务描述前加"**别动手，给我方案**"
- Claude Code 会自动识别并切换为 Plan 模式
- Plan 模式下它会主动询问细节、拆解步骤、输出完整方案
- 人工确认方案无误后，再授权它进入"狂编模式"全力执行

**实际应用场景**：
需要改动多个文件、涉及架构调整、或需求描述还不够清晰时，先用 Plan 模式对齐，再放手执行。

---

#### 概念二：claude.md + Memory 规则体系

**是什么**：通过文档和记忆模块，把你的偏好、规范持久化写入 Claude Code，让它"记住你"。

**为什么重要**：
每次重新交代规则既低效又容易遗漏；把规则固化后，Claude Code 每次启动都会自动遵守，越用越顺手。

**详细解释**：

| 存储位置 | 适合放什么内容 | 作用范围 |
|---|---|---|
| `claude.md`（项目根目录） | 项目技术栈、代码风格、目录结构约定等 | 当前项目 |
| Memory | 个人习惯、通用偏好、沟通风格等 | 跨项目通用 |

- 对话过程中随时可以说"**记住这一点**"，Claude Code 会将该内容追加到记忆中，下次不再犯同样错误

**实际应用场景**：
统一团队代码风格、固定回复语言、记录项目特有的命名规范等。

---

#### 概念三：语音输入（按住 H 键）

**是什么**：Claude Code 内置的语音输入功能，按住 `H` 键即可调出，用说话代替打字来输入 Prompt。

**为什么重要**：
- 速度优势：说话速度约是打字速度的 **3 倍**
- 质量优势：口述时人会自然说得更详细、更完整，AI 获得的上下文更充分，输出质量更高

**详细解释**：
口头表达更接近人自然思考的方式，不容易因为"懒得打字"而省略关键细节，从而让 AI 的理解更准确、产出更周全。

**实际应用场景**：
描述复杂需求、解释业务背景、或手边不方便打字时，用语音输入效率更高。

### 2.4 速查表

| 技巧 | 触发方式 | 核心效果 | 新手注意事项 |
|---|---|---|---|
| Plan 模式（先商量） | 说"别动手，给我方案" | 先对齐方案再执行，避免返工 | 狂编模式需在设置中手动勾选才能启用 |
| claude.md 规则 | 项目根目录新建 `claude.md` | 固化项目偏好，自动遵守 | 大项目建议项目规则和个人偏好分开存放 |
| Memory 规则 | 对话中说"记住这一点" | 跨项目通用的个人偏好持久化 | 可随时动态追加，无需手动编辑文件 |
| 语音输入 | 按住 `H` 键 | 输入速度提升约3倍，描述更详细 | 说话比打字更自然，不要刻意简化描述 |

### 2.5 延伸思考

1. **Plan 模式的边界在哪里？** 什么样的任务适合先走 Plan 模式，什么样的简单任务反而不需要这一步，直接让它动手更高效？

2. **claude.md 应该写多细？** 规则写得太少没有约束效果，写得太多又可能干扰 AI 的判断——如何找到"规则密度"的平衡点？

3. **语音输入会带来新的问题吗？** 口述容易产生口语化表达、逻辑跳跃或信息冗余，这些对 AI 的理解是帮助还是干扰？如何让语音输入的质量更稳定？

### 2.6 一句话记住它

> 用 Claude Code 的正确姿势是：**先商量方向、再用规则驯化它、最后用嘴喂饱它**——让 AI 越来越懂你，而不是每次从零开始。
