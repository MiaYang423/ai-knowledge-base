### 2.3 核心概念精讲

#### 概念一：斜杠命令（Slash Command）

**是什么**：Claude Code 中以 `/` 开头的快捷指令，用于触发特定功能操作。

**为什么重要**：斜杠命令是使用 Claude Code 的核心交互方式，不懂命令就等于不会用工具。

**详细解释**：
- 默认情况下，所有斜杠命令均为英文
- 中文用户需要反复查阅文档或开翻译才能理解每条命令的含义
- 使用频率高，英文障碍会严重拖慢工作效率

**实际应用场景**：每次在 Claude Code 中输入 `/` 触发命令列表时，汉化后可直接看到中文描述，无需翻译。

---

#### 概念二：全局汉化（深层设置）

**是什么**：通过修改 Claude Code 的底层配置，将界面命令描述统一替换为中文。

**为什么重要**：一次配置，永久生效，彻底解决英文命令看不懂的问题，不再依赖翻译工具。

**详细解释**：
- 作用范围：全局，覆盖所有斜杠命令
- 操作位置：深层设置（具体路径视频中演示）
- 效果：命令名称和描述变成"大白话中文"

**实际应用场景**：配置完成后，打开 Claude Code 的命令面板，所有功能一目了然，新手也能快速找到需要的命令。

---

#### 概念三：自动中文说明提示（"说明书"模式）

**是什么**：在每条命令执行时，自动在下方弹出粉红色中文说明框，解释当前操作的内容和原因。

**为什么重要**：让命令执行过程从"黑箱"变成"透明箱"，用户不仅知道结果，还能理解过程，边用边学。

**详细解释**：

| 属性 | 说明 |
|------|------|
| 触发时机 | 每次命令运行时自动弹出 |
| 显示位置 | 命令执行区域下方 |
| 显示样式 | 粉红色中文提示框 |
| 内容维度1 | 告诉你"它在干嘛" |
| 内容维度2 | 告诉你"为什么要这么干" |

**实际应用场景**：运行一条构建或调试命令时，提示框自动出现，解释"正在编译文件，因为检测到代码变更"，帮助新手理解工具行为逻辑。

### 2.5 延伸思考

1. **汉化配置的边界在哪里？** 全局汉化改变的是界面描述，还是会影响 Claude Code 实际执行命令的逻辑？如果底层命令仍是英文，汉化的本质是"翻译层"还是"替换层"？

2. **"说明书"提示对学习效率的影响是什么？** 自动弹出的中文解释能帮助新手更快理解工具原理，但长期依赖提示是否会阻碍用户真正掌握 Claude Code 的底层机制？

3. **"邪修"方法的可持续性如何？** Claude Code 版本更新后，这套汉化配置是否需要重新设置？有没有更官方或更稳定的中文支持路径值得期待？
