### 2.3 核心概念精讲

#### 概念一：Design Prompt Dev

**是什么**：一个聚合了大量前端 UI 风格 Prompt 的网站，可以一键复制风格描述 Prompt。

**为什么重要**：AI coding 工具（Claude Code、Cursor 等）需要明确的 Prompt 才能生成好看的界面，Design Prompt Dev 省去了自己描述风格的麻烦。

**详细解释**：
- 网站内提供多种前端视觉风格（极简、科技感、卡片式等）
- 点击"Get Prompt"直接复制该风格的描述 Prompt
- 将 Prompt 粘贴到 AI coding 工具中，让 AI 按该风格渲染前端界面

**实际应用场景**：做产品 demo 时想快速换一个好看的风格，不想自己写样式描述，直接用这个网站取 Prompt 扔给 AI。

---

#### 概念二：M3 Material IO（material.io）

**是什么**：谷歌官方开源的设计系统 Material Design 3，包含完整的设计规范、配色方案和 UI 组件。

**为什么重要**：设计规范是专业产品的基础，M3 是业界公认的成熟体系，用它做出来的产品自带专业感。

**详细解释**：
- 提供完整的颜色系统（Color Scheme）、排版规范、间距规则
- 包含大量标准组件（Button、Card、Navigation 等）
- 可以将设计规范文档或组件截图提供给 AI，让 AI 按谷歌规范生成界面

**实际应用场景**：做面向企业或需要专业形象的 AI 产品时，用 M3 规范约束 AI 生成的界面风格，避免五花八门的样式。

---

#### 概念三：Design MD（designmd.ai）

**是什么**：类似 Design Prompt Dev 的前端模板网站，额外支持下载 MD 文件和 MCP 接口连接。

**为什么重要**：除了 Prompt 模式，还支持 MCP 协议，可以让 Claude Code / Windsurf 直接调用，实现设计到代码的自动化打通。

**详细解释**：
- 提供多种前端模板，可下载为 Markdown 文件
- 复制模板内容（即一段 Prompt）发给 AI，让 AI 修改前端界面
- 支持 MCP 接口，AI coding 工具可直接连接，无需手动复制粘贴

**实际应用场景**：想要更自动化的工作流时，通过 MCP 让 Claude Code 直接读取 Design MD 的模板并应用到项目中。

---

#### 概念四：UI UI Pro Max（GitHub）

**是什么**：GitHub 上的一个专业前端 UI 组件库，适合深度 AI coding 场景下的产品界面设计。

**为什么重要**：当你认真做一个完整的 AI 产品（而不只是 demo）时，需要一套可复用、可扩展的 UI 体系，而不是一次性的 Prompt 风格。

**详细解释**：
- 下载后可在本地使用
- 根据产品需求选取对应组件，设计前端界面
- 适合长期迭代的 AI coding 产品

**实际应用场景**：自己在做一个完整的 AI 工具产品，需要统一的 UI 组件体系支撑多个页面和功能模块。
