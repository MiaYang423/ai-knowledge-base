## 二、核心教案

### 2.1 这个视频在讲什么（一句话）

**借鉴 Claude Code 的"渐进式披露"机制，用"按需学习 + 轻量存档"代替全量跟进，既节省时间和脑容量，又从根本上缓解 AI 信息焦虑。**

### 2.3 核心概念精讲

#### 概念一：Progressive Disclosure（渐进式披露）

**是什么**：Claude Code 的上下文管理机制——不在 Session 开始时把所有 Skill 全部加载，而是先只读取每个 Skill 的名称和简介，需要时才加载具体内容。

**为什么重要**：Context Window 的容量是有限的，如果一次性全量加载，窗口会立刻被占满，反而无法高效工作。

**详细解释**：

| 阶段 | Claude Code 的做法 | 对应的学习行为 |
|------|-------------------|--------------|
| 初次接触 | 读取 Skill 名称 + 简介 | 花 2~3 分钟了解工具是什么、能做什么 |
| 存档备查 | 记录 Skill 索引，不占用主窗口 | 把工具信息存入文件夹/笔记 |
| 按需深入 | 真正用到时才加载完整内容 | 有实际需求时再做深度学习 |

**实际应用场景**：在 Claude Code 中管理大量 Skill 时，避免 Context Window 过载，保持每次 Session 的高效运行。

---

#### 概念二：按需学习（Need-based Learning）

**是什么**：只在有明确使用需求或强烈兴趣时，才对某个 AI 工具或产品进行深度学习；其余情况只做轻量了解并存档。

**为什么重要**：AI 工具的发布速度远超个人消化速度，全量跟进既不现实，也会制造持续的焦虑感和时间浪费。

**详细解释**：

遇到新工具时，先问自己两个问题：

1. **我现在手上的项目能用到它吗？**
2. **我是否觉得它非常有意思、立马想尝试？**

| 判断结果 | 行动 |
|---------|------|
| 两个问题有一个答案为"是" | 立刻深入学习、上手尝试 |
| 两个问题答案都是"否"（约占 80~90%） | 花 2~3 分钟了解概貌，存档备查，暂不深入 |

**实际应用场景**：刷 X、小红书、科技新闻时看到新工具，用这套判断框架快速决策，避免被每一个"爆款工具"牵着走。

---

#### 概念三：轻量存档（Light Archiving）

**是什么**：对暂时不需要深入了解的工具或产品，用 2~3 分钟做最小化记录（是什么、能做什么），存入固定的文件夹或笔记，以备日后检索。

**为什么重要**：它是按需学习得以实现的基础设施——没有存档，"需要时再学"就变成了"需要时找不到"。

**详细解释**：

- 存档内容：工具名称、核心功能、来源链接（对应 Claude Code 的 Skill 名称 + 简介）
- 存档位置：固定的文件夹、笔记软件、书签等，能快速检索即可
- 触发深读：当某天真正遇到对应需求时，打开存档，再做完整学习

**实际应用场景**：某周刷到一个有趣的 AI 图像工具，当下不需要，花 2 分钟记下存档；三周后做一个需要图像处理的项目，翻出存档，再深入研究。

### 2.5 延伸思考

1. **存档本身也会过载吗？** 如果每次都存档，文件夹里积累的条目越来越多，检索成本也会上升——你会怎么给自己的"工具索引"做二次管理？

2. **"按需"的判断标准是否会随时间漂移？** 今天判断"用不到"的工具，可能三个月后成为行业标配。如何设计一个机制，定期重新审视存档中的内容，而不是让它永远沉睡？

3. **这套方法是否适用于所有类型的学习？** 渐进式披露在 AI 工具学习上似乎很有效，但对于需要系统性建构知识体系的领域（如编程基础、数学），"按需、碎片化"的方式是否依然成立？

---

### 2.6 一句话记住它

> 不是每个 AI 工具都值得你现在学——像 Claude Code 管理上下文一样管理你的注意力：先存档，真需要时再深入，焦虑自然就少了。
