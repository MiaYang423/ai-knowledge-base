# 教案：普通人用好这三个 Claude Skills 就够了

## 一、视频原文

其实你并不需要那么多 Claude Skills。最近 Claude Skills 有多火，相信大家都有所感受。目前 Skills 市场这个网站上 Skills 的数量已经达到了恐怖的 8 万个了，并且还在快速增长中。面对那么多新奇的 Skills，很多人像进了琳琅满目的超市一样，什么都想带回家，结果就是囤积了一堆 Skills，真正用上的没几个，彻底掉入效率陷阱。

那么作为创作者，我们筛选 Skills 的标准是什么？哪些是必装的？今天我从创作的三大环节——信息输入、内容输出、创作流程打包——从这三个角度给大家推荐三个核心 Skills，帮你在大家都能用 Skills 快速创作的时代保持差异化。

**1. 信息输入：Notebook Skill**

AI 时代下，人人都可以成为内容创作者，平台每天有海量的内容和你在竞争。如何让你的作品在一堆同质化内容中脱颖而出？我觉得核心就在于信息摄入的差异。没有内容是凭空捏造的，好的内容都是优质内容摄入的产物。我们要做的就是利用 Notebook Skill 这个工具管理我们信息源的质量。

Notebook Claude Skill 能够打通知识库与 Claude，直接对话。我们只用我们的审美和偏好来筛选内容，自动化的生产交给 Claude Skills。你可以在 Claude 直接上传 PDF、YouTube 链接到 Notebook LLM，基本也可以直接从 Claude 查询 Google Notebook LLM，获取 Gemini 提供的附带来源引用支持的答案。保持差异化的同时还能大幅减少幻觉，因为答案仅仅来自咱们上传的文档。

**2. 内容输出：Obsidian Skill**

创作者的输出包含笔记沉淀和将内容可视化地呈现给用户。我这里推荐 Obsidian，是因为它的本地存储、双链功能、Markdown 格式等特点，非常适合 AI 时代优质内容的自动化创作。生产后 Claude 可以直接帮你写入 Obsidian，不需要去折腾那些云端工具的授权和接口问题。

同时最近 Obsidian 的 CEO 亲自下场写了三个 Skills，能帮你自动写笔记、画 Canvas 架构图、创建 Base 数据库。很多人觉得这只是方便笔记管理而已，其实现在内容泛滥的时代，差异化的优质内容是不需要复杂包装的，最重要的是你的观点和选题足够有价值，以及你的分发效率有多快，根本不需要搞得太花里胡哨。我平时演示就直接在 Obsidian 里的 Excalidraw 画图，或者利用 Napkin 文字信息可视化的呈现能力做图，你也可以利用 Obsidian 使用 Skills 的 Canvas 架构图来进行信息传递。

**3. 将一切标准化：Skills Creator**

有了优质的内容输入和体系化高效率的内容输出，我们还需要将我们的创作过程标准化，封装成 Skills 持续使用，这样才会有规模效应。所以你需要 Skills Creator 这个官方 Skills，它能通过引导你进行自然语言对话来创建 Skills。你和它的头脑风暴，或者直接把当前的 AI 工作流发给它，也可以是你跟 Agent 的对话记录，根本不需要懂 Skills 的开发标准。

给 Claude 输入：把我们的对话整理好，包括我的提示词迭代和你的解法，存成一个 Skills，放到这个目录里，起个清晰的名字和描述，以后我会高频率复用。一个专属于你的自动化创作的 Skills 就搞定了。

以上三个 Skills 只是举例，大家可以试着结合自己的需求场景来进行替换和补充，底层逻辑就是求精不求多，这样才能举一反三，一通百通。以上就是今天分享的全部内容，如果有帮助到你，欢迎给个一键四连支持一下。我是 Jaggy，关注我，陪你在 AI 时代无限生长。

### 2.3 核心概念精讲

#### 概念一：Notebook Skill（信息输入管理）

**是什么**：一个能将外部文档（PDF、YouTube 等）导入知识库，并让 Claude 基于这些文档回答问题的 Skill，底层调用 Google Notebook LLM（Gemini 驱动）。

**为什么重要**：
AI 时代内容同质化严重，差异化的核心在于"信息摄入的差异"。好内容是优质信息输入的产物，而不是凭空生成的。

**详细解释**：

| 功能点 | 说明 |
|---|---|
| 支持导入格式 | PDF 文件、YouTube 视频链接 |
| 回答来源 | 仅基于你上传的文档，附带来源引用 |
| 核心优势 | 大幅减少幻觉，答案可溯源 |
| 操作方式 | 在 Claude 界面直接上传或查询，无需额外配置 |

**实际应用场景**：
把你精心筛选的行业报告 PDF、优质 YouTube 视频导入知识库，让 Claude 基于这些内容创作，而不是依赖通用训练数据，从而产出别人写不出来的内容。

---

#### 概念二：Obsidian Skill（内容输出体系）

**是什么**：基于 Obsidian 笔记软件的 Claude Skill，支持 Claude 直接将生成内容写入本地 Obsidian 库，并通过官方 Skills 实现笔记、架构图、数据库的自动化创建。

**为什么重要**：
创作者的输出不只是"写完发出去"，还需要沉淀和可视化。Obsidian 的本地存储特性避免了云端工具的授权和接口问题，让 Claude 写入更顺畅。

**详细解释**：

| Obsidian 特点 | 对创作的价值 |
|---|---|
| 本地存储 | 无需云端授权，Claude 可直接写入 |
| 双链功能 | 知识点之间相互关联，便于复用 |
| Markdown 格式 | 与 AI 生成内容天然兼容 |
| Excalidraw 插件 | 直接在库内手绘架构图 |
| Canvas 架构图（官方 Skill）| 自动生成可视化信息结构 |

**实际应用场景**：
用 Claude 生成内容后，直接指令写入 Obsidian 对应目录；需要配图时在 Excalidraw 里画图，或用 Napkin 做文字可视化，全程不离开本地环境。

---

#### 概念三：Skills Creator（流程标准化封装）

**是什么**：Claude 官方提供的 Skill，通过自然语言对话引导用户把自己的工作流或提示词迭代过程封装成一个可复用的专属 Skill。

**为什么重要**：
有了好的输入和输出工具，如果每次都要重新描述流程，效率依然低下。将创作过程标准化封装，才能产生规模效应，让重复工作一键完成。

**详细解释**：

| 输入方式 | 示例 |
|---|---|
| 工作流描述 | 直接把当前 AI 工作流文字发给它 |
| 对话记录 | 把你与 Agent 的历史对话粘贴进去 |
| 提示词迭代记录 | 把多轮调试过的提示词和解法一起发给它 |

封装指令示例（视频原话）：
> "把我们的对话整理好，包括我的提示词迭代和你的解法，存成一个 Skill，放到这个目录里，起个清晰的名字和描述，以后我会高频率复用。"

**实际应用场景**：
当你摸索出一套稳定的选题→写作→配图创作流程后，用 Skills Creator 一次性封装，之后只需调用这个 Skill，无需重复输入背景和要求。

### 2.4 速查表

| 维度 | Notebook Skill | Obsidian Skill | Skills Creator |
|---|---|---|---|
| **对应环节** | 信息输入 | 内容输出 | 流程标准化 |
| **解决的问题** | 内容同质化、信息来源不可靠 | 输出散乱、可视化麻烦 | 流程无法复用、效率无法规模化 |
| **核心能力** | 基于私有文档问答，附来源引用 | 本地自动写入笔记、画图、建库 | 自然语言创建专属 Skill |
| **底层技术** | Google Notebook LLM（Gemini） | Obsidian 本地存储 + Markdown | Claude 官方 Skill 生成器 |
| **上手难度** | 低（上传文件即用） | 低（无需云端授权） | 低（不需要懂开发） |
| **核心价值** | 减少幻觉，保持信息差异化 | 提升分发效率，内容可沉淀 | 一次封装，高频复用 |

### 2.5 延伸思考

1. **信息筛选本身也是核心竞争力**：Notebook Skill 的价值取决于你导入的文档质量，那么在"用什么工具"之前，"如何建立自己的高质量信息源筛选标准"是不是更值得优先思考？

2. **标准化的边界在哪里**：Skills Creator 能把创作流程封装复用，但创作中有多少环节适合标准化、又有多少环节恰恰需要保留"非标准化"的个人判断？过度封装会不会反而削弱创作的差异化？

3. **工具组合的迁移成本**：视频推荐的三个 Skill 深度绑定了 Notebook LLM、Obsidian、Claude 这一套生态，如果其中某个工具发生重大变化（如 API 政策调整），整套系统的稳定性如何保障？是否需要预留替代方案？

---

### 2.6 一句话记住它

> 不是 Skills 越多越强——围绕"输入→输出→标准化"三个环节各选一个精准 Skill，才是普通创作者在 AI 时代建立差异化的正确姿势。
